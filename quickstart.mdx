Get your API key, install the SDK, and display a feed. Your account comes with demo content so you can see results immediately.

## Prerequisites

| Requirement | Details |
|---|---|
| ShortKit account | Sign up at [portal.shortkit.dev](https://portal.shortkit.dev) or [contact us](https://calendly.com/michael-shortkit/20-min-meeting) |
| Publishable key | `pk_live_...` from **Settings > API Keys** in the portal |
| SDK installed | Follow the [Installation](/sdk/installation) guide for your platform |

## 1. Install the SDK

If you haven't already, add ShortKit to your project. See [Installation](/sdk/installation) for full details and troubleshooting.

<Tabs>
  <Tab title="Swift">
    In Xcode, go to **File > Add Package Dependencies** and enter:

    ```
    https://github.com/shortkit/shortkit-ios
    ```

    Select the `ShortKitSDK` product and add it to your app target.
  </Tab>
  <Tab title="Kotlin">
    Add to your app-level `build.gradle.kts`:

    ```kotlin build.gradle.kts
    dependencies {
        implementation("dev.shortkit:shortkit:0.2.12")
    }
    ```

    Sync your project.
  </Tab>
  <Tab title="React Native">
    ```bash
    npm install @shortkitsdk/react-native
    cd ios && pod install
    ```
  </Tab>
  <Tab title="Expo">
    ```bash
    npx expo install @shortkitsdk/react-native
    ```

    Add the config plugin to your `app.json`:

    ```json app.json
    {
      "expo": {
        "plugins": [
          "@shortkitsdk/react-native"
        ]
      }
    }
    ```

    Then regenerate native projects:

    ```bash
    npx expo prebuild --clean
    ```
  </Tab>
  <Tab title="Web">
    Add the script tag to your HTML:

    ```html
    <script src="https://sdk.shortkit.dev/web/0.2.11/shortkit.min.js"></script>
    ```
  </Tab>
</Tabs>

## 2. Display the feed

Every new account is seeded with demo videos, so your feed has content right away. Create a new view (or add to an existing one) and pass your publishable key:

<Tabs>
  <Tab title="SwiftUI">
    ```swift FeedScreen.swift
    import SwiftUI
    import ShortKitSDK

    struct FeedScreen: View {
        let shortKit = ShortKit(
            apiKey: "pk_live_your_publishable_key",
            config: FeedConfig()
        )

        var body: some View {
            ShortKitFeedView(shortKit: shortKit)
                .ignoresSafeArea()
        }
    }
    ```

    Add `FeedScreen()` to your app's navigation or set it as your root view.
  </Tab>
  <Tab title="UIKit">
    ```swift FeedViewController.swift
    import UIKit
    import ShortKitSDK

    class FeedViewController: UIViewController {
        private let shortKit = ShortKit(
            apiKey: "pk_live_your_publishable_key",
            config: FeedConfig()
        )

        override func viewDidLoad() {
            super.viewDidLoad()

            let feedVC = ShortKitFeedViewController(shortKit: shortKit)
            addChild(feedVC)
            feedVC.view.frame = view.bounds
            feedVC.view.autoresizingMask = [.flexibleWidth, .flexibleHeight]
            view.addSubview(feedVC.view)
            feedVC.didMove(toParent: self)
        }
    }
    ```

    Present or push `FeedViewController` from your navigation stack.
  </Tab>
  <Tab title="Kotlin">
    ```kotlin FeedActivity.kt
    import android.os.Bundle
    import androidx.appcompat.app.AppCompatActivity
    import com.shortkit.sdk.ShortKit
    import com.shortkit.sdk.config.FeedConfig
    import com.shortkit.sdk.feed.ShortKitFeedFragment

    class FeedActivity : AppCompatActivity() {
        override fun onCreate(savedInstanceState: Bundle?) {
            super.onCreate(savedInstanceState)
            setContentView(R.layout.activity_feed)

            val shortKit = ShortKit(
                context = this,
                apiKey = "pk_live_your_publishable_key",
                config = FeedConfig()
            )

            if (savedInstanceState == null) {
                supportFragmentManager.beginTransaction()
                    .replace(R.id.feed_container, ShortKitFeedFragment.newInstance(shortKit))
                    .commit()
            }
        }
    }
    ```

    Your layout needs a container for the fragment:

    ```xml res/layout/activity_feed.xml
    <?xml version="1.0" encoding="utf-8"?>
    <FrameLayout xmlns:android="http://schemas.android.com/apk/res/android"
        android:id="@+id/feed_container"
        android:layout_width="match_parent"
        android:layout_height="match_parent" />
    ```
  </Tab>
  <Tab title="React Native">
    ```tsx FeedScreen.tsx
    import React from 'react';
    import { ShortKitProvider, ShortKitFeed } from '@shortkitsdk/react-native';

    export default function FeedScreen() {
      return (
        <ShortKitProvider apiKey="pk_live_your_publishable_key" config={{}}>
          <ShortKitFeed style={{ flex: 1 }} />
        </ShortKitProvider>
      );
    }
    ```
  </Tab>
  <Tab title="Expo">
    ```tsx FeedScreen.tsx
    import React from 'react';

    ```

    Run with `npx expo run:ios` or `npx expo run:android`.
  </Tab>
  <Tab title="Web">
    ```html index.html
    <!DOCTYPE html>
    <html>
    <head>
      <meta charset="utf-8" />
      <meta name="viewport" content="width=device-width, initial-scale=1" />
      <title>ShortKit Feed</title>
      <style>
        body { margin: 0; }
        #feed { width: 100%; height: 100vh; }
      </style>
    </head>
    <body>
      <div id="feed"></div>
      <script src="https://sdk.shortkit.dev/web/0.2.11/shortkit.min.js"></script>
      <script>
        const { ShortKit } = window.ShortKit;
        const sk = new ShortKit({ apiKey: 'pk_live_your_publishable_key' });
        sk.createFeed(document.getElementById('feed'));
      </script>
    </body>
    </html>
    ```
  </Tab>
</Tabs>

Run your app. You should see a full-screen, vertically-scrolling video feed with the demo content.

## Adding your own content

The demo videos are there to get you started. When you're ready to add your own content, you have three options:

- **Portal upload.** Go to **Content** in the [admin portal](https://portal.shortkit.dev) and upload video files directly.
- **Server-side import.** Use the [Import API](/api/content/import) to bulk import from URLs (S3, GCS, your CDN, or even TikTok/Instagram).
- **Direct upload.** Use the [Upload API](/api/content/upload) to get signed URLs for client-side or server-side file uploads.

See the [Upload Pipeline](/guides/upload-pipeline) guide for the full walkthrough.

## Next steps

Now that you have a working feed with demo content, here's the typical path forward:

1. **Upload your own content.** Replace the demo videos with your own. You can [upload through the portal](/guides/upload-pipeline), [import from URLs](/api/content/import) (including TikTok and Instagram), or [upload directly](/api/content/upload) from your server or client app.
2. **Build an overlay.** Add your own UI on top of the video player: titles, like buttons, share sheets, author info. See [Overlays](/sdk/overlays).
3. **Configure the feed.** Adjust feed height, mute behavior, and feed source. See [Configuration](/sdk/configuration).
4. **Identify users.** Link feed activity to your logged-in users with `setUserId` so analytics and future recommendations are per-user. See [Identity](/sdk/identity).
5. **Track engagement.** Query watch time, completion rates, and custom events. See [Analytics](/sdk/analytics).
