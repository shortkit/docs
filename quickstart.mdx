Get your API key, upload a video, and display a feed.

## Prerequisites

| Requirement | Details |
|---|---|
| ShortKit account | Sign up at [portal.shortkit.dev](https://portal.shortkit.dev) or [contact us](https://calendly.com/michael-shortkit/20-min-meeting) |
| API keys | Publishable key (`pk_live_...`) from **Settings > API Keys** in the portal |
| Platform | iOS 16+, Android 8+ (API 24), React Native 0.76+, Expo SDK 52+, or any modern browser |

## 1. Install the SDK

<Tabs>
  <Tab title="Swift">
    In Xcode, go to **File > Add Package Dependencies** and enter:

    ```
    https://github.com/shortkit/shortkit-ios
    ```

    Or add to your `Package.swift`:

    ```swift Package.swift
    dependencies: [
        .package(url: "https://github.com/shortkit/shortkit-ios.git", from: "0.1.0")
    ]
    ```

    Add `ShortKitSDK` to your target's dependencies.
  </Tab>
  <Tab title="Kotlin">
    Add the dependency to your app-level `build.gradle.kts`:

    ```kotlin build.gradle.kts
    dependencies {
        implementation("dev.shortkit:shortkit:0.1.0")
    }
    ```

    Sync your project. No custom repository URL is needed — ShortKit is published to Maven Central.
  </Tab>
  <Tab title="React Native">
    ```bash
    npm install @shortkitsdk/react-native
    ```
  </Tab>
  <Tab title="Expo">
    ```bash
    npx expo install @shortkitsdk/react-native
    ```

    Add the config plugin to your `app.json`:

    ```json app.json
    {
      "expo": {
        "plugins": [
          "@shortkitsdk/react-native"
        ]
      }
    }
    ```

    Then regenerate native projects:

    ```bash
    npx expo prebuild --clean
    ```
  </Tab>
  <Tab title="Web">
    No installation needed. Add the script tag in step 5.
  </Tab>
</Tabs>

## 2. Upload a video

Your feed needs content. Upload a video through the portal or the API.

**Option A: Portal** — Go to **Content** in the [admin portal](https://portal.shortkit.dev) and upload a video file.

**Option B: API** — Create an upload session and PUT your file to the signed URL:

```bash
# Create an upload session
curl -X POST https://api.shortkit.dev/v1/uploads \
  -H "Authorization: Bearer sk_live_your_secret_key" \
  -H "Content-Type: application/json" \
  -d '{"title": "My First Video"}'

# Upload the file to the signed URL returned in the response
curl -X PUT "<uploadUrl from response>" \
  -H "Content-Type: video/mp4" \
  --data-binary @video.mp4
```

Processing takes a few seconds. The video status changes from `waiting_for_upload` to `processing` to `ready`. You can poll `GET /v1/uploads/{id}` to check, or watch the status in the portal.

<Info>
  See the [upload pipeline guide](/guides/upload-pipeline) for details on metadata, captions, and carousel content.
</Info>

## 3. Create an embed

An embed defines which content appears in your feed and how it behaves. Create one in the portal or via the API.

**Option A: Portal** — Go to **Embeds** in the [admin portal](https://portal.shortkit.dev), click **New Embed**, choose the **Feed** mode, and save. Copy the embed ID (`emb_...`) from the page header.

**Option B: API**:

```bash
curl -X POST https://api.shortkit.dev/v1/embeds \
  -H "Authorization: Bearer sk_live_your_secret_key" \
  -H "Content-Type: application/json" \
  -d '{"name": "My App Feed", "mode": "feed"}'
```

The response includes the embed ID:

```json
{
  "data": {
    "id": "emb_abc123",
    "name": "My App Feed",
    "mode": "feed",
    ...
  }
}
```

## 4. Display the feed

Pass your publishable key and embed ID to the SDK to render a full-screen video feed.

<Tabs>
  <Tab title="SwiftUI">
    ```swift
    import SwiftUI
    import ShortKitSDK

    struct FeedScreen: View {
        let shortKit = ShortKit(
            apiKey: "pk_live_your_publishable_key",
            config: FeedConfig(),
            embedId: "emb_your_embed_id"
        )

        var body: some View {
            ShortKitFeedView(shortKit: shortKit)
                .ignoresSafeArea()
        }
    }
    ```
  </Tab>
  <Tab title="UIKit">
    ```swift
    import UIKit
    import ShortKitSDK

    class FeedViewController: UIViewController {
        override func viewDidLoad() {
            super.viewDidLoad()

            let shortKit = ShortKit(
                apiKey: "pk_live_your_publishable_key",
                config: FeedConfig(),
                embedId: "emb_your_embed_id"
            )
            let feedVC = ShortKitFeedViewController(shortKit: shortKit)
            addChild(feedVC)
            feedVC.view.frame = view.bounds
            feedVC.view.autoresizingMask = [.flexibleWidth, .flexibleHeight]
            view.addSubview(feedVC.view)
            feedVC.didMove(toParent: self)
        }
    }
    ```
  </Tab>
  <Tab title="Kotlin">
    ```kotlin
    import com.shortkit.sdk.ShortKit
    import com.shortkit.sdk.config.FeedConfig
    import com.shortkit.sdk.feed.ShortKitFeedFragment

    class FeedActivity : AppCompatActivity() {
        override fun onCreate(savedInstanceState: Bundle?) {
            super.onCreate(savedInstanceState)
            setContentView(R.layout.activity_feed)

            val shortKit = ShortKit(
                context = this,
                apiKey = "pk_live_your_publishable_key",
                embedId = "emb_your_embed_id",
                config = FeedConfig()
            )

            supportFragmentManager.beginTransaction()
                .replace(R.id.container, ShortKitFeedFragment.newInstance(shortKit))
                .commit()
        }
    }
    ```
  </Tab>
  <Tab title="React Native">
    ```tsx

    ```
  </Tab>

Run your app. You should see a vertically-scrolling video feed with your uploaded content.

## Next steps

You have a working feed. Here's where to go from here:

- **Add an overlay** — Display titles, descriptions, and action buttons on top of your videos. See [Overlays](/sdk/overlays) and [Custom overlays](/guides/custom-overlays).
- **Configure feed behavior** — Adjust height, mute behavior, carousel and survey modes. See [Configuration](/sdk/configuration).
- **Track engagement** — Pass custom dimensions and query analytics. See [Analytics](/sdk/analytics).
- **Identify users** — Link feed activity to your logged-in users with `setUserId`. See [User identity](/sdk/identity).
