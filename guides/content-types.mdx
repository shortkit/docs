ShortKit feeds are polymorphic. Each position in the feed can be a video, a live stream, an image carousel, a video carousel, a survey, or an ad slot. The SDK represents this with the `FeedItem` enum, and each case wraps a typed model with all the data you need to render it.

## Feed item types at a glance

| Type | Model | Purpose |
|---|---|---|
| Video | `ContentItem` | HLS video with metadata, captions, and custom fields |
| Live stream | `ContentItem` with `isLive: true` | Real-time broadcast with a live viewer count. See [Live Streams](/guides/live-streams). |
| Image carousel | `ImageCarouselItem` | Horizontally paged images with optional auto-scroll |
| Video carousel | `VideoCarouselItem` | Horizontally paged videos with carousel-level metadata |
| Survey | `SurveyItem` | Single-question poll with selectable options |
| Ad slot | `AdSlot` | Server-positioned ad placement filled by your ad provider |

Videos are the primary content type. Each video is an HLS stream with optional caption tracks and arbitrary custom metadata.

### Custom metadata

`customMetadata` is a `[String: JSONValue]` dictionary that accepts strings, numbers, booleans, nested objects, and null. Values are set when you upload content through the API or portal and delivered to the SDK as-is.

Custom metadata can also be used as a filter dimension. Set `FeedFilter(metadata: ["country": "Japan"])` on your `FeedConfig` to show only content with matching metadata values.

### Fallback URLs

`fallbackUrl` is an optional static MP4 URL that the SDK uses when the primary HLS stream fails. This is useful when migrating a video library to ShortKit — you can provide your existing hosted URLs as fallbacks while transitioning to ShortKit's streaming infrastructure.

The SDK activates the fallback automatically in three scenarios:

1. **Player error** — The HLS stream fails during playback. The SDK switches to the fallback MP4 and resumes from the same position.
2. **Proactive next-slot detection** — The SDK pre-watches the next video in the feed. If its HLS stream fails before the user swipes, the SDK swaps in the fallback so the transition is seamless.
3. **Missing primary URL** — If the primary `streamingUrl` is empty or invalid, the SDK uses the fallback immediately.

Fallback activation is tracked as a `playbackFallback` engagement event with a `reason` field (`player_failed`, `proactive_next_slot`, or `no_primary_url`).

<Note>
Fallback URLs are only used in custom feed mode with `FeedInput.video`. Pass the fallback URL when constructing your feed items.
</Note>

|---|---|---|
| `id` | `String` | Unique video carousel identifier |

<Note>
The `initialPageIndex` field is available on the iOS and React Native SDKs. Android and Flutter SDK support ships in an upcoming release — on those platforms the field is silently ignored and the carousel always lands on page 0.
</Note>

### How video carousels differ from image carousels

| Aspect | Image carousel | Video carousel |
|---|---|---|
| **Content** | Static images | HLS video streams |
| **Paging** | Managed by your overlay (JS FlatList in RN) | Managed natively by the SDK (ViewPager2 / UICollectionView) |
| **Playback** | No player | SDK handles player assignment, buffering, and handoff between pages |
| **Overlay role** | Full UI: images, paging, text, actions | Chrome only: titles, buttons, page indicators. The SDK renders the video underneath. |

## Ad slot

Ad slots are server-positioned placeholders in the feed that get filled by your ad provider at runtime. The position and targeting configuration are set in the Admin Portal or via the API. The SDK loads and displays ads automatically when an `adProvider` is configured.

## The FeedItem enum

The SDK wraps all five content types in a single discriminated union. The API response includes a `"type"` field (`"content"`, `"image_carousel"`, `"video_carousel"`, `"survey"`, `"ad_slot"`) that the SDK uses for deserialization. Missing `"type"` defaults to `"content"` for backward compatibility.

<Tabs>
  <Tab title="Swift">

        case content(ContentItem)
        case adSlot(AdSlot)
        case imageCarousel(ImageCarouselItem)

        public var isAdSlot: Bool
        public var isImageCarousel: Bool
        public var isVideoCarousel: Bool
        public var isSurvey: Bool
        public var isNonVideo: Bool

        // Typed unwrap accessors (return nil if the case doesn't match)
        public var contentItem: ContentItem?
        public var adSlot: AdSlot?
        public var imageCarouselItem: ImageCarouselItem?
        public var videoCarouselItem: VideoCarouselItem?

## Handling each type in a custom UI

When you set `videoOverlay: .none`, `carouselOverlay: .none`, `videoCarouselOverlay: .none`, and `surveyOverlay: .none`, the SDK renders raw content without any overlay UI. You build your own by switching on `FeedItem` and using the data available on each model.

<Tabs>
  <Tab title="Swift">
    ```swift
    shortKit.player.currentItem
        .receive(on: DispatchQueue.main)
        .sink { [weak self] item in
            guard let item else { return }
            self?.titleLabel.text = item.title
            self?.authorLabel.text = item.author
            self?.descriptionLabel.text = item.description

            // Access custom metadata
            if case .string(let section) = item.customMetadata?["section"] {
                self?.sectionLabel.text = section
            }

            // Check for captions
            if let track = item.captionTracks.first {
                self?.captionsButton.isHidden = false
            }
        }
        .store(in: &cancellables)
    ```

    For non-video types, switch on the `FeedItem` enum directly:

    ```swift
    switch feedItem {
    case .content(let video):
        // video.streamingUrl — HLS manifest
        // video.captionTracks — available subtitles
        // video.customMetadata — app-specific fields
        configureVideoUI(video)

    case .imageCarousel(let carousel):
        // carousel.images — array of CarouselImage (url + alt)
        // carousel.autoScrollInterval — seconds between pages, or nil
        // carousel.title, carousel.author, carousel.section — text metadata
        configureCarouselUI(carousel)

    case .videoCarousel(let videoCarousel):
        // videoCarousel.videos — array of ContentItem (each with playback metadata, captions, etc.)
        // videoCarousel.title, videoCarousel.author, videoCarousel.section — carousel-level metadata
        // videoCarousel.articleUrl — link to related article
        // Note: the SDK handles horizontal paging and video playback natively.
        // Your overlay provides chrome (titles, buttons, page indicators) on top.
        configureVideoCarouselUI(videoCarousel)

    case .survey(let survey):
        // survey.question — the question string
        // survey.options — array of SurveyOption (id + text)
        // survey.autoAdvanceDelay — seconds after selection, or nil
        configureSurveyUI(survey)

    case .adSlot(let slot):
        // Handled by the ad provider — no custom UI needed
        break
    }
    ```
  </Tab>
  <Tab title="Kotlin">
    ```kotlin
    when (feedItem) {
        is FeedItem.Content -> {
            val video = feedItem.item
            // video.streamingUrl — HLS manifest
            // video.captionTracks — available subtitles
            // video.customMetadata — app-specific fields
            configureVideoUI(video)
        }
        is FeedItem.ImageCarousel -> {
            val carousel = feedItem.item
            // carousel.images — list of CarouselImage (url + alt)
            // carousel.autoScrollInterval — seconds between pages, or null
            configureCarouselUI(carousel)
        }
        is FeedItem.VideoCarousel -> {
            val videoCarousel = feedItem.item
            // videoCarousel.videos — list of ContentItem (each with playback metadata, captions, etc.)
            // videoCarousel.title, videoCarousel.author, videoCarousel.section — carousel-level metadata
            // SDK handles paging and playback; overlay provides chrome on top.
            configureVideoCarouselUI(videoCarousel)
        }
        is FeedItem.Survey -> {
            val survey = feedItem.item
            // survey.question — the question string
            // survey.options — list of SurveyOption (id + text)
            configureSurveyUI(survey)
        }
        is FeedItem.AdSlot -> {
            // Handled by the ad provider
        }
    }
    ```
