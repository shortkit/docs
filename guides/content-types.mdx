ShortKit feeds are polymorphic. Each position in the feed can be a video, an image carousel, a survey, or an ad slot. The SDK represents this with the `FeedItem` enum, and each case wraps a typed model with all the data you need to render it.

## Feed item types at a glance

| Type | Model | Purpose |
|---|---|---|
| Video | `ContentItem` | HLS video with metadata, captions, and custom fields |
| Image carousel | `ImageCarouselItem` | Horizontally paged images with optional auto-scroll |
| Survey | `SurveyItem` | Single-question poll with selectable options |
| Ad slot | `AdSlot` | Server-positioned ad placement filled by your ad provider |

---

## Video

Videos are the primary content type. Each video is an HLS stream with optional caption tracks and arbitrary custom metadata.

| Field | Type | Description |
|---|---|---|
| `id` | `String` | Unique content identifier |
| `playbackId` | `String?` | Playback ID for this content item. Used when referencing content in custom feeds via `FeedInput.video(playbackId:)`. |
| `title` | `String` | Display title |
| `description` | `String?` | Optional body text |
| `duration` | `Double` | Video length in seconds |
| `streamingUrl` | `String` | HLS manifest URL (`.m3u8`). The SDK handles adaptive bitrate selection, buffering, and format switching automatically. |
| `thumbnailUrl` | `String` | Poster image URL, displayed while the video loads |
| `captionTracks` | `[CaptionTrack]` | Available caption/subtitle tracks (see below) |
| `customMetadata` | `[String: JSONValue]?` | Arbitrary key-value pairs attached at upload time. Use this for section labels, tags, or any app-specific data. |
| `author` | `String?` | Byline text |
| `articleUrl` | `String?` | Link to a full article. When present, the news overlay template shows a "Read more" button. |
| `commentCount` | `Int?` | Comment count displayed by the news overlay template |

### Caption tracks

Each video can include multiple caption tracks. The SDK loads and renders captions automatically when the user enables them.

### Custom metadata

`customMetadata` is a `[String: JSONValue]` dictionary that accepts strings, numbers, booleans, nested objects, and null. Values are set when you upload content through the API or portal and delivered to the SDK as-is.

Custom metadata can also be used as a filter dimension. Set `FeedFilter(metadata: ["country": "Japan"])` on your `FeedConfig` to show only content with matching metadata values. See [Feed Ranking — Server-side feed filtering](/guides/feed-customization#server-side-feed-filtering) for details.

<Tabs>
  <Tab title="Swift">
    ```swift
    // Reading custom metadata from a ContentItem
    if let section = item.customMetadata?["section"],
       case .string(let value) = section {
        sectionLabel.text = value.uppercased()
    }

    // JSONValue cases
    public enum JSONValue: Codable, Equatable, Sendable {
        case string(String)
        case number(Double)
        case bool(Bool)
        case object([String: JSONValue])
        case null
    }
    ```
  </Tab>
  <Tab title="Kotlin">
    ```kotlin
    val section = item.customMetadata?.get("section")?.asString()
    sectionLabel.text = section?.uppercase()
    ```
  </Tab>
  <Tab title="React Native">
    ```typescript
    const section = item.customMetadata?.["section"];
    if (typeof section === "string") {
      return <Text>{section.toUpperCase()}</Text>;
    }
    ```

### Receiving survey responses

Survey responses are delivered through the `SurveyOverlay` protocol's `onSurveyResponse` callback. When using a custom survey overlay, the SDK sets this callback on your overlay instance. Call it when the user selects an option. See [Overlays — Survey overlay](/sdk/overlays#survey-overlay) for the full protocol reference.

## Ad slot

Ad slots are server-positioned placeholders in the feed that get filled by your ad provider at runtime. The position and targeting configuration are set in the [Admin Portal](/portal/ad-management) or via the API. The SDK loads and displays ads automatically when an `adProvider` is configured.

For full ad integration details, see [Ad integration](/sdk/ads).

## The FeedItem enum

The SDK wraps all four content types in a single discriminated union. The API response includes a `"type"` field (`"content"`, `"image_carousel"`, `"survey"`, `"ad_slot"`) that the SDK uses for deserialization. Missing `"type"` defaults to `"content"` for backward compatibility.

<Tabs>
  <Tab title="Swift">
    ```swift
    public enum FeedItem: Sendable, Hashable, Codable {
        case content(ContentItem)
        case adSlot(AdSlot)
        case imageCarousel(ImageCarouselItem)
        case survey(SurveyItem)

        // Convenience booleans
        public var isContent: Bool
        public var isAdSlot: Bool
        public var isImageCarousel: Bool
        public var isSurvey: Bool
        public var isNonVideo: Bool

        // Typed unwrap accessors (return nil if the case doesn't match)
        public var contentItem: ContentItem?
        public var adSlot: AdSlot?
        public var imageCarouselItem: ImageCarouselItem?
        public var surveyItem: SurveyItem?

        // Stable identifier for diffable data source
        public var id: String
    }
    ```
  </Tab>
  <Tab title="Kotlin">
    ```kotlin
    sealed class FeedItem {
        data class Content(val item: ContentItem) : FeedItem()
        data class AdSlot(val slot: dev.shortkit.sdk.AdSlot) : FeedItem()
        data class ImageCarousel(val item: ImageCarouselItem) : FeedItem()
        data class Survey(val item: SurveyItem) : FeedItem()

        val id: String
        val isContent: Boolean
        val isAdSlot: Boolean
        val isImageCarousel: Boolean
        val isSurvey: Boolean
        val isNonVideo: Boolean
    }
    ```
  </Tab>
  <Tab title="React Native">
    ```typescript
    type FeedItem =
      | { type: "content"; item: ContentItem }
      | { type: "ad_slot"; item: AdSlot }
      | { type: "image_carousel"; item: ImageCarouselItem }
      | { type: "survey"; item: SurveyItem };
    ```
  </Tab>
</Tabs>

## Handling each type in a custom UI

When you set `videoOverlay: .none`, `carouselOverlay: .none`, and `surveyOverlay: .none`, the SDK renders raw content without any overlay UI. You build your own by switching on `FeedItem` and using the data available on each model.

<Tabs>
  <Tab title="Swift">
    ```swift
    shortKit.player.currentItem
        .receive(on: DispatchQueue.main)
        .sink { [weak self] item in
            guard let item else { return }
            self?.titleLabel.text = item.title
            self?.authorLabel.text = item.author
            self?.descriptionLabel.text = item.description

            // Access custom metadata
            if case .string(let section) = item.customMetadata?["section"] {
                self?.sectionLabel.text = section
            }

            // Check for captions
            if let track = item.captionTracks.first {
                self?.captionsButton.isHidden = false
            }
        }
        .store(in: &cancellables)
    ```

    For non-video types, switch on the `FeedItem` enum directly:

    ```swift
    switch feedItem {
    case .content(let video):
        // video.streamingUrl — HLS manifest
        // video.captionTracks — available subtitles
        // video.customMetadata — app-specific fields
        configureVideoUI(video)

    case .imageCarousel(let carousel):
        // carousel.images — array of CarouselImage (url + alt)
        // carousel.autoScrollInterval — seconds between pages, or nil
        // carousel.title, carousel.author, carousel.section — text metadata
        configureCarouselUI(carousel)

    case .survey(let survey):
        // survey.question — the question string
        // survey.options — array of SurveyOption (id + text)
        // survey.autoAdvanceDelay — seconds after selection, or nil
        configureSurveyUI(survey)

    case .adSlot(let slot):
        // Handled by the ad provider — no custom UI needed
        break
    }
    ```
  </Tab>
  <Tab title="Kotlin">
    ```kotlin
    when (feedItem) {
        is FeedItem.Content -> {
            val video = feedItem.item
            // video.streamingUrl — HLS manifest
            // video.captionTracks — available subtitles
            // video.customMetadata — app-specific fields
            configureVideoUI(video)
        }
        is FeedItem.ImageCarousel -> {
            val carousel = feedItem.item
            // carousel.images — list of CarouselImage (url + alt)
            // carousel.autoScrollInterval — seconds between pages, or null
            configureCarouselUI(carousel)
        }
        is FeedItem.Survey -> {
            val survey = feedItem.item
            // survey.question — the question string
            // survey.options — list of SurveyOption (id + text)
            configureSurveyUI(survey)
        }
        is FeedItem.AdSlot -> {
            // Handled by the ad provider
        }
    }
    ```
  </Tab>
  <Tab title="React Native">
    ```tsx
    function FeedItemView({ feedItem }: { feedItem: FeedItem }) {
      switch (feedItem.type) {
        case "content":
          return <VideoPlayer item={feedItem.item} />;

        case "image_carousel":
          return <CarouselView item={feedItem.item} />;

        case "survey":
          return <SurveyView item={feedItem.item} />;

        case "ad_slot":
          // Handled by the ad provider
          return null;
      }
    }
    ```
  </Tab>
</Tabs>
