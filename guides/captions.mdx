ShortKit supports WebVTT captions that are auto-generated during transcoding, embedded in the HLS stream, or manually uploaded as external `.vtt` files. Captions are delivered per-video as `CaptionTrack` objects on `ContentItem.captionTracks`.

## CaptionTrack model

Each video can have zero or more caption tracks. The `source` field tells you where the track came from.

<Tabs>
  <Tab title="Swift">
    ```swift
    public struct CaptionTrack: Codable, Equatable, Sendable {
        public let language: String       // BCP 47 language code, e.g. "en"
        public let label: String          // Human-readable label, e.g. "English"
        public let source: CaptionSource  // Where this track originated
        public let url: String?           // URL to the .vtt file (nil for embedded tracks)
    }

    public enum CaptionSource: String, Codable, Sendable {
        case embedded   // Baked into the HLS manifest
        case external   // Uploaded .vtt file hosted on your CDN
        case generated  // Auto-generated during transcoding
    }
    ```
  </Tab>
  <Tab title="Kotlin">

        GENERATED

  </Tab>

## Selecting a caption track

When multiple tracks are available (e.g. English and Spanish), select one by BCP 47 language code. If the requested language does not exist on the current video, the call is a no-op.

<Tabs>

## Observing caption state

Three publishers let you react to caption changes in real time.

| Publisher | Type | Description |
|---|---|---|
| `captionsEnabled` | `Bool` | Whether captions are currently turned on |
| `activeCaptionTrack` | `CaptionTrack?` | The selected track, or `nil` if none is active |
| `activeCue` | `VTTCue?` | The caption text visible right now, updated at ~10 Hz |

<Tabs>
  <Tab title="Swift">
    ```swift
    // React to enable/disable
    shortKit.player.captionsEnabled.sink { enabled in
        captionButton.isSelected = enabled
    }

    // Show which track is active
    shortKit.player.activeCaptionTrack.sink { track in
        trackLabel.text = track?.label ?? "Off"
    }

    // Display the current cue text
    shortKit.player.activeCue.sink { cue in
        captionLabel.text = cue?.text
        captionLabel.isHidden = (cue == nil)
    }
    ```
  </Tab>
  <Tab title="Kotlin">
    ```kotlin
    shortKit.player.captionsEnabled.collect { enabled ->
        captionButton.isSelected = enabled
    }

    shortKit.player.activeCaptionTrack.collect { track ->
        trackLabel.text = track?.label ?: "Off"
    }

    shortKit.player.activeCue.collect { cue ->
        captionLabel.text = cue?.text
        captionLabel.isVisible = cue != null
    }
    ```
  </Tab>
  <Tab title="React Native">
    ```tsx
    const { captionsEnabled, activeCaptionTrack, activeCue } = useShortKitPlayer();

    return (
      <>
        <Button
          title={captionsEnabled ? "CC On" : "CC Off"}
          onPress={() => setCaptionsEnabled(!captionsEnabled)}
        />
        <Text>{activeCaptionTrack?.label ?? "Off"}</Text>
        {activeCue && <Text style={styles.caption}>{activeCue.text}</Text>}
      </>
    );
    ```
  </Tab>
  <Tab title="Web">
    ```js
    sk.player.on('captionsChange', ({ enabled, track }) => {
      console.log('Captions:', enabled ? track?.label : 'Off');
    });

    sk.player.on('cueChange', (cue) => {
      const captionEl = document.getElementById('caption');
      captionEl.textContent = cue?.text ?? '';
      captionEl.hidden = !cue;
    });
    ```
  </Tab>
</Tabs>

### VTTCue structure

Each cue carries a time range and the caption text:

```swift
public struct VTTCue: Equatable, Sendable {
    public let startTime: Double   // Seconds
    public let endTime: Double     // Seconds
    public let text: String        // Caption text (may contain line breaks)
}
```

## Caption rendering

Subscribe to the `activeCue` publisher and render the text however you want.

<Tabs>
  <Tab title="Swift">
    ```swift
    import Combine

    class CustomCaptionView: UIView {
        private let label = UILabel()
        private var cancellable: AnyCancellable?

        func bind(to player: ShortKitPlayer) {
            cancellable = player.activeCue
                .receive(on: DispatchQueue.main)
                .sink { [weak self] cue in
                    self?.label.text = cue?.text
                    self?.isHidden = (cue == nil)
                }
        }
    }
    ```
  </Tab>
  <Tab title="Kotlin">
    ```kotlin
    class CustomCaptionView(context: Context) : TextView(context) {
        fun bind(player: ShortKitPlayer) {
            lifecycleScope.launch {
                player.activeCue.collect { cue ->
                    text = cue?.text
                    isVisible = cue != null
                }
            }
        }
    }
    ```
  </Tab>
  <Tab title="React Native">
    ```tsx
    import { useShortKitPlayer } from "@shortkitsdk/react-native";
    import { Text, StyleSheet } from "react-native";

    function CustomCaption() {
      const { activeCue } = useShortKitPlayer();

      if (!activeCue) return null;

      return <Text style={styles.caption}>{activeCue.text}</Text>;
    }

    const styles = StyleSheet.create({
      caption: {
        position: "absolute",
        bottom: 80,
        alignSelf: "center",
        color: "#fff",
        backgroundColor: "rgba(0,0,0,0.7)",
        paddingHorizontal: 12,
        paddingVertical: 6,
        borderRadius: 4,
        fontSize: 16,
      },
    });
    ```
  </Tab>
  <Tab title="Web">
    ```js app.js
    const feed = sk.createFeed(document.getElementById('feed'), {
      config: {
        overlay: (container, { item, player, feed }) => {
          const caption = document.createElement('div');
          caption.style.cssText =
            'position:absolute;bottom:80px;left:50%;transform:translateX(-50%);' +
            'color:#fff;background:rgba(0,0,0,0.7);padding:6px 12px;border-radius:4px;';
          caption.hidden = true;
          container.appendChild(caption);

          player.on('cueChange', (cue) => {
            caption.textContent = cue?.text ?? '';
            caption.hidden = !cue;
          });
        },
      },
    });
    ```
  </Tab>

## Caption sources

| Source | How it works | When to use |
|---|---|---|
| `generated` | Captions are auto-generated during transcoding. Tracks appear on `ContentItem.captionTracks` automatically. | Default for all uploaded videos. No extra work needed. |
| `external` | You upload a `.vtt` file and attach it to a content item via the API or portal. The `url` field points to the hosted file. | When you have professional or corrected captions. |
| `embedded` | Captions are muxed into the HLS manifest. The player reads them directly from the stream. `url` is `nil`. | When your encoding pipeline bakes captions into HLS. |
