ShortKit drops a short-form video feed into your app: you add a feed component, and ShortKit handles vertical paging, preloading, playback, adaptive streaming, and delivery. Your team manages content through the dashboard or the [REST API](/docs/api).

Get your API keys, install the SDK, and display a feed. Your account comes with demo content, so you see a working feed before uploading anything.

## 1. Get your API keys

Sign up at [portal.shortkit.dev](https://portal.shortkit.dev), then go to **Settings** then **API Keys**. You need two keys:

| Key | Prefix | Where it's used |
|---|---|---|
| Publishable key | `pk_live_...` | Client apps — initializes the SDK |
| Secret key | `sk_live_...` | Your server — uploads, content management, the [REST API](/docs/api) |

<Callout type="warn" title="Keep the secret key server-side">
Never embed `sk_live_...` in a client app. The SDK only needs the publishable key.
</Callout>

## 3. Render a feed

Create a screen and pass your publishable key. Every new account is seeded with demo videos, so the feed has content immediately.

<Tabs groupId="platform" persist items={['iOS','Android','Flutter','React Native','Web']}>
<Tab value="iOS">

```swift title="FeedScreen.swift"
import SwiftUI
import ShortKitSDK

struct FeedScreen: View {
    let shortKit = ShortKit(apiKey: "pk_live_your_publishable_key")

    var body: some View {
        ShortKitFeedView(shortKit: shortKit)
            .ignoresSafeArea()
    }
}
```

Using UIKit instead? Embed `ShortKitFeedViewController` — see the [iOS setup guide](/docs/guides/setup/ios).

</Tab>
<Tab value="Android">

```kotlin title="FeedActivity.kt"
import android.os.Bundle
import androidx.appcompat.app.AppCompatActivity
import com.shortkit.sdk.ShortKit
import com.shortkit.sdk.config.FeedConfig
import com.shortkit.sdk.feed.ShortKitFeedFragment

class FeedActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_feed)

        val shortKit = ShortKit(
            context = this,
            apiKey = "pk_live_your_publishable_key",
        )

        if (savedInstanceState == null) {
            supportFragmentManager.beginTransaction()
                .replace(R.id.feed_container, ShortKitFeedFragment.newInstance(shortKit, FeedConfig()))
                .commit()
        }
    }
}
```

The layout needs a full-screen `FrameLayout` with id `feed_container` — see the [Android setup guide](/docs/guides/setup/android).

</Tab>
<Tab value="Flutter">

```dart title="feed_screen.dart"
import 'package:flutter/material.dart';
import 'package:shortkit_flutter/shortkit_flutter.dart';

class FeedScreen extends StatelessWidget {
  const FeedScreen({super.key});

  @override
  Widget build(BuildContext context) {
    return ShortKitProvider.apiKey(
      apiKey: 'pk_live_your_publishable_key',
      child: const Scaffold(
        body: ShortKitFeed(),
      ),
    );
  }
}
```

</Tab>
<Tab value="React Native">

```tsx title="FeedScreen.tsx"
import React from 'react';
import { ShortKitProvider, ShortKitFeed } from '@shortkitsdk/react-native';

export default function FeedScreen() {
  return (
    <ShortKitProvider apiKey="pk_live_your_publishable_key">
      <ShortKitFeed style={{ flex: 1 }} />
    </ShortKitProvider>
  );
}
```

Feed configuration goes on `<ShortKitFeed>`, not the provider — see [feed configuration](/docs/guides/reference/feed-config).

</Tab>
<Tab value="Web">

```html title="index.html"
<!DOCTYPE html>
<html>

```

</Tab>
</Tabs>

Run your app. You should see a full-screen, vertically scrolling video feed playing the demo content.

## 4. Add your own content

Replace the demo videos with your own:

- **Portal upload** — go to **Content** in the [Admin Portal](https://portal.shortkit.dev) and upload video files directly.
- **Import from URLs** — bulk import from object storage, your CDN, or social links via the API.
- **Direct upload** — request signed upload URLs from your server with the [upload content endpoint](/docs/api/content/upload-content).

The [upload videos guide](/docs/api/guides/upload-videos) walks through all three.
