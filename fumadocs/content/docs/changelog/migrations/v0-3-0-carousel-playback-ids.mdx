---
title: "Carousel playback IDs"
description: "Migrate videoCarousel feed inputs from host-built URLs to playback IDs."
---

<Callout type="info" title="In effect since 0.2.x">
This change is already live in all current SDK releases: iOS 0.2.58, Android 0.2.57, React Native 0.2.57, and Flutter 0.2.59. If you're upgrading from an older SDK, follow the before/after steps below to update your carousel feed inputs.
</Callout>

The `videoCarousel` feed input no longer accepts fully-built URLs and takes only playback IDs — matching how the single-video feed input already works. This is a breaking change for custom feeds that include video carousels.

Video carousels render on iOS, Android, Flutter, and React Native. The Web SDK does not render carousels yet.

## Why this changed

Previously, your app constructed `streamingUrl` and `thumbnailUrl` strings for each carousel slide. This required you to reimplement URL construction, and host-built URLs could diverge from the SDK's streaming domains. The `video` feed input already built these URLs for you; carousels did not.

Now you pass a playback ID per slide and the SDK constructs the streaming and thumbnail URLs, exactly as it does for `video` feed inputs.

## What changed

### Before (older SDKs)

```ts title="feedInput.ts"
const feedInput: FeedInput = {
  type: 'videoCarousel',
  item: {
    id: 'thread-42',
    videos: [
      {
        id: 'thread-42-v1',
        playbackId: 'abc123',
        title: 'Question',
        streamingUrl: 'https://stream.example-cdn.com/abc123.m3u8',
        thumbnailUrl: 'https://images.example-cdn.com/abc123/thumbnail.jpg?time=0',
        captionTracks: [],
      },
      // ... one full object per slide ...
    ],
    title: 'Community AMA',
  },
};
```

### After (current SDKs)

```ts title="feedInput.ts"
const feedInput: FeedInput = {
  type: 'videoCarousel',
  item: {
    id: 'thread-42',
    videos: [
      { playbackId: 'abc123' },
      { playbackId: 'def456' },
      { playbackId: 'ghi789', fallbackUrl: 'https://cdn.example.com/ghi789.mp4' },
    ],
    title: 'Community AMA',
  },
};
```

Each slide carries only `playbackId` and an optional `fallbackUrl`. Carousel-level fields (`id`, `title`, `description`, `author`, `section`, `articleUrl`) are unchanged.

## Per-slide metadata

The new slide input does not carry titles, durations, or other per-slide fields. If your overlay needs per-slide metadata (question text, author names) that ShortKit does not store, maintain your own `playbackId` to context map. Look it up in your overlay component using `activeVideo.playbackId`. The SDK still provides the full `activeVideo` on every page change via `VideoCarouselOverlayProps.activeVideo`. See [carousel control](/docs/guides/customize/carousel-control) for the overlay surface.

## Upgrade steps

1. Update your package to the current SDK version.
2. In your feed mapper, delete URL construction for carousel slides.
3. Return only `{ playbackId, fallbackUrl? }` objects for each video.
4. If your custom carousel overlay iterated `carouselItem.videos[].streamingUrl`, refactor to use `activeVideo` for the current slide and maintain a host-side map for the rest.

## Update each platform

<Tabs groupId="platform" persist items={['iOS', 'Android', 'Flutter', 'React Native']}>
<Tab value="iOS">

Replace `VideoCarouselItem` with `VideoCarouselInput` at the `FeedInput.videoCarousel` construction site:

```swift title="FeedMapper.swift"
let input = FeedInput.videoCarousel(
    VideoCarouselInput(
        id: "thread-42",
        videos: [
            VideoCarouselVideoInput(playbackId: "abc123"),
            VideoCarouselVideoInput(playbackId: "def456"),
        ],
        title: "Community AMA"
    )
)
```

</Tab>
<Tab value="Android">

Replace `VideoCarouselItem` with `VideoCarouselInput` at the `FeedInput.VideoCarousel` construction site:

```kotlin title="FeedMapper.kt"
val input = FeedInput.VideoCarousel(
    VideoCarouselInput(
        id = "thread-42",
        videos = listOf(
            VideoCarouselVideoInput(playbackId = "abc123"),
            VideoCarouselVideoInput(playbackId = "def456"),
        ),
        title = "Community AMA",
    )
)
```

</Tab>
<Tab value="Flutter">

Replace `VideoCarouselItem` with `VideoCarouselInput` at the `FeedInput.videoCarousel` construction site:

```dart title="feed_mapper.dart"
final input = FeedInput.videoCarousel(
  item: VideoCarouselInput(
    id: 'thread-42',
    videos: [
      VideoCarouselVideoInput(playbackId: 'abc123'),
      VideoCarouselVideoInput(playbackId: 'def456'),
    ],
    title: 'Community AMA',
  ),
);
```

</Tab>
<Tab value="React Native">

```bash
npm install @shortkitsdk/react-native@^0.2.57
```

Return the compact shape from your feed mapper, as shown in the before/after above. Your overlay still receives the fully-hydrated `carouselItem` via `VideoCarouselOverlayProps` — only the input shape changes.

</Tab>
