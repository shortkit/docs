## June 25, 2026 â€” Web SDK 0.6.0, mobile SDKs 0.2.60

**API**

- Removed the dedicated publication-URL endpoints. To attach an article link to a content item, set a `publicationUrl` key in `customMetadata` via `PATCH /v1/content/{id}` â€” see [Attaching an article link](/docs/api/guides/captions-and-metadata#attaching-an-article-link).
- Feed items no longer emit a top-level `publisherUrl` field. The link now travels on `customMetadata.publicationUrl`, which the feed already emits.

**SDK**

- The `articleUrl` property is removed from feed items, carousel items, and custom-feed inputs across the Web (`0.6.0`), iOS, Android, React Native, and Flutter (`0.2.60`) SDKs. Read and write the article link via `customMetadata.publicationUrl` instead â€” for example `item.customMetadata?.publicationUrl`.

## June 10, 2026 â€” Flutter SDK

**SDK**

- Added `scrollToItem` so you can programmatically scroll the feed to a specific item.
- Added an `active` prop to pause and resume feed playback when the widget is hidden or shown (for example, behind a tab switch).

## June 8, 2026 â€” iOS, Android, React Native 0.2.55

**SDK**

- Added `switchFeedSource(to:)` on iOS for switching the running feed between feed modes at runtime without recreating the feed view.
- Caption tracks are now exposed on `ContentItem` across iOS and Android, with captions resolved lazily on Android so they don't block feed loading.
- Carousel long-press events now include the tap's `x`/`y` coordinates on Android and React Native, so you can anchor context menus to the press location.
- iOS adds an `onModalFeedPresented` hook and an interactive swipe-to-dismiss transition for modally presented feed screens.

## June 4, 2026 â€” API and SDK 0.2.52

**API**

- `createLiveStream` now accepts `customMetadata`, so live streams can carry the same metadata key-value pairs as uploaded content.
- Metadata feed filters now support OR semantics: pass repeated values for the same key and the feed returns content matching any of them.

**SDK**

- `FeedFilter.metadata` accepts arrays of values per key on all platforms â€” `[String: [String]]` on iOS, `Map<String, List<String>>` on Android and Flutter, and `string | string[]` in React Native.

## June 1, 2026 â€” API and SDK 0.2.50

**API**

- Captions are now generated automatically for all new uploads, with automatic language detection â€” no configuration required.
- Added a `content.playback_id_updated` webhook that fires when an ended live stream's recording becomes available, so your backend can react to the playback ID changing.
- Ended live streams with recordings now report `LiveStreamStatus.dvr` and play back as on-demand video instead of showing a "stream ended" screen.

**SDK**

- Added `ShortKitPlayer.downloadCurrentItem()` to let viewers save the current video as an MP4, with optional per-organization watermarking.
