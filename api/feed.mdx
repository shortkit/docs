The Feed API is the primary endpoint consumed by the ShortKit SDK. It returns a polymorphic list of items -- videos, image carousels, surveys, and ad slots -- ordered by editorial controls and recency.

## Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/v1/feed` | Full feed (videos, carousels, surveys, ad slots) |
| `GET` | `/v1/feed/videos` | Video-only feed (no carousels) |

Both endpoints require a **publishable key** via the `X-API-Key` header.

---

## GET /v1/feed

Returns a paginated, polymorphic feed. Each item in the `data` array has a `type` discriminator: `content`, `image_carousel`, `survey`, or `ad_slot`.

### Authentication

```bash
X-API-Key: pk_live_your_publishable_key
```

### Query parameters

<ParamField query="limit" type="integer" default="10">
  Items per page. Range: 1--50.
</ParamField>

<ParamField query="cursor" type="string">
  Opaque pagination cursor from a previous response's `meta.next_cursor`.
</ParamField>

### Headers

| Header | Required | Description |
|--------|----------|-------------|
| `X-API-Key` | Yes | Publishable key (`pk_live_...` or `pk_test_...`) |
| `X-User-Id` | No | Current user ID. Used for survey deduplication (`once_per_user` rule). |
| `X-Session-Id` | No | Current session ID. Used for survey deduplication (`once_per_session`, `max_per_session` rules). |

### Example request

```bash
curl "https://api.shortkit.dev/v1/feed?limit=10" \
  -H "X-API-Key: pk_live_your_publishable_key" \
  -H "X-User-Id: user_456" \
  -H "X-Session-Id: sess_abc"
```

## Feed item types

Every item in the `data` array carries a `type` field. Parse this discriminator to determine the schema.

### `content` (video)

A playable video with HLS streaming URL, thumbnail, and optional captions.

<ResponseField name="type" type="string" required>
  Always `"content"`.
</ResponseField>

<ResponseField name="id" type="string" required>
  Content UUID.
</ResponseField>

<ResponseField name="title" type="string" required>
  Display title.
</ResponseField>

<ResponseField name="description" type="string | null">
  Optional description text.
</ResponseField>

<ResponseField name="duration" type="number" required>
  Video duration in seconds.
</ResponseField>

<ResponseField name="streamingUrl" type="string" required>
  HLS streaming URL (`https://stream.shortkit.dev/{playback_id}.m3u8`). Pass directly to your video player.
</ResponseField>

<ResponseField name="thumbnailUrl" type="string" required>
  Thumbnail image URL for the poster frame.
</ResponseField>

<ResponseField name="captionTracks" type="array" required>
  Available caption tracks. Each track has `language` (ISO 639-1), `label` (human-readable name), `source` (`"generated"` or `"uploaded"`), and `url` (VTT file URL).
</ResponseField>

<ResponseField name="customMetadata" type="object" required>
  Arbitrary key-value pairs set when the content was created or updated. Use this to pass application-specific data (article URLs, category tags, deep links) through to your UI layer.
</ResponseField>

<ResponseField name="author" type="string | null">
  Content author name.
</ResponseField>

<ResponseField name="section" type="string | null">
  Editorial section (e.g., `"Sports"`, `"Weather"`).
</ResponseField>

<ResponseField name="publisherUrl" type="string | null">
  Primary publication URL associated with this content.
</ResponseField>

### `image_carousel`

A swipeable set of images with optional auto-scroll.

<ResponseField name="type" type="string" required>
  Always `"image_carousel"`.
</ResponseField>

<ResponseField name="id" type="string" required>
  Content UUID.
</ResponseField>

<ResponseField name="images" type="array" required>
  Ordered array of image objects. Each has `url` (image source) and `alt` (alt text, may be `null`).
</ResponseField>

<ResponseField name="autoScrollInterval" type="number | null">
  Seconds between automatic slide transitions. `null` means auto-scroll is disabled.
</ResponseField>

<ResponseField name="caption" type="string | null">
  Caption text displayed below the carousel.
</ResponseField>

<ResponseField name="title" type="string | null">
  Display title.
</ResponseField>

<ResponseField name="description" type="string | null">
  Optional description text.
</ResponseField>

<ResponseField name="author" type="string | null">
  Content author name.
</ResponseField>

<ResponseField name="section" type="string | null">
  Editorial section.
</ResponseField>

<ResponseField name="publisherUrl" type="string | null">
  Primary publication URL.
</ResponseField>

### `survey`

An in-feed poll injected between content items based on placement rules.

<ResponseField name="type" type="string" required>
  Always `"survey"`.
</ResponseField>

<ResponseField name="id" type="string" required>
  Survey UUID.
</ResponseField>

<ResponseField name="question" type="string" required>
  The question text displayed to the user.
</ResponseField>

<ResponseField name="options" type="array" required>
  Answer options. Each has `id` (option UUID) and `text` (display label). Submit the selected `id` to `POST /v1/surveys/{survey_id}/responses`.
</ResponseField>

<ResponseField name="autoAdvanceDelay" type="number | null">
  Seconds to wait after the user responds before auto-advancing to the next feed item.
</ResponseField>

### `ad_slot`

A server-inserted ad placement marker. The SDK loads and renders the ad creative using the provided configuration.

<ResponseField name="type" type="string" required>
  Always `"ad_slot"`.
</ResponseField>

<ResponseField name="id" type="string" required>
  Server-assigned ad slot identifier (e.g., `ad_slot_p0_3_0`).
</ResponseField>

<ResponseField name="position" type="integer" required>
  Zero-based position of this ad slot in the assembled feed.
</ResponseField>

<ResponseField name="adTagUrl" type="string | null">
  VAST tag URL for the ad request. Used by ad providers such as Google IMA.
</ResponseField>

<ResponseField name="adUnitId" type="string | null">
  Ad unit identifier for providers that use unit-based lookups.
</ResponseField>

<ResponseField name="targeting" type="object" required>
  Key-value pairs passed to the ad provider for targeting (e.g., `{"content_category": "sports"}`).
</ResponseField>

<ResponseField name="skipPolicy" type="string" required>
  Controls skip behavior. One of: `"non_skippable"`, `"skippable_after"`, `"always_skippable"`.
</ResponseField>

<ResponseField name="skipDelaySeconds" type="integer" required>
  Seconds the user must wait before skipping. Only meaningful when `skipPolicy` is `"skippable_after"`.
</ResponseField>

<ResponseField name="adFormat" type="string | null">
  `"native"` when native ads are enabled, otherwise `null`.
</ResponseField>

<ResponseField name="nativeFormatId" type="string | null">
  Identifier for the native ad format template. Present only when `adFormat` is `"native"`.
</ResponseField>

<ResponseField name="nativeAssetKeys" type="object | null">
  Mapping of asset keys used to extract creative fields from the native ad response. Present only when `adFormat` is `"native"`. Contains: `videoUrl`, `headline`, `ctaText`, `sponsorName`, `sponsorLogo`.
</ResponseField>

## GET /v1/feed/videos

Same as `GET /v1/feed` but returns only video content items. Image carousels are excluded. Surveys and ad slots are still injected.

### Example request

```bash
curl "https://api.shortkit.dev/v1/feed/videos?limit=10" \
  -H "X-API-Key: pk_live_your_publishable_key"
```

The response uses the same envelope and pagination as `/v1/feed`. The `data` array will contain `content` (video), `survey`, and `ad_slot` items -- never `image_carousel`.

---

## Pagination

Both feed endpoints use cursor-based pagination. The cursor is an opaque, base64-encoded token derived from `(created_at, id)` of the last content item.

### How it works

1. Make an initial request without a `cursor` parameter.
2. If more items are available, the response includes `meta.next_cursor`.
3. Pass that value as the `cursor` query parameter in your next request.
4. When `meta.next_cursor` is absent, you have reached the end of the feed.

```bash
# Page 1
curl "https://api.shortkit.dev/v1/feed?limit=10" \
  -H "X-API-Key: pk_live_your_publishable_key"

# Page 2 -- use next_cursor from the previous response
curl "https://api.shortkit.dev/v1/feed?limit=10&cursor=eyJjYSI6Ij..." \
  -H "X-API-Key: pk_live_your_publishable_key"
```

<Note>
  The cursor field is `meta.next_cursor` (not `meta.cursor`). The cursor is computed from content items only -- injected surveys and ad slots do not affect cursor stability.
</Note>

## Ordering

Content is returned in a two-tier sort order:

1. **Pinned items first** -- content with a `pinPosition` value appears at the top, sorted by `pinPosition` ascending.
2. **Reverse-chronological** -- all remaining content is sorted by `created_at` descending, with `id` descending as a tiebreaker.

### Filtered out

The following content never appears in the feed:

| Condition | Behavior |
|-----------|----------|
| Status is not `ready` | Excluded (still processing, errored, or archived) |
| `isSuppressed` is `true` | Excluded by editorial suppression |
| `publishAt` is in the future | Excluded until the scheduled publish time |
| `expiresAt` is in the past | Excluded after the expiration time |

## Survey injection

Active surveys are injected into the feed between content items. The server evaluates placement rules to determine which surveys to show and where.

### Placement rules

Each survey has an ordered set of placement rules configured via the admin API:

| Rule | Effect |
|------|--------|
| `min_videos_before` | The survey is inserted after `n` content items. Required on every survey. |
| `once_per_user` | Skip this survey if `X-User-Id` has already responded. |
| `once_per_session` | Skip this survey if `X-Session-Id` has already responded. |
| `max_per_session` | Limit the total number of surveys injected in a single feed response. |

### Priority

When multiple surveys target the same position, the survey with the higher `priority` value wins. Only one survey occupies a given position.

### Deduplication

Pass `X-User-Id` and `X-Session-Id` headers to enable deduplication. Without these headers, `once_per_user` and `once_per_session` rules cannot be evaluated and those surveys will always be included.

## Ad slot interleaving

When ads are enabled for your organization (via `PUT /v1/ad-config`), ad slot items are interleaved into the feed after content ordering and survey injection.

### Configuration

Ad interleaving is controlled by the organization's `AdConfig`:

| Setting | Default | Description |
|---------|---------|-------------|
| `enabled` | `false` | Master toggle. No ad slots are inserted when `false`. |
| `firstAdOffset` | `3` | Number of content items before the first ad slot. |
| `frequency` | `5` | Number of content items between subsequent ad slots. |
| `maxPerPage` | `3` | Maximum ad slots per page of results. |

### Insertion logic

1. The server counts content items (not surveys) in the assembled feed.
2. After `firstAdOffset` content items, the first ad slot is inserted.
3. Every `frequency` content items after that, another ad slot is inserted.
4. Insertion stops when `maxPerPage` is reached.

Ad slots do not affect pagination cursors. The cursor is computed from content items before ad interleaving occurs.

### Native ads

When `nativeEnabled` is `true` in the ad config, ad slots include `adFormat: "native"` and provide `nativeFormatId` and `nativeAssetKeys` fields. The SDK uses these keys to extract creative fields (video URL, headline, CTA, sponsor info) from the native ad response.

## Error responses

| HTTP Status | Error code | Description |
|-------------|------------|-------------|
| 400 | `invalid_request` | Invalid `cursor` or `limit` value |
| 401 | `invalid_api_key` | Missing or invalid `X-API-Key` header |
| 429 | `rate_limited` | Too many requests. Retry after the `Retry-After` header value. |
| 500 | `internal_error` | Unexpected server error. Contact support with the `request_id`. |

```json
{
  "data": null,
  "meta": {
    "request_id": "req_err123"
  },
  "errors": [
    {
      "code": "invalid_api_key",
      "message": "The provided API key is not valid."
    }
  ]
}
```
