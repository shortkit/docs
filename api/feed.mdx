The Feed API is the primary endpoint consumed by the ShortKit SDK. It returns a ranked, paginated list of video content ready for playback. The SDK calls this endpoint automatically, but you can also call it directly for server-side rendering or custom integrations.

## Endpoint

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/v1/feed` | Get the video feed |

## Authentication

The Feed API uses a **publishable key** passed via the `X-API-Key` header. Publishable keys are safe to embed in client-side code and mobile apps.

```bash
X-API-Key: pk_live_your_publishable_key
```

## Get feed

Retrieves a paginated feed of video content for your organization.

```bash
GET /v1/feed
```

### Query parameters

<ParamField query="limit" type="integer" default={10}>
  Number of items to return (1--50).
</ParamField>

<ParamField query="cursor" type="string">
  Opaque pagination cursor from a previous response. Pass this to fetch the next page.
</ParamField>

### Example request

```bash
curl "https://api.shortkit.dev/v1/feed?limit=10" \
  -H "X-API-Key: pk_live_your_publishable_key"
```

### Feed item object

Each item in the `data` array contains:

<ResponseField name="id" type="string">
  Content identifier (e.g., `sk_content_abc123`).
</ResponseField>

<ResponseField name="title" type="string">
  Display title.
</ResponseField>

<ResponseField name="description" type="string | null">
  Optional description text.
</ResponseField>

<ResponseField name="duration" type="number">
  Video duration in seconds.
</ResponseField>

<ResponseField name="streamingUrl" type="string">
  HLS streaming URL. Pass this directly to your video player.
</ResponseField>

<ResponseField name="thumbnailUrl" type="string">
  Thumbnail image URL for the poster frame.
</ResponseField>

<ResponseField name="captionTracks" type="array">
  Available caption tracks. Each track has `language` (ISO 639-1 code), `label` (human-readable name),
  `source` (`"embedded"` for captions in the HLS stream, `"external"` for sidecar files),
  and `url` (`null` for embedded, a URL for external).
</ResponseField>

<ResponseField name="customMetadata" type="object">
  Arbitrary key-value pairs set when the content was created or updated.
  Use this to pass application-specific data (article URLs, category tags, deep links)
  through to your app's UI layer.
</ResponseField>

## Pagination

The Feed API uses **cursor-based pagination**. This provides consistent performance regardless of how deep you paginate.

1. Make an initial request without a `cursor`.
2. If `meta.nextCursor` is present in the response, pass it as the `cursor` query parameter on the next request.
3. When `meta.nextCursor` is `null`, you have reached the end of the feed.

```bash
# First page
curl "https://api.shortkit.dev/v1/feed?limit=10" \
  -H "X-API-Key: pk_live_your_publishable_key"

# Next page (using cursor from previous response)
curl "https://api.shortkit.dev/v1/feed?limit=10&cursor=eyJjYSI6IjIwMjYtMDItMDVUMTE6MzA6MDBaIiwiaWQiOiJza19jb250ZW50X2RlZjQ1NiJ9" \
  -H "X-API-Key: pk_live_your_publishable_key"
```

<Note>
  Cursors are opaque strings. Do not parse, modify, or construct them. They may change format
  between API versions.
</Note>

## Feed assembly

The feed is assembled using the following rules, applied in order:

| Rule | Behavior |
|------|----------|
| **Status filter** | Only content with status `ready` is included. |
| **Suppression** | Content where `isSuppressed` is `true` is excluded. |
| **Publish window** | Content with a `publishAt` timestamp in the future is excluded. |
| **Expiration window** | Content with an `expiresAt` timestamp in the past is excluded. |
| **Pinned items first** | Content with a `pinPosition` is placed at its specified position, ordered ascending. |
| **Reverse chronological** | All remaining (unpinned) content is ordered newest first. |

### Ranking signals

| Signal | Description |
|--------|-------------|
| Recency | Newer content appears before older content |
| Pin position | Pinned items override chronological ordering |
| Editorial controls | Suppression, publish windows, and expiration are respected |

<Note>
  Advanced ranking signals (engagement-weighted scoring, topic affinity, personalization) will be
  available in a future release.
</Note>

## SDK integration

The ShortKit SDK handles feed fetching, pagination, and video playback automatically. You do not need to call the Feed API directly unless you are building a custom integration.

<Tabs>
  <Tab title="iOS (Swift)">
    ```swift
    import ShortKit

    // Initialize with your publishable key
    let config = ShortKitConfig(apiKey: "pk_live_your_publishable_key")
    let shortKit = ShortKit(config: config)

    // Present the feed as a full-screen view
    let feedView = ShortKitFeedView()
    present(feedView, animated: true)

    // Or fetch feed data manually
    let feed = try await shortKit.fetchFeed(limit: 10)

    for item in feed.items {
        print(item.title)
        print(item.streamingUrl)
    }

    // Load next page
    if let cursor = feed.nextCursor {
        let nextPage = try await shortKit.fetchFeed(
            limit: 10,
            cursor: cursor
        )
    }
    ```
  </Tab>

## Error responses

| HTTP Status | Error code | Description |
|-------------|------------|-------------|
| 401 | `invalid_api_key` | The `X-API-Key` header is missing or the key is not valid. |
| 429 | `rate_limited` | Too many requests. Retry after the duration in the `Retry-After` header. |
| 500 | `internal_error` | An unexpected error occurred. Contact support if it persists. |

All error responses follow the standard envelope format:

```json
{
  "data": null,
  "meta": {
    "requestId": "req_err123"
  },
  "errors": [
    {
      "code": "invalid_api_key",
      "message": "The provided API key is not valid.",
      "status": 401
    }
  ]
}
```
