The Content API manages your entire video library. Use it to upload new videos, list content, retrieve details, update metadata and editorial controls, delete content, and perform bulk operations on multiple items at once.

## Endpoints

| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| `POST` | `/v1/uploads` | Secret key | Create an upload session |
| `GET` | `/v1/uploads/{id}` | Secret key | Check upload/processing status |
| `GET` | `/v1/content` | Secret key | List all content |
| `GET` | `/v1/content/{id}` | Secret key | Get content by ID |
| `PATCH` | `/v1/content/{id}` | Secret key | Update content |
| `DELETE` | `/v1/content/{id}` | Secret key | Delete content |
| `POST` | `/v1/content/bulk/update` | Secret key | Update multiple content items |
| `POST` | `/v1/content/bulk/archive` | Secret key | Archive multiple content items |
| `POST` | `/v1/content/bulk/tag` | Secret key | Add or remove tags from multiple items |

## Content object

Every content item has the following shape:

```json
{
  "id": "sk_content_abc123",
  "title": "Breaking: Weather Alert",
  "description": "Severe thunderstorm warning for the metro area",
  "status": "ready",
  "duration": 32.5,
  "streamingUrl": "https://video.shortkit.dev/sk_content_abc123/stream.m3u8",
  "thumbnailUrl": "https://images.shortkit.dev/sk_content_abc123/thumbnail.jpg",
  "captionTracks": [
    {
      "language": "en",
      "label": "English",
      "source": "embedded",
      "url": null
    }
  ],
  "tags": ["weather", "severe", "breaking"],
  "customMetadata": {
    "articleUrl": "https://example.com/weather-alert",
    "category": "severe-weather"
  },
  "pinPosition": null,
  "isSuppressed": false,
  "publishAt": null,
  "expiresAt": null,
  "createdAt": "2026-02-05T12:00:00Z",
  "updatedAt": "2026-02-05T12:03:45Z"
}
```

### Field reference

<ResponseField name="id" type="string">
  Unique content identifier (e.g., `sk_content_abc123`).
</ResponseField>

<ResponseField name="title" type="string">
  Display title.
</ResponseField>

<ResponseField name="description" type="string | null">
  Optional description text.
</ResponseField>

<ResponseField name="status" type="string">
  Current lifecycle status. One of: `waiting_for_upload`, `processing`, `ready`, `error`, `archived`.
</ResponseField>

<ResponseField name="duration" type="number | null">
  Video duration in seconds. Available once status is `ready`.
</ResponseField>

<ResponseField name="streamingUrl" type="string | null">
  HLS streaming URL. Available once status is `ready`.
</ResponseField>

<ResponseField name="thumbnailUrl" type="string | null">
  Thumbnail image URL. Available once status is `ready`.
</ResponseField>

<ResponseField name="captionTracks" type="array | null">
  Available caption tracks. Each track includes `language`, `label`, `source` (`"embedded"` or `"external"`), and `url` (null for embedded captions).
</ResponseField>

<ResponseField name="tags" type="string[]">
  Content tags for categorization and filtering.
</ResponseField>

<ResponseField name="customMetadata" type="object">
  Arbitrary key-value pairs set during upload or update. Passed through to the feed response.
</ResponseField>

<ResponseField name="pinPosition" type="integer | null">
  When set, pins this content to a specific position in the feed (1 = top). `null` means unpinned.
</ResponseField>

<ResponseField name="isSuppressed" type="boolean">
  When `true`, this content is excluded from the feed regardless of status or scheduling.
</ResponseField>

<ResponseField name="publishAt" type="string | null">
  ISO 8601 timestamp. Content will not appear in the feed before this time. `null` means immediately available.
</ResponseField>

<ResponseField name="expiresAt" type="string | null">
  ISO 8601 timestamp. Content is automatically excluded from the feed after this time. `null` means no expiration.
</ResponseField>

<ResponseField name="createdAt" type="string">
  Creation timestamp (ISO 8601).
</ResponseField>

<ResponseField name="updatedAt" type="string">
  Last modification timestamp (ISO 8601).
</ResponseField>

## Status lifecycle

Content moves through these statuses:

```
POST /v1/uploads
      |
      v
waiting_for_upload  -->  processing  -->  ready  -->  archived
                              |
                              v
                            error
```

| Status | Description | Visible in feed? |
|--------|-------------|------------------|
| `waiting_for_upload` | Upload session created; file not yet received | No |
| `processing` | Video is being transcoded; thumbnails and captions are being generated | No |
| `ready` | Processing complete; content is available for playback | Yes (subject to editorial controls) |
| `error` | Processing failed | No |
| `archived` | Manually archived or expired | No |

Content with status `ready` appears in the feed only when all of these conditions are met:
- `isSuppressed` is `false`
- `publishAt` is `null` or in the past
- `expiresAt` is `null` or in the future

## Uploads

The Uploads API lets you ingest video content into ShortKit. You create an upload session, receive a signed upload URL, and PUT your file directly to it. ShortKit handles transcoding, thumbnail generation, and subtitle extraction automatically.

### Upload flow

```
1. POST /v1/uploads          -- Create upload session (returns signed URL + content ID)
2. PUT  {uploadUrl}           -- Upload file directly to the signed URL
3. Processing                 -- ShortKit transcodes, generates thumbnails and captions
4. Ready                      -- Content appears in your feed (or poll status / receive webhook)
```

### Upload methods

| Method | Best for | Max size |
|--------|----------|----------|
| [Direct upload](#create-upload) | Most use cases | 5 GB |
| [Server-side SDK](#upload-sdk-helpers) | Automated pipelines with progress tracking | 5 GB |

## Create upload

Creates a new upload session and returns a signed upload URL. The upload URL is valid for 60 minutes.

```bash
POST /v1/uploads
```

### Authentication

Requires a **secret key** via the `Authorization` header.

### Request body

<ParamField body="title" type="string" required>
  Display title for the content (max 200 characters).
</ParamField>

<ParamField body="description" type="string">
  Optional description (max 2000 characters).
</ParamField>

<ParamField body="tags" type="string[]">
  Tags for categorization and filtering.
</ParamField>

<ParamField body="customMetadata" type="object">
  Arbitrary key-value pairs attached to the content. Use this for your own application data
  such as article URLs, campaign IDs, or internal references.
</ParamField>

<ParamField body="generateSubtitles" type="boolean" default={true}>
  When `true`, ShortKit automatically generates English captions from the audio track
  using speech-to-text. Captions are embedded in the HLS stream once processing completes.
</ParamField>

### Example request

```bash
curl -X POST https://api.shortkit.dev/v1/uploads \
  -H "Authorization: Bearer sk_live_your_secret_key" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Breaking: Weather Alert",
    "description": "Severe thunderstorm warning for the metro area",
    "tags": ["weather", "severe", "breaking"],
    "customMetadata": {
      "articleUrl": "https://example.com/weather-alert",
      "category": "severe-weather"
    },
    "generateSubtitles": true
  }'
```

### Response

```json
{
  "data": {
    "id": "sk_content_abc123",
    "uploadUrl": "https://uploads.shortkit.dev/direct/sk_content_abc123?token=eyJ...",
    "status": "waiting_for_upload",
    "title": "Breaking: Weather Alert",
    "description": "Severe thunderstorm warning for the metro area",
    "tags": ["weather", "severe", "breaking"],
    "customMetadata": {
      "articleUrl": "https://example.com/weather-alert",
      "category": "severe-weather"
    },
    "createdAt": "2026-02-05T12:00:00Z"
  },
  "meta": {
    "request_id": "req_xyz789"
  }
}
```

The `uploadUrl` is a signed URL for direct file upload. It expires after 60 minutes.

## Upload the file

Once you have the signed `uploadUrl`, upload your video file with an HTTP `PUT` request:

```bash
curl -X PUT "${uploadUrl}" \
  -H "Content-Type: video/mp4" \
  --data-binary @video.mp4
```

<Note>
  The `PUT` request goes directly to the signed upload URL, not to `api.shortkit.dev`.
  No `Authorization` header is needed -- the URL itself contains the upload credentials.
</Note>

After the PUT completes successfully, ShortKit begins processing the video automatically.
The content status transitions from `waiting_for_upload` to `processing`.

### Expected processing times

| Step | Duration |
|------|----------|
| Upload (30s video, ~50 MB) | 5--30 seconds depending on network speed |
| Transcoding and thumbnail generation | 1--5 minutes for a 1-minute video |
| Subtitle generation (if enabled) | Additional 1--3 minutes |
| **Total: upload to ready** | **~2--8 minutes** |

### Supported formats

#### Video formats

| Format | Extensions | MIME types |
|--------|------------|------------|
| MP4 | `.mp4`, `.m4v` | `video/mp4` |
| QuickTime | `.mov` | `video/quicktime` |
| WebM | `.webm` | `video/webm` |
| MKV | `.mkv` | `video/x-matroska` |

#### Video requirements

| Property | Requirement |
|----------|-------------|
| Duration | 1--180 seconds |
| Resolution | 480p -- 4K |
| Frame rate | 24--60 fps |
| Max bitrate | 50 Mbps |
| Max file size | 5 GB |
| Codec | H.264, H.265, VP9, AV1 |

#### Encoding recommendations

For the fastest processing and best playback quality:

- **Container:** MP4 (`.mp4`)
- **Video codec:** H.264 High profile
- **Audio codec:** AAC-LC, stereo, 128 kbps or higher
- **Orientation:** Vertical (9:16) for short-form feeds
- **Resolution:** 1080 x 1920 (1080p vertical)
- **Frame rate:** 30 fps
- **Bitrate:** 6--10 Mbps for 1080p

<Warning>
  Files that do not meet the requirements above may fail during processing.
  Check content status after upload to detect failures early.
</Warning>

### Upload SDK helpers

Server-side SDKs provide upload helpers with progress tracking and automatic retries:

<Tabs>
  <Tab title="JavaScript">
    ```javascript
    import { ShortkitAdmin } from '@shortkit/node';

    const shortkit = new ShortkitAdmin({ apiKey: 'sk_live_...' });

    // Create upload and send file in one call
    const content = await shortkit.uploads.create({
      file: fs.createReadStream('video.mp4'),
      title: 'Weather Alert',
      tags: ['weather', 'breaking'],
      customMetadata: { articleUrl: 'https://example.com/alert' },
      onProgress: (progress) => {
        console.log(`${progress.percent}% uploaded`);
      }
    });

    console.log(content.id);     // sk_content_abc123
    console.log(content.status); // waiting_for_upload | processing

    // Poll until ready
    const ready = await shortkit.uploads.waitForReady(content.id, {
      pollInterval: 5000, // ms
      timeout: 600000     // 10 minutes
    });

    console.log(ready.status);       // ready
    console.log(ready.streamingUrl); // https://video.shortkit.dev/...
    ```
  </Tab>
  <Tab title="Python">
    ```python
    from shortkit import ShortkitAdmin

    shortkit = ShortkitAdmin(api_key="sk_live_...")

    # Create upload and send file in one call
    with open("video.mp4", "rb") as f:
        content = shortkit.uploads.create(
            file=f,
            title="Weather Alert",
            tags=["weather", "breaking"],
            custom_metadata={"article_url": "https://example.com/alert"},
            on_progress=lambda p: print(f"{p.percent}% uploaded"),
        )

    print(content.id)      # sk_content_abc123
    print(content.status)  # waiting_for_upload | processing

    # Poll until ready
    ready = shortkit.uploads.wait_for_ready(
        content.id,
        poll_interval=5,  # seconds
        timeout=600,      # 10 minutes
    )

    print(ready.status)         # ready
    print(ready.streaming_url)  # https://video.shortkit.dev/...
    ```
  </Tab>
</Tabs>

### Upload error handling

| Error code | Description | Solution |
|------------|-------------|----------|
| `invalid_request` | Missing required fields (e.g., `title`) | Include all required fields in the request body |
| `file_too_large` | File exceeds 5 GB limit | Compress or trim the video |
| `invalid_media_type` | Unsupported video format | Use a supported format (MP4, MOV, WebM, MKV) |
| `duration_exceeded` | Video longer than 180 seconds | Trim to 180 seconds or less |
| `upload_expired` | Signed upload URL has expired | Create a new upload session |

```javascript
try {
  const content = await shortkit.uploads.create({ file, title: 'My Video' });
} catch (error) {
  switch (error.code) {
    case 'file_too_large':
      // Compress or trim the video before re-uploading
      break;
    case 'invalid_media_type':
      // Convert to a supported format
      break;
    case 'upload_expired':
      // Create a new upload session
      break;
  }
}
```

### Pagination

This endpoint uses **cursor-based pagination**. When there are more results, `meta.nextCursor` contains an opaque cursor token. Pass it as the `cursor` query parameter on the next request to fetch the next page. When `meta.nextCursor` is `null`, you have reached the end of the result set.

```bash
# Next page -- use the cursor from the previous response
curl "https://api.shortkit.dev/v1/content?limit=20&cursor=eyJjYSI6IjIwMjYtMDItMDVUMTI6MDA6MDBaIiwiaWQiOiJza19jb250ZW50X2FiYzEyMyJ9" \
  -H "Authorization: Bearer sk_live_your_secret_key"
```

<Warning>
  Cursor tokens are opaque and may change format between API versions. Do not parse, decode, or construct them manually. Always use the exact value returned by the API.
</Warning>

## Update content

Updates content metadata and editorial controls. You cannot replace the source video file after upload -- create a new upload instead.

```bash
PATCH /v1/content/{id}
```

### Path parameters

<ParamField path="id" type="string" required>
  Content ID to update.
</ParamField>

### Request body

All fields are optional. Only include the fields you want to change.

<ParamField body="title" type="string">
  Updated display title (max 200 characters).
</ParamField>

<ParamField body="description" type="string">
  Updated description (max 2000 characters). Pass `null` to clear.
</ParamField>

<ParamField body="tags" type="string[]">
  Replaces all existing tags.
</ParamField>

<ParamField body="customMetadata" type="object">
  Merged with existing metadata. Set a key to `null` to remove it.
</ParamField>

<ParamField body="pinPosition" type="integer | null">
  Pin this content to a specific feed position (1 = top of feed). Set to `null` to unpin.
  Multiple items can be pinned; they are ordered by `pinPosition` ascending before unpinned content.
</ParamField>

<ParamField body="isSuppressed" type="boolean">
  Set to `true` to immediately hide this content from the feed. Set to `false` to restore it.
</ParamField>

<ParamField body="publishAt" type="string | null">
  ISO 8601 timestamp. Content will not appear in the feed before this time. Set to `null` for immediate availability.
</ParamField>

<ParamField body="expiresAt" type="string | null">
  ISO 8601 timestamp. Content is excluded from the feed after this time. Set to `null` to remove expiration.
</ParamField>

### Example request

```bash
curl -X PATCH https://api.shortkit.dev/v1/content/sk_content_abc123 \
  -H "Authorization: Bearer sk_live_your_secret_key" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Updated: Severe Weather Warning",
    "pinPosition": 1,
    "expiresAt": "2026-02-06T00:00:00Z",
    "tags": ["weather", "severe", "featured"],
    "customMetadata": {
      "featured": true,
      "category": null
    }
```

### Editorial controls

Use these fields to control how and when content appears in the feed:

| Control | Field | Effect |
|---------|-------|--------|
| **Pin** | `pinPosition` | Locks content to a specific position in the feed. Pinned items appear before unpinned items, ordered by `pinPosition` ascending. |
| **Suppress** | `isSuppressed` | Immediately removes content from the feed without deleting it. Useful for temporary takedowns. |
| **Schedule** | `publishAt` | Content is invisible in the feed until this timestamp passes. |
| **Expire** | `expiresAt` | Content is automatically removed from the feed after this timestamp. The content is not deleted and can be made visible again by updating or clearing the `expiresAt` value. |

## Delete content

Archives a content item. Archived content is immediately removed from the feed but its metadata is retained. The content status changes to `archived`.

```bash
DELETE /v1/content/{id}
```

### Path parameters

<ParamField path="id" type="string" required>
  Content ID to archive.
</ParamField>

### Example request

```bash
curl -X DELETE https://api.shortkit.dev/v1/content/sk_content_abc123 \
  -H "Authorization: Bearer sk_live_your_secret_key"
```

### Response

```json
{
  "data": {
    "id": "sk_content_abc123",
    "deleted": true
  },
  "meta": {
    "request_id": "req_del001"
  }
}
```

<Note>
  Deleting content sets its status to `archived`. The content no longer appears in the feed
  but can still be retrieved via `GET /v1/content/{id}`.
</Note>

Bulk operations allow you to update or archive multiple content items in a single request, reducing API calls and improving efficiency for managing large content libraries.

</ParamField>

<ParamField body="continueOnError" type="boolean" default={false}>

## Bulk best practices

<AccordionGroup>
  <Accordion title="Use continueOnError for resilience">
    When processing large batches, enable `continueOnError` to ensure one failure does not stop the entire operation.
  </Accordion>
  <Accordion title="Handle partial failures">
    Always check the `failed` array in responses and implement retry logic for failed items.
  </Accordion>
  <Accordion title="Batch by logical groups">
    Group related content together. If one group fails, other groups remain unaffected.
  </Accordion>
</AccordionGroup>
