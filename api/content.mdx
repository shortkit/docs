The Content API manages your video library. Use it to list content, retrieve details, update metadata and editorial controls, and delete content. Content is created via the [Uploads API](/api/uploads) and delivered to end users via the [Feed API](/api/feed).

## Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/v1/content` | List all content |
| `GET` | `/v1/content/{id}` | Get content by ID |
| `PATCH` | `/v1/content/{id}` | Update content |
| `DELETE` | `/v1/content/{id}` | Delete content |

All Content API endpoints require a **secret key** via the `Authorization` header.

## Content object

Every content item has the following shape:

```json
{
  "id": "sk_content_abc123",
  "title": "Breaking: Weather Alert",
  "description": "Severe thunderstorm warning for the metro area",
  "status": "ready",
  "duration": 32.5,
  "streamingUrl": "https://video.shortkit.dev/sk_content_abc123/stream.m3u8",
  "thumbnailUrl": "https://images.shortkit.dev/sk_content_abc123/thumbnail.jpg",
  "captionTracks": [
    {
      "language": "en",
      "label": "English",
      "source": "embedded",
      "url": null
    }
  ],
  "tags": ["weather", "severe", "breaking"],
  "customMetadata": {
    "articleUrl": "https://example.com/weather-alert",
    "category": "severe-weather"
  },
  "pinPosition": null,
  "isSuppressed": false,
  "publishAt": null,
  "expiresAt": null,
  "createdAt": "2026-02-05T12:00:00Z",
  "updatedAt": "2026-02-05T12:03:45Z"
}
```

### Field reference

<ResponseField name="id" type="string">
  Unique content identifier (e.g., `sk_content_abc123`).
</ResponseField>

<ResponseField name="title" type="string">
  Display title.
</ResponseField>

<ResponseField name="description" type="string | null">
  Optional description text.
</ResponseField>

<ResponseField name="status" type="string">
  Current lifecycle status. One of: `waiting_for_upload`, `processing`, `ready`, `error`, `archived`.
</ResponseField>

<ResponseField name="duration" type="number | null">
  Video duration in seconds. Available once status is `ready`.
</ResponseField>

<ResponseField name="streamingUrl" type="string | null">
  HLS streaming URL. Available once status is `ready`.
</ResponseField>

<ResponseField name="thumbnailUrl" type="string | null">
  Thumbnail image URL. Available once status is `ready`.
</ResponseField>

<ResponseField name="captionTracks" type="array | null">
  Available caption tracks. Each track includes `language`, `label`, `source` (`"embedded"` or `"external"`), and `url` (null for embedded captions).
</ResponseField>

<ResponseField name="tags" type="string[]">
  Content tags for categorization and filtering.
</ResponseField>

<ResponseField name="customMetadata" type="object">
  Arbitrary key-value pairs set during upload or update. Passed through to the feed response.
</ResponseField>

<ResponseField name="pinPosition" type="integer | null">
  When set, pins this content to a specific position in the feed (1 = top). `null` means unpinned.
</ResponseField>

<ResponseField name="isSuppressed" type="boolean">
  When `true`, this content is excluded from the feed regardless of status or scheduling.
</ResponseField>

<ResponseField name="publishAt" type="string | null">
  ISO 8601 timestamp. Content will not appear in the feed before this time. `null` means immediately available.
</ResponseField>

<ResponseField name="expiresAt" type="string | null">
  ISO 8601 timestamp. Content is automatically excluded from the feed after this time. `null` means no expiration.
</ResponseField>

<ResponseField name="createdAt" type="string">
  Creation timestamp (ISO 8601).
</ResponseField>

<ResponseField name="updatedAt" type="string">
  Last modification timestamp (ISO 8601).
</ResponseField>

## Status lifecycle

Content moves through these statuses:

```
POST /v1/uploads
      |
      v
waiting_for_upload  -->  processing  -->  ready  -->  archived
                              |
                              v
                            error
```

| Status | Description | Visible in feed? |
|--------|-------------|------------------|
| `waiting_for_upload` | Upload session created; file not yet received | No |
| `processing` | Video is being transcoded; thumbnails and captions are being generated | No |
| `ready` | Processing complete; content is available for playback | Yes (subject to editorial controls) |
| `error` | Processing failed | No |
| `archived` | Manually archived or expired | No |

Content with status `ready` appears in the feed only when all of these conditions are met:
- `isSuppressed` is `false`
- `publishAt` is `null` or in the past
- `expiresAt` is `null` or in the future

## List content

Retrieves a paginated list of all content in your organization.

```bash
GET /v1/content
```

### Query parameters

<ParamField query="limit" type="integer" default={20}>
  Items per page (1--100).
</ParamField>

<ParamField query="cursor" type="string">
  Pagination cursor from a previous response.
</ParamField>

<ParamField query="status" type="string">
  Filter by status: `waiting_for_upload`, `processing`, `ready`, `error`, `archived`.
</ParamField>

<ParamField query="tags" type="string">
  Comma-separated tags to filter by.
</ParamField>

### Example request

```bash
curl "https://api.shortkit.dev/v1/content?limit=20&status=ready&tags=weather" \
  -H "Authorization: Bearer sk_live_your_secret_key"
```

        {

When there are more results, `meta.nextCursor` contains an opaque cursor string. Pass it as the `cursor` query parameter on the next request. When there are no more results, `nextCursor` is `null`.

## Update content

Updates content metadata and editorial controls. You cannot replace the source video file after upload -- create a new upload instead.

```bash
PATCH /v1/content/{id}
```

### Path parameters

<ParamField path="id" type="string" required>
  Content ID to update.
</ParamField>

### Request body

All fields are optional. Only include the fields you want to change.

<ParamField body="title" type="string">
  Updated display title (max 200 characters).
</ParamField>

<ParamField body="description" type="string">
  Updated description (max 2000 characters). Pass `null` to clear.
</ParamField>

<ParamField body="tags" type="string[]">
  Replaces all existing tags.
</ParamField>

<ParamField body="customMetadata" type="object">
  Merged with existing metadata. Set a key to `null` to remove it.
</ParamField>

<ParamField body="pinPosition" type="integer | null">
  Pin this content to a specific feed position (1 = top of feed). Set to `null` to unpin.
  Multiple items can be pinned; they are ordered by `pinPosition` ascending before unpinned content.
</ParamField>

<ParamField body="isSuppressed" type="boolean">
  Set to `true` to immediately hide this content from the feed. Set to `false` to restore it.
</ParamField>

<ParamField body="publishAt" type="string | null">
  ISO 8601 timestamp. Content will not appear in the feed before this time. Set to `null` for immediate availability.
</ParamField>

<ParamField body="expiresAt" type="string | null">
  ISO 8601 timestamp. Content is excluded from the feed after this time. Set to `null` to remove expiration.
</ParamField>

### Example request

```bash
curl -X PATCH https://api.shortkit.dev/v1/content/sk_content_abc123 \
  -H "Authorization: Bearer sk_live_your_secret_key" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Updated: Severe Weather Warning",
    "pinPosition": 1,
    "expiresAt": "2026-02-06T00:00:00Z",
    "tags": ["weather", "severe", "featured"],
    "customMetadata": {
      "featured": true,
      "category": null
    }
  }'
```

### Response

Returns the full updated content object:

```json
{
  "data": {
    "id": "sk_content_abc123",
    "title": "Updated: Severe Weather Warning",
    "description": "Severe thunderstorm warning for the metro area",
    "status": "ready",
    "duration": 32.5,
    "streamingUrl": "https://video.shortkit.dev/sk_content_abc123/stream.m3u8",
    "thumbnailUrl": "https://images.shortkit.dev/sk_content_abc123/thumbnail.jpg",
    "captionTracks": [
      {
        "language": "en",
        "label": "English",
        "source": "embedded",
        "url": null
      }
    ],
    "tags": ["weather", "severe", "featured"],
    "customMetadata": {
      "articleUrl": "https://example.com/weather-alert",
      "featured": true
    },
    "pinPosition": 1,
    "isSuppressed": false,
    "publishAt": null,
    "expiresAt": "2026-02-06T00:00:00Z",
    "createdAt": "2026-02-05T12:00:00Z",
    "updatedAt": "2026-02-05T14:30:00Z"
  },
  "meta": {
    "requestId": "req_upd001"
  }
}
```

### Editorial controls

Use these fields to control how and when content appears in the feed:

| Control | Field | Effect |
|---------|-------|--------|
| **Pin** | `pinPosition` | Locks content to a specific position in the feed. Pinned items appear before unpinned items, ordered by `pinPosition` ascending. |
| **Suppress** | `isSuppressed` | Immediately removes content from the feed without deleting it. Useful for temporary takedowns. |
| **Schedule** | `publishAt` | Content is invisible in the feed until this timestamp passes. |
| **Expire** | `expiresAt` | Content is automatically removed from the feed after this timestamp. The content is not deleted -- its status changes to `archived`. |

## Delete content

Permanently deletes a content item and all associated data (video files, thumbnails, captions).

```bash
DELETE /v1/content/{id}
```

### Path parameters

<ParamField path="id" type="string" required>
  Content ID to delete.
</ParamField>

### Example request

```bash
curl -X DELETE https://api.shortkit.dev/v1/content/sk_content_abc123 \
  -H "Authorization: Bearer sk_live_your_secret_key"
```

### Response

```json
{
  "data": {
    "id": "sk_content_abc123",
    "deleted": true
  },
  "meta": {
    "requestId": "req_del001"
  }
}
```

<Warning>
  Deletion is permanent. All video files, thumbnails, captions, and associated engagement data
  are removed within 24 hours. This action cannot be undone.
</Warning>
