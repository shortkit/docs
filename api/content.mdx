The Content API manages your video and carousel library. Use it to create content items, attach carousel images and caption tracks, set editorial controls, search by natural language, and archive items you no longer need.

## Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | [`/v1/content`](#create-content) | Create a content item |
| `GET` | [`/v1/content`](#list-content) | List content with filters |
| `GET` | [`/v1/content/search`](#search-content) | Search content |
| `GET` | [`/v1/content/{id}`](#get-content) | Get a single content item |
| `PATCH` | [`/v1/content/{id}`](#update-content) | Update content fields |
| `DELETE` | [`/v1/content/{id}`](#delete-content) | Archive a content item |
| `POST` | [`/v1/content/{id}/images`](#upload-carousel-image) | Upload a carousel image |
| `DELETE` | [`/v1/content/{id}/images/{image_id}`](#delete-carousel-image) | Delete a carousel image |
| `PUT` | [`/v1/content/{id}/images/order`](#reorder-carousel-images) | Reorder carousel images |
| `POST` | [`/v1/content/{id}/captions`](#upload-caption-track) | Upload a caption track (VTT/SRT) |
| `DELETE` | [`/v1/content/{id}/captions/{track_id}`](#delete-caption-track) | Delete a caption track |

All endpoints require a **secret key** (`Authorization: Bearer sk_...`) or a **portal JWT** (`Authorization: Bearer <jwt>`).

<ResponseField name="tags" type="string[]">

<ResponseField name="images" type="array">
  Carousel images. Each entry has `id`, `imageUrl`, `altText`, `position`, and `createdAt`. Empty for video content.
</ResponseField>

<ResponseField name="pinPosition" type="integer | null">

<ResponseField name="height" type="integer | null">

<ResponseField name="aspectRatio" type="string | null">

<ResponseField name="maxFrameRate" type="number | null">

</ResponseField>

<ResponseField name="videoQuality" type="string | null">
  Mux video quality tier (e.g., `"plus"`, `"basic"`).

<ResponseField name="resolutionTier" type="string | null">
  Maximum stored resolution (e.g., `"1080p"`, `"720p"`).
</ResponseField>

<ResponseField name="carouselCaption" type="string | null">

</ResponseField>

<ResponseField name="autoScrollInterval" type="number | null">
  Seconds between automatic image transitions in a carousel. `null` for videos.
</ResponseField>

<ResponseField name="views" type="integer">
  View count for this content item.
</ResponseField>

<ResponseField name="createdAt" type="string">
  Creation timestamp (ISO 8601).
</ResponseField>

<ResponseField name="updatedAt" type="string">

</ResponseField>

## Status lifecycle

| Status | Description | Visible in feed? |
|--------|-------------|------------------|
| `waiting_for_upload` | Upload session created, video file not yet received | No |
| `processing` | Video is being transcoded, thumbnails and captions are being generated | No |
| `ready` | Processing complete, content is playable | Yes (subject to editorial controls) |
| `error` | Processing failed | No |
| `archived` | Soft-deleted via `DELETE /v1/content/{id}` | No |

Carousels skip the upload/processing stages and are created with status `ready` immediately.

Content with status `ready` appears in the feed only when all of these conditions are met:
- `isSuppressed` is `false`
- `publishAt` is `null` or in the past
- `expiresAt` is `null` or in the future

## Create content

Creates a new content item. Videos are created with status `waiting_for_upload` -- use the [Uploads API](/api/content#upload-flow) to upload the video file. Carousels are created with status `ready` immediately.

```bash
POST /v1/content
```

### Authentication

Secret key or portal JWT via the `Authorization` header.

### Request body

<ParamField body="contentType" type="string" default="video">
  `"video"` or `"carousel"`.
</ParamField>

<ParamField body="title" type="string" required>
  Display title. Max 200 characters.
</ParamField>

<ParamField body="description" type="string">
  Optional description. Max 2000 characters.
</ParamField>

<ParamField body="tags" type="string[]">
  Tags for categorization and filtering.
</ParamField>

<ParamField body="caption" type="string">
  Carousel caption text. Max 500 characters. Only applies to carousels.
</ParamField>

<ParamField body="autoScrollInterval" type="number">
  Seconds between automatic image transitions. Only applies to carousels.
</ParamField>

<ParamField body="pinPosition" type="integer">
  Pin this content to a fixed feed position (0-indexed).
</ParamField>

<ParamField body="section" type="string">
  Editorial section. Max 200 characters.
</ParamField>

<ParamField body="author" type="string">
  Content author name. Max 200 characters.
</ParamField>

<ParamField body="publishAt" type="string">
  ISO 8601 timestamp. Content will not appear in the feed before this time.
</ParamField>

<ParamField body="expiresAt" type="string">
  ISO 8601 timestamp. Content is removed from the feed after this time.
</ParamField>

### Example request

```bash
curl -X POST https://api.shortkit.dev/v1/content \
  -H "Authorization: Bearer sk_live_your_secret_key" \
  -H "Content-Type: application/json" \
  -d '{
    "contentType": "carousel",
    "title": "Top 5 Weekend Destinations",
    "section": "Travel",
    "author": "Travel Desk",
    "tags": ["travel", "weekend"],
    "caption": "Swipe through our picks for this weekend.",
    "autoScrollInterval": 5.0
  }'
```

### Response (201)

```json
{
  "data": {

    "contentType": "carousel",
    "title": "Top 5 Weekend Destinations",
    "description": null,
    "status": "ready",
    "duration": null,
    "streamingUrl": null,
    "thumbnailUrl": null,
    "section": "Travel",
    "author": "Travel Desk",
    "tags": ["travel", "weekend"],
    "customMetadata": {},
    "captionTracks": [],
    "publicationUrls": [],
    "images": [],
    "carouselCaption": "Swipe through our picks for this weekend.",
    "autoScrollInterval": 5.0,
    "pinPosition": null,
    "isSuppressed": false,
    "publishAt": null,
    "expiresAt": null,
    "width": null,
    "height": null,
    "aspectRatio": null,
    "maxFrameRate": null,
    "videoQuality": null,
    "resolutionTier": null,
    "views": 0,
```

## List content

Returns a paginated list of content items in your organization.

```bash
GET /v1/content
```

### Authentication

Secret key or portal JWT via the `Authorization` header.

### Query parameters

<ParamField query="limit" type="integer" default={20}>
  Items per page (1--100).
</ParamField>

<ParamField query="cursor" type="string">
  Opaque pagination cursor from a previous response's `meta.nextCursor`. Omit for the first page.
</ParamField>

<ParamField query="status" type="string">
  Filter by status: `waiting_for_upload`, `processing`, `ready`, `error`, `archived`.
</ParamField>

<ParamField query="tags" type="string">
  Comma-separated tag filter. Returns content matching any of the given tags.
</ParamField>

### Example request

```bash
curl "https://api.shortkit.dev/v1/content?limit=20&status=ready&tags=weather,breaking" \
  -H "Authorization: Bearer sk_live_your_secret_key"
```

### Response

```json
{
  "data": [
    {
      "id": "d290f1ee-6c54-4b01-90e6-d701748f0851",
      "contentType": "video",
      "title": "Breaking: Weather Alert",
      "status": "ready",
      "duration": 32.5,
      "section": "Weather",
      "author": "Jane Doe",
      "tags": ["weather", "severe", "breaking"],
      "createdAt": "2026-02-05T12:00:00Z",
      "updatedAt": "2026-02-05T12:03:45Z"
    }
  ],
  "meta": {
    "nextCursor": "eyJjYSI6IjIwMjYtMDItMDVUMTI6MDA6MDBaIiwiaWQiOiJkMjkwZjFlZS02YzU0In0",
    "request_id": "req_list001"
  }
}
```

### Pagination

This endpoint uses cursor-based pagination on `(created_at, id)`. When there are more results, `meta.nextCursor` contains an opaque token. Pass it as the `cursor` query parameter to fetch the next page. When `meta.nextCursor` is absent, you have reached the end.

```bash
# Fetch the next page
curl "https://api.shortkit.dev/v1/content?limit=20&cursor=eyJjYSI6IjIwMjYtMDIt..." \
  -H "Authorization: Bearer sk_live_your_secret_key"
```

<Warning>
  Cursor tokens are opaque. Do not parse, decode, or construct them manually.
</Warning>

## Search content

Search content by title, description, author, tags, and custom metadata. Returns content items matching all search terms.

```bash
GET /v1/content/search
```

### Authentication

Secret key or portal JWT via the `Authorization` header.

### Query parameters

<ParamField query="q" type="string" required>
  Search query. 1--500 characters. Multiple words are treated as separate terms — all must match.
</ParamField>

<ParamField query="limit" type="integer" default={20}>
  Maximum results (1--50).
</ParamField>

### Example request

```bash
curl "https://api.shortkit.dev/v1/content/search?q=weather%20alert&limit=5" \
  -H "Authorization: Bearer sk_live_your_secret_key"
```

### Response

Returns standard content objects ordered by creation date (newest first):

```json
{
  "data": [
    {
      "id": "d290f1ee-6c54-4b01-90e6-d701748f0851",
      "contentType": "video",
      "title": "Breaking: Weather Alert",
      "status": "ready",
      "duration": 32.5,
      "streamingUrl": "https://stream.shortkit.dev/DS00Spx1CV902MCtPj5WknGlR102V5HFkDe.m3u8",
      "thumbnailUrl": "https://image.shortkit.dev/DS00Spx1CV902MCtPj5WknGlR102V5HFkDe/thumbnail.jpg?time=0",
      "tags": ["weather", "severe"],
      "createdAt": "2026-02-05T12:00:00Z",
      "updatedAt": "2026-02-05T12:03:45Z"
    }
  ],
  "meta": {
    "request_id": "req_search001"
  }
}
```

Search matches against title, description, author, tags, and custom metadata fields. Each search term must appear in at least one of these fields for a content item to be included.

## Update content

Updates content fields. Only provided fields are changed -- omitted fields are left untouched. You cannot replace the source video file; create a new upload instead.

```bash
PATCH /v1/content/{id}
```

### Authentication

Secret key or portal JWT via the `Authorization` header.

### Path parameters

<ParamField path="id" type="string" required>
  Content UUID.
</ParamField>

### Request body

All fields are optional. Include only the fields you want to change.

<ParamField body="title" type="string">
  Updated display title. Max 200 characters.
</ParamField>

<ParamField body="description" type="string">
  Updated description. Max 2000 characters.
</ParamField>

<ParamField body="tags" type="string[]">
  Replaces all existing tags.
</ParamField>

<ParamField body="customMetadata" type="object">
  Merged with existing metadata. New keys are added, existing keys are overwritten, and keys set to `null` are removed.
</ParamField>

<ParamField body="section" type="string">
  Editorial section.
</ParamField>

<ParamField body="author" type="string">
  Content author name.
</ParamField>

<ParamField body="pinPosition" type="integer | null">
  Pin to a specific feed position. Set to `null` to unpin.
</ParamField>

<ParamField body="isSuppressed" type="boolean">
  Set to `true` to hide from the feed. Set to `false` to restore.
</ParamField>

<ParamField body="carouselCaption" type="string">
  Carousel caption text.
</ParamField>

<ParamField body="autoScrollInterval" type="number">
  Carousel auto-scroll interval in seconds.
</ParamField>

<ParamField body="publishAt" type="string | null">
  ISO 8601 timestamp. Set to `null` for immediate availability.
</ParamField>

<ParamField body="expiresAt" type="string | null">
  ISO 8601 timestamp. Set to `null` to remove expiration.
</ParamField>

### Example request

```bash
curl -X PATCH https://api.shortkit.dev/v1/content/d290f1ee-6c54-4b01-90e6-d701748f0851 \
  -H "Authorization: Bearer sk_live_your_secret_key" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Updated: Severe Weather Warning",
    "section": "Breaking News",
    "pinPosition": 1,
    "expiresAt": "2026-02-06T00:00:00Z",
    "customMetadata": {
      "featured": true,
      "category": null
    }
  }'
```

### Response

Returns the full updated [content object](#content-object).

### customMetadata merge behavior

`customMetadata` is merged, not replaced. Given existing metadata `{"a": 1, "b": 2}` and an update of `{"b": 3, "c": 4, "a": null}`, the result is `{"b": 3, "c": 4}`. Key `"a"` was removed because its value was `null`, key `"b"` was overwritten, and key `"c"` was added.

## Delete content

Soft-deletes a content item by setting its status to `archived`. Archived content is removed from the feed but can still be retrieved via `GET /v1/content/{id}`.

```bash
DELETE /v1/content/{id}
```

### Authentication

Secret key or portal JWT via the `Authorization` header.

### Path parameters

<ParamField path="id" type="string" required>
  Content UUID to archive.
</ParamField>

### Example request

```bash
curl -X DELETE https://api.shortkit.dev/v1/content/d290f1ee-6c54-4b01-90e6-d701748f0851 \
  -H "Authorization: Bearer sk_live_your_secret_key"
```

### Response

```json
{
  "data": {
    "id": "d290f1ee-6c54-4b01-90e6-d701748f0851",
    "deleted": true
  },
  "meta": {
    "request_id": "req_del001"
  }
}
```

## Carousel images

Carousel content supports up to 10 images. Upload images, delete them, or reorder them with the endpoints below.

### Upload carousel image

Uploads an image to a carousel content item.

```bash
POST /v1/content/{id}/images
```

### Authentication

Secret key or portal JWT via the `Authorization` header.

### Path parameters

<ParamField path="id" type="string" required>
  Content UUID. Must be a carousel (`contentType: "carousel"`).
</ParamField>

### Request body

`multipart/form-data` with the following fields:

<ParamField body="file" type="File" required>
  Image file (JPEG, PNG, WebP, etc.).
</ParamField>

<ParamField body="position" type="integer" default={0}>
  Sort order for this image in the carousel.
</ParamField>

<ParamField body="altText" type="string">
  Accessibility alt text for the image.
</ParamField>

### Example request

```bash
curl -X POST https://api.shortkit.dev/v1/content/d290f1ee-.../images \
  -H "Authorization: Bearer sk_live_your_secret_key" \
  -F "file=@beach.jpg" \
  -F "position=0" \
  -F "altText=Sunset over the beach"
```

### Response (201)

```json
{
  "data": {
    "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "imageUrl": "https://storage.googleapis.com/shortkit-assets/orgs/.../image.jpg",
    "altText": "Sunset over the beach",
    "position": 0,
    "createdAt": "2026-02-05T12:05:00Z"
  },
  "meta": {
    "request_id": "req_img001"
  }
}
```

Returns `400` if the content is not a carousel or already has 10 images.

### Delete carousel image

```bash
DELETE /v1/content/{id}/images/{image_id}
```

### Authentication

Secret key or portal JWT via the `Authorization` header.

### Path parameters

<ParamField path="id" type="string" required>
  Content UUID.
</ParamField>

<ParamField path="image_id" type="string" required>
  Image UUID to delete.
</ParamField>

### Example request

```bash
curl -X DELETE https://api.shortkit.dev/v1/content/d290f1ee-.../images/a1b2c3d4-... \
  -H "Authorization: Bearer sk_live_your_secret_key"
```

### Response

```json
{
  "data": { "deleted": true },
  "meta": { "request_id": "req_imgdel001" }
}
```

### Reorder carousel images

Sets the display order of all images in a carousel. Pass the full list of image IDs in the desired order.

```bash
PUT /v1/content/{id}/images/order
```

### Authentication

Secret key or portal JWT via the `Authorization` header.

### Path parameters

<ParamField path="id" type="string" required>
  Content UUID.
</ParamField>

### Request body

<ParamField body="imageIds" type="string[]" required>
  Ordered array of image UUIDs. Each image's `position` is set to its index in this array.
</ParamField>

### Example request

```bash
curl -X PUT https://api.shortkit.dev/v1/content/d290f1ee-.../images/order \
  -H "Authorization: Bearer sk_live_your_secret_key" \
  -H "Content-Type: application/json" \
  -d '{
    "imageIds": [
      "image-uuid-3",
      "image-uuid-1",
      "image-uuid-2"
    ]
  }'
```

### Response

Returns the reordered image array:

```json
{
  "data": [
    { "id": "image-uuid-3", "imageUrl": "...", "altText": "...", "position": 0, "createdAt": "..." },
    { "id": "image-uuid-1", "imageUrl": "...", "altText": null, "position": 1, "createdAt": "..." },
    { "id": "image-uuid-2", "imageUrl": "...", "altText": null, "position": 2, "createdAt": "..." }
  ],
  "meta": { "request_id": "req_reorder001" }
}
```

## Caption tracks

Upload VTT or SRT caption files to any content item. Auto-generated captions (from Mux) appear automatically after video processing -- use these endpoints to add additional languages or replace generated tracks.

### Upload caption track

```bash
POST /v1/content/{id}/captions
```

### Authentication

Secret key or portal JWT via the `Authorization` header.

### Path parameters

<ParamField path="id" type="string" required>
  Content UUID.
</ParamField>

### Request body

`multipart/form-data` with the following fields:

<ParamField body="file" type="File" required>
  Caption file. Must have a `.vtt` or `.srt` extension.
</ParamField>

<ParamField body="language" type="string" default="en">
  ISO 639-1 language code (e.g., `"en"`, `"es"`, `"fr"`).
</ParamField>

<ParamField body="label" type="string" default="English">
  Human-readable label shown in the caption picker.
</ParamField>

### Example request

```bash
curl -X POST https://api.shortkit.dev/v1/content/d290f1ee-.../captions \
  -H "Authorization: Bearer sk_live_your_secret_key" \
  -F "file=@captions_es.vtt" \
  -F "language=es" \
  -F "label=Spanish"
```

### Response (201)

```json
{
  "data": {
    "id": "b2c3d4e5-f6a7-8901-bcde-f12345678901",
    "language": "es",
    "label": "Spanish",
    "source": "external",
    "url": "https://storage.googleapis.com/shortkit-assets/orgs/.../captions_es.vtt",
    "muxTrackId": null,
```

## Editorial controls

Use these fields on `PATCH /v1/content/{id}` to control how and when content appears in the feed:

| Control | Field | Effect |
|---------|-------|--------|
| **Pin** | `pinPosition` | Locks content to a fixed position in the feed. Pinned items appear before unpinned items, ordered by `pinPosition` ascending. Set to `null` to unpin. |
| **Suppress** | `isSuppressed` | Immediately hides content from the feed without archiving it. Set to `false` to restore. |
| **Schedule** | `publishAt` | Content is invisible in the feed until this timestamp passes. Set to `null` for immediate availability. |
| **Expire** | `expiresAt` | Content is automatically removed from the feed after this timestamp. The content is not deleted -- update or clear `expiresAt` to make it visible again. |

## Error responses

All content endpoints return errors in the standard envelope format:

| HTTP Status | Error code | Cause |
|-------------|------------|-------|
| `400` | `invalid_request` | Invalid content ID, missing required fields, invalid `contentType`, unsupported file format |
| `401` | `invalid_api_key` | Missing or invalid API key |
| `403` | `forbidden` | Publishable key used on a secret-key endpoint |
| `404` | `not_found` | Content, image, or caption track not found |
| `429` | `rate_limited` | Too many requests |
