---
title: "Update Content"
api: "PATCH /v1/content/{id}"
description: "Update fields on a content item. Only the fields you include in the request body are modified."
---

## Path parameters

<ParamField path="id" type="string" required>
  The content item's UUID.
</ParamField>

## Request body

All fields are optional — include only the fields you want to change.

<ParamField body="title" type="string">
  Display title. Max 200 characters.
</ParamField>

<ParamField body="description" type="string">
  Optional description. Max 2000 characters.
</ParamField>

<ParamField body="tags" type="string[]">
  Replaces existing tags entirely.
</ParamField>

<ParamField body="section" type="string">
  Editorial section. Max 200 characters.
</ParamField>

<ParamField body="author" type="string">
  Content author name. Max 200 characters.
</ParamField>

<ParamField body="publishStatus" type="string">
  Set to `"published"` or `"unpublished"` to control feed visibility. Only applies when `uploadStatus` is `ready`.
</ParamField>

<ParamField body="publishAt" type="string">
  ISO 8601 timestamp. Content won't appear in the feed before this time.
</ParamField>

<ParamField body="expiresAt" type="string">
  ISO 8601 timestamp. Content is removed from the feed after this time.
</ParamField>

<ParamField body="autoScrollInterval" type="number">
  Carousel auto-scroll interval in seconds. Only applies to carousels.
</ParamField>

<ParamField body="origin" type="string">
  `"ios_upload"` or `"other"`. Retroactively marks content as iOS-recorded to enable color-accurate thumbnail extraction. When set to `"ios_upload"` on a video that has already finished processing, thumbnail re-extraction is triggered automatically. See [Upload → origin](/api/content/upload#param-origin) for details on the field.
</ParamField>

<ParamField body="customMetadata" type="object">
  Merged with existing metadata. To remove a key, set it to `null`.
</ParamField>

## Metadata merge behavior

`customMetadata` uses merge semantics — new keys are added, existing keys are updated, and keys set to `null` are removed. Other fields (like `tags`) are replaced entirely.

```json
// Before: {"articleUrl": "...", "priority": "high", "campaign": "summer"}

// Request: {"customMetadata": {"priority": null, "campaign": "fall"}}

// After:  {"articleUrl": "...", "campaign": "fall"}
```
