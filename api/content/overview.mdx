---
title: "Content"
description: "Create, list, update, search, and delete content items. Content represents videos and image carousels in your feed."
---

A content item is the core resource in ShortKit. It represents either a **video** or an **image carousel** that appears in your feed.

Videos go through a processing lifecycle (`waiting_for_upload` → `processing` → `ready`), while carousels are immediately `ready` after creation. Content only appears in the feed when its status is `ready` and all editorial controls (scheduling, suppression, expiration) allow it.

### Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | [`/v1/content`](#create-content) | Create a content item |
| `GET` | [`/v1/content`](#list-content) | List content with filters |
| `GET` | [`/v1/content/{id}`](#get-content) | Get a single content item |
| `GET` | [`/v1/content/count`](#get-content-count) | Get content count |
| `PATCH` | [`/v1/content/{id}`](#update-content) | Update content fields |
| `DELETE` | [`/v1/content/{id}`](#delete-content) | Archive a content item |
| `GET` | [`/v1/content/search`](#search-content) | Search content |

```json
  },
  "captionTracks": [],
  "publicationUrls": [],
  "images": [],
  "pinPosition": null,
  "isSuppressed": false,
  "publishAt": null,
  "expiresAt": null,
  "width": 1080,
  "height": 1920,
  "aspectRatio": "9:16",

  "createdAt": "2026-02-05T12:00:00Z",
  "updatedAt": "2026-02-05T12:03:45Z"

```

### Attributes

<ResponseField name="id" type="string">
  Unique identifier (UUID) for the content item.
</ResponseField>

<ResponseField name="contentType" type="string">
  Either `"video"` or `"carousel"`.
</ResponseField>

<ResponseField name="title" type="string">
  Display title shown in the feed. Max 200 characters.
</ResponseField>

<ResponseField name="description" type="string | null">
  Optional longer description. Max 2000 characters.
</ResponseField>

<ResponseField name="status" type="string">
  Lifecycle status. One of `waiting_for_upload`, `processing`, `ready`, `error`, or `archived`. See [Status lifecycle](#status-lifecycle) below.
</ResponseField>

<ResponseField name="duration" type="number | null">
  Video duration in seconds. Set automatically when processing completes. `null` for carousels.
</ResponseField>

<ResponseField name="streamingUrl" type="string | null">
  HLS streaming URL for video playback. Set automatically when processing completes. `null` for carousels.
</ResponseField>

<ResponseField name="thumbnailUrl" type="string | null">
  Thumbnail image URL. For videos, auto-generated from the first frame. For carousels, the first image.
</ResponseField>

<ResponseField name="section" type="string | null">
  Editorial section (e.g., `"Sports"`, `"News"`). Max 200 characters. Passed through to the feed.
</ResponseField>

<ResponseField name="author" type="string | null">
  Content author name. Max 200 characters. Passed through to the feed.
</ResponseField>

<ResponseField name="tags" type="string[]">
  Tags for categorization and filtering.
</ResponseField>

<ResponseField name="customMetadata" type="object">
  Arbitrary key-value pairs passed through to the feed. Use this for article URLs, campaign IDs, deep links, or any app-specific data.
</ResponseField>

<ResponseField name="captionTracks" type="array">
  Caption tracks attached to this content. Each track has `id`, `language` (ISO 639-1), `label`, `source`, `url`, `status`, and `createdAt`.
</ResponseField>

<ResponseField name="images" type="array">
  Carousel images. Each has `id`, `imageUrl`, `altText`, `position`, and `createdAt`. Empty for videos.
</ResponseField>

<ResponseField name="pinPosition" type="integer | null">
  When set, locks this content to a fixed position in the feed. `null` means unpinned.
</ResponseField>

<ResponseField name="isSuppressed" type="boolean">
  When `true`, this content is hidden from the feed regardless of other settings.
</ResponseField>

<ResponseField name="publishAt" type="string | null">
  ISO 8601 timestamp. Content won't appear in the feed before this time. `null` means immediately available.
</ResponseField>

<ResponseField name="expiresAt" type="string | null">
  ISO 8601 timestamp. Content is removed from the feed after this time. `null` means no expiration.
</ResponseField>

<ResponseField name="width" type="integer | null">
  Video width in pixels. Set after processing.
</ResponseField>

<ResponseField name="height" type="integer | null">
  Video height in pixels. Set after processing.
</ResponseField>

<ResponseField name="aspectRatio" type="string | null">
  Aspect ratio string (e.g., `"9:16"`, `"16:9"`).
</ResponseField>

<ResponseField name="views" type="integer">
  Total view count.
</ResponseField>

<ResponseField name="createdAt" type="string">
  ISO 8601 creation timestamp.
</ResponseField>

<ResponseField name="updatedAt" type="string">
  ISO 8601 last modification timestamp.
</ResponseField>

### Status lifecycle

| Status | Description | In feed? |
|--------|-------------|----------|
| `waiting_for_upload` | Upload session created, video file not yet received | No |
| `processing` | Video is being transcoded and thumbnails/captions generated | No |
| `ready` | Processing complete, content is playable | Yes |
| `error` | Processing failed | No |
| `archived` | Soft-deleted via DELETE | No |

Carousels skip `waiting_for_upload` and `processing` — they're created as `ready` immediately.

Content with status `ready` appears in the feed only when **all** of these are true:
- `isSuppressed` is `false`
- `publishAt` is `null` or in the past
- `expiresAt` is `null` or in the future

## Create content

```
POST /v1/content
```

Creates a new content item. Videos are created with status `waiting_for_upload` — use the [Uploads API](/api/uploads) to upload the video file. Carousels are created with status `ready` immediately.

### Request body

<ParamField body="contentType" type="string" default="video">
  `"video"` or `"carousel"`.
</ParamField>

<ParamField body="title" type="string" required>
  Display title. Max 200 characters.
</ParamField>

<ParamField body="description" type="string">
  Optional description. Max 2000 characters.
</ParamField>

<ParamField body="tags" type="string[]">
  Tags for categorization and filtering.
</ParamField>

<ParamField body="section" type="string">
  Editorial section (e.g., `"Sports"`, `"News"`). Max 200 characters.
</ParamField>

<ParamField body="author" type="string">
  Content author name. Max 200 characters.
</ParamField>

<ParamField body="caption" type="string">
  Carousel caption text. Max 500 characters. Only applies to carousels.
</ParamField>

<ParamField body="autoScrollInterval" type="number">
  Seconds between automatic image transitions. Only applies to carousels.
</ParamField>

<ParamField body="pinPosition" type="integer">
  Pin this content to a fixed feed position (0-indexed).
</ParamField>

<ParamField body="publishAt" type="string">
  ISO 8601 timestamp. Content won't appear in the feed before this time.
</ParamField>

<ParamField body="expiresAt" type="string">
  ISO 8601 timestamp. Content is removed from the feed after this time.
</ParamField>

### Example request

## List content

```
GET /v1/content
```

Returns a paginated list of content items in your organization.

### Query parameters

<ParamField query="limit" type="integer" default={20}>
  Items per page. Min 1, max 100.
</ParamField>

<ParamField query="cursor" type="string">
  Pagination cursor from a previous response's `meta.nextCursor`. Omit for the first page.
</ParamField>

<ParamField query="status" type="string">
  Filter by status: `waiting_for_upload`, `processing`, `ready`, `error`, or `archived`.
</ParamField>

<ParamField query="tags" type="string">
  Comma-separated tag filter. Returns content matching **any** of the given tags.
</ParamField>

### Example request

<Tabs>
  <Tab title="cURL">
    ```bash
    curl "https://api.shortkit.dev/v1/content?limit=20&status=ready&tags=weather,breaking" \
      -H "Authorization: Bearer sk_live_your_secret_key"
    ```
  </Tab>
  <Tab title="Python">
    ```python
    import requests

    resp = requests.get(
        "https://api.shortkit.dev/v1/content",
        headers={"Authorization": "Bearer sk_live_your_secret_key"},
        params={
            "limit": 20,
            "status": "ready",
            "tags": "weather,breaking",
        },
    )
    items = resp.json()["data"]
    next_cursor = resp.json()["meta"].get("nextCursor")
    ```
  </Tab>
  <Tab title="Node.js">
    ```javascript
    const resp = await fetch(
      "https://api.shortkit.dev/v1/content?limit=20&status=ready&tags=weather,breaking",
      {
        headers: { Authorization: "Bearer sk_live_your_secret_key" },
      }
    );
    const { data: items, meta } = await resp.json();
    const nextCursor = meta.nextCursor;
    ```
  </Tab>
</Tabs>

### Pagination

This endpoint uses cursor-based pagination on `(created_at, id)`. When more results exist, `meta.nextCursor` contains an opaque token. Pass it as the `cursor` query parameter to fetch the next page.

```bash
# Fetch the next page
curl "https://api.shortkit.dev/v1/content?limit=20&cursor=eyJjYSI6IjIwMjYtMDIt..." \
  -H "Authorization: Bearer sk_live_your_secret_key"
```

When `meta.nextCursor` is absent, you've reached the end.

<Warning>
  Cursor tokens are opaque. Do not parse, decode, or construct them manually.
</Warning>
