title: "List Content"

---

## Query parameters

<ParamField query="limit" type="integer" default={20}>
  Items per page. Min 1, max 100.
</ParamField>

<ParamField query="cursor" type="string">
  Pagination cursor from a previous response's `meta.nextCursor`. Omit for the first page.
</ParamField>

<ParamField query="uploadStatus" type="string">
  Filter by upload status: `pending`, `processing`, `ready`, or `error`.
</ParamField>

<ParamField query="tags" type="string">
  Comma-separated tag filter. Returns content matching **any** of the given tags.
</ParamField>

<RequestExample>
```bash cURL
curl "https://api.shortkit.dev/v1/content?limit=20&uploadStatus=ready&tags=weather,breaking" \
  -H "Authorization: Bearer sk_live_your_secret_key"
```

```python Python
import requests

resp = requests.get(

        "uploadStatus": "ready",
        "tags": "weather,breaking",
    },

items = resp.json()["data"]
```

## Pagination

This endpoint uses cursor-based pagination. When more results exist, `meta.nextCursor` contains an opaque token — pass it as the `cursor` query parameter to fetch the next page.

```bash
# Fetch the next page
curl "https://api.shortkit.dev/v1/content?limit=20&cursor=eyJjYSI6IjIwMjYtMDIt..." \
  -H "Authorization: Bearer sk_live_your_secret_key"
```

When `meta.nextCursor` is absent, you've reached the end.

## Filtering

You can filter by `uploadStatus` and `tags` using the query parameters above. Additional filters (by `publishStatus`, `section`, `contentType`, `author`, and date ranges) are coming soon.
