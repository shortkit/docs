Bulk operations allow you to update or archive multiple content items in a single request, reducing API calls and improving efficiency for managing large content libraries.

## Endpoints

| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| `POST` | `/v1/content/bulk/update` | Secret key | Update multiple content items |
| `POST` | `/v1/content/bulk/archive` | Secret key | Archive multiple content items |
| `POST` | `/v1/content/bulk/tag` | Secret key | Add or remove tags from multiple items |

## Bulk update

Update metadata, editorial controls, or scheduling for multiple content items.

```bash
POST /v1/content/bulk/update
```

### Request body

<ParamField body="items" type="array" required>
  Array of update operations (max 100 items).

  <Expandable title="Item properties">
    <ParamField body="id" type="string" required>
      Content ID to update.
    </ParamField>
    <ParamField body="title" type="string">
      New title.
    </ParamField>
    <ParamField body="description" type="string">
      New description.
    </ParamField>
    <ParamField body="tags" type="string[]">
      Replace tags.
    </ParamField>
    <ParamField body="customMetadata" type="object">
      Merge custom metadata.
    </ParamField>
    <ParamField body="pinPosition" type="integer">
      Pin to a specific feed position (1-indexed). Set to `null` to unpin.
    </ParamField>
    <ParamField body="isSuppressed" type="boolean">
      Suppress content from appearing in the feed.
    </ParamField>
    <ParamField body="publishAt" type="string">
      Scheduled publish time (ISO 8601). Content appears in the feed after this time.
    </ParamField>
    <ParamField body="expiresAt" type="string">
      Expiration time (ISO 8601). Content is removed from the feed after this time.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="continueOnError" type="boolean" default={false}>
  Continue processing remaining items if individual items fail.
</ParamField>

### Example request

```bash
curl -X POST https://api.shortkit.dev/v1/content/bulk/update \
  -H "Authorization: Bearer sk_live_your_secret_key" \
  -H "Content-Type: application/json" \
  -d '{
    "items": [
      {
        "id": "sk_content_abc123",
        "title": "Updated Weather Alert",
        "pinPosition": 1
      },
      {
        "id": "sk_content_def456",
        "tags": ["featured", "breaking"],
        "isSuppressed": false
      }
    ],
    "continueOnError": true
  }'
```

### Response

```json
{
  "data": {
    "successful": [
      { "id": "sk_content_abc123", "status": "updated" },
      { "id": "sk_content_def456", "status": "updated" }
    ],
    "failed": [],
    "totalProcessed": 2,
    "totalSuccessful": 2,
    "totalFailed": 0
  },
  "meta": {
    "request_id": "req_xyz789"
  }
}
```

### Partial failure response

When `continueOnError` is `true` and some items fail:

```json
{
  "data": {
    "successful": [
      { "id": "sk_content_abc123", "status": "updated" }
    ],
    "failed": [
      {
        "id": "sk_content_invalid",
        "error": {
          "code": "not_found",
          "message": "Content not found"
        }
      }
    ],
    "totalProcessed": 2,
    "totalSuccessful": 1,
    "totalFailed": 1
  },
  "meta": {
    "request_id": "req_xyz789"
  }
}
```

## Bulk archive

Archive multiple content items. Archived content is removed from the feed but its metadata and analytics are preserved.

```bash
POST /v1/content/bulk/archive
```

### Request body

<ParamField body="ids" type="string[]" required>
  Content IDs to archive (max 100).
</ParamField>

<ParamField body="continueOnError" type="boolean" default={false}>
  Continue if individual archival operations fail.
</ParamField>

### Example request

```bash
curl -X POST https://api.shortkit.dev/v1/content/bulk/archive \
  -H "Authorization: Bearer sk_live_your_secret_key" \
  -H "Content-Type: application/json" \
  -d '{
    "ids": ["sk_content_abc123", "sk_content_def456", "sk_content_ghi789"],
    "continueOnError": true
  }'
```

### Response

```json
{
  "data": {
    "successful": [
      { "id": "sk_content_abc123", "status": "archived" },
      { "id": "sk_content_def456", "status": "archived" },
      { "id": "sk_content_ghi789", "status": "archived" }
    ],
    "failed": [],
    "totalProcessed": 3,
    "totalSuccessful": 3,
    "totalFailed": 0
  },
  "meta": {
    "request_id": "req_abc456"
  }
}
```

## Bulk tag

Add or remove tags from multiple content items at once.

```bash
POST /v1/content/bulk/tag
```

### Request body

<ParamField body="ids" type="string[]" required>
  Content IDs to modify (max 100).
</ParamField>

<ParamField body="addTags" type="string[]">
  Tags to add.
</ParamField>

<ParamField body="removeTags" type="string[]">
  Tags to remove.
</ParamField>

### Example request

```bash
curl -X POST https://api.shortkit.dev/v1/content/bulk/tag \
  -H "Authorization: Bearer sk_live_your_secret_key" \
  -H "Content-Type: application/json" \
  -d '{
    "ids": ["sk_content_abc123", "sk_content_def456"],
    "addTags": ["featured", "homepage"],
    "removeTags": ["draft"]
  }'
```

### Response

```json
{
  "data": {
    "successful": [
      {
        "id": "sk_content_abc123",
        "tags": ["featured", "homepage", "weather"]
      },
      {
        "id": "sk_content_def456",
        "tags": ["featured", "homepage"]
      }
    ],
    "failed": [],
    "totalProcessed": 2,
    "totalSuccessful": 2,
    "totalFailed": 0
  },
  "meta": {
    "request_id": "req_tag789"
  }
}
```

## Rate limits

Bulk operations have specific rate limits:

| Limit | Value |
|-------|-------|
| Items per request | 100 |
| Requests per minute | 60 |
| Items per minute | 1,000 |

For larger operations, space your requests accordingly.

## Best practices

<AccordionGroup>
  <Accordion title="Use continueOnError for resilience">
    When processing large batches, enable `continueOnError` to ensure one failure does not stop the entire operation.
  </Accordion>
  <Accordion title="Handle partial failures">
    Always check the `failed` array in responses and implement retry logic for failed items.
  </Accordion>
  <Accordion title="Batch by logical groups">
    Group related content together. If one group fails, other groups remain unaffected.
  </Accordion>
</AccordionGroup>
