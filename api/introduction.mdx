The ShortKit API provides programmatic access to all platform capabilities -- content ingestion, feed delivery, remote configuration, engagement analytics, and identity resolution.

## Base URL

All API requests use the following base URL:

```
https://api.shortkit.dev/v1
```

All endpoints are served over HTTPS. Requests made over plain HTTP will be rejected.

## Content type

All request and response bodies use JSON. Include the `Content-Type` header on requests that contain a body:

```
Content-Type: application/json
```

## API versioning

The API version is included in the URL path (`/v1/`). Breaking changes are only introduced in new major versions.

Non-breaking additions -- such as new response fields, new optional parameters, and new endpoints -- are added to the current version without a version bump. Your integration should gracefully ignore unrecognized fields.

## Response envelope

All responses use a consistent envelope structure with `data`, `meta`, and `errors` fields.

### Success response (single resource)

```json
{
  "data": {
    "id": "sk_content_abc123",
    "title": "Breaking: Weather Alert",
    "description": "Severe thunderstorm warning for the metro area",
    "status": "ready",
    "duration": 32.5,
    "streamingUrl": "https://video.shortkit.dev/d7f2a9b1c3e4.m3u8",
    "thumbnailUrl": "https://images.shortkit.dev/d7f2a9b1c3e4/thumbnail.jpg",
    "customMetadata": {
      "articleUrl": "https://weather.com/severe-storm",
      "category": "severe-weather"
    }
  },
  "meta": {
    "request_id": "req_8f3a2b1c"
  }
}
```

### Success response (list)

```json
{
  "data": [
    {
      "id": "sk_content_abc123",
      "title": "Breaking: Weather Alert",
      "duration": 32.5
    },
    {
      "id": "sk_content_def456",
      "title": "Game Highlights: Final Four",
      "duration": 45.0
    }
  ],
  "meta": {
    "next_cursor": "eyJjYSI6IjIwMjYtMDItMDVUMTI6MDA6MDBaIiwiaWQiOiJza19jb250ZW50X2RlZjQ1NiJ9",
    "request_id": "req_9d4e3f2a"
  }
}
```

### Error response

```json
{
  "data": null,
  "meta": {
    "request_id": "req_7c1b4a5d"
  },
  "errors": [
    {
      "code": "invalid_request",
      "message": "The 'limit' parameter must be an integer between 1 and 50.",
      "field": "limit"
    }
  ]
}
```

<Note>
  Every response -- success or error -- includes a `meta.request_id` field. Include this value when contacting support to enable fast debugging.
</Note>

## Pagination

List endpoints use **cursor-based pagination** with opaque cursor tokens. This provides consistent performance regardless of how deep into the result set you paginate.

### Parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `limit` | integer | `10` | Number of items to return per page. Maximum: `50`. |
| `cursor` | string | -- | Opaque cursor token from the previous response's `meta.next_cursor`. Omit for the first page. |

### Paginating through results

```bash
# First page
curl https://api.shortkit.dev/v1/feed?limit=10 \
  -H "X-API-Key: pk_live_your_publishable_key"

# Next page -- use the cursor from the previous response
curl "https://api.shortkit.dev/v1/feed?limit=10&cursor=eyJjYSI6IjIwMjYtMDItMDVUMTI6MDA6MDBaIiwiaWQiOiJza19jb250ZW50X2RlZjQ1NiJ9" \
  -H "X-API-Key: pk_live_your_publishable_key"
```

When `meta.next_cursor` is `null` or absent, you have reached the end of the result set.

<Warning>
  Cursor tokens are opaque and may change format between API versions. Do not parse, decode, or construct them manually. Always use the exact value returned by the API.
</Warning>

## Rate limiting

API requests are rate-limited per organization to ensure platform stability.

| Plan | Requests per minute |
|------|---------------------|
| Standard | 1,000 |
| Pro | 5,000 |
| Enterprise | Custom |

Every response includes rate limit headers:

```
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 947
X-RateLimit-Reset: 1738785660
```

| Header | Description |
|--------|-------------|
| `X-RateLimit-Limit` | Maximum requests allowed per minute for your plan. |
| `X-RateLimit-Remaining` | Requests remaining in the current window. |
| `X-RateLimit-Reset` | Unix timestamp (seconds) when the rate limit window resets. |

When you exceed the limit, the API returns a `429 Too Many Requests` response with a `Retry-After` header indicating how many seconds to wait before retrying:

```
HTTP/1.1 429 Too Many Requests
Retry-After: 12
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 0
X-RateLimit-Reset: 1738785660
```

## Environments

ShortKit provides two fully isolated environments. Data created with `live` keys is completely separate from data created with `test` keys.

| Environment | Key prefixes | Purpose |
|-------------|--------------|---------|
| **Live** | `pk_live_`, `sk_live_` | Production traffic, real content |
| **Test** | `pk_test_`, `sk_test_` | Development and testing |

Use environment-specific API keys. A `test` key cannot access `live` data and vice versa.
