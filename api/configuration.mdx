The Configuration API delivers a sparse config overlay to the SDK at runtime. The overlay contains only the fields you have explicitly changed -- the SDK merges it on top of its compiled-in defaults. This means you can change player behavior, theming, and feed settings without shipping an app update.

## Endpoints

| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| `GET` | `/v1/config` | Publishable key | Retrieve the active config overlay |
| `PUT` | `/v1/config` | Secret key | Update the config overlay |

## How the overlay works

The SDK ships with sensible defaults for every configurable property. When the SDK initializes, it calls `GET /v1/config` to fetch any overrides you have set. The response is **sparse** -- it only includes fields that differ from defaults. The SDK deep-merges the overlay on top of its defaults.

```
SDK defaults (compiled-in)
    + config overlay (from GET /v1/config)
    ─────────────────────────────────────
    = effective configuration
```

For example, if the SDK default for `controls.scrubber` is `true` and you set it to `false` via the overlay, the overlay response contains only that field. Every other setting keeps its default value.

## Get config overlay

Retrieve the current config overlay for your organization. The SDK calls this endpoint automatically on initialization.

```bash
GET /v1/config
```

### Authentication

Include your publishable key in the `X-API-Key` header.

### Headers

| Header | Description |
|--------|-------------|
| `X-API-Key` | Your publishable key (`pk_live_...` or `pk_test_...`) |
| `X-User-Id` | Optional. User identifier for future per-user experiment assignments. |

### Example request

```bash
curl https://api.shortkit.dev/v1/config \
  -H "X-API-Key: pk_live_your_publishable_key"
```

### Example response

The response contains only the fields that differ from SDK defaults:

```json
{
  "data": {
    "controls": {
      "scrubber": false
    },
    "theme": {
      "primaryColor": "#FF5733"
    }
  },
  "meta": {
    "request_id": "req_abc123"
  }
}
```

If no overrides have been set, the response is an empty object:

```json
{
  "data": {},
  "meta": {
    "request_id": "req_def456"
  }
}
```

### Merge example

Given these SDK defaults:

```json
{
  "controls": { "scrubber": true, "muteButton": true, "shareButton": true },
  "playback": { "autoplay": true, "muted": true, "loop": true, "preloadCount": 3 },
  "captions": { "defaultOn": false },
  "theme": { "primaryColor": "#FFFFFF", "backgroundColor": "#000000" },
  "feed": { "pageSize": 10, "refreshInterval": 300 }
}
```

And this overlay from `GET /v1/config`:

```json
{
  "controls": { "scrubber": false },
  "theme": { "primaryColor": "#FF5733" }
}
```

The SDK resolves the effective configuration as:

```json
{
  "controls": { "scrubber": false, "muteButton": true, "shareButton": true },
  "playback": { "autoplay": true, "muted": true, "loop": true, "preloadCount": 3 },
  "captions": { "defaultOn": false },
  "theme": { "primaryColor": "#FF5733", "backgroundColor": "#000000" },
  "feed": { "pageSize": 10, "refreshInterval": 300 }
}
```

Only `controls.scrubber` and `theme.primaryColor` changed. Everything else kept its default.

## Update config overlay

Set or update the config overlay for your organization. Only include the fields you want to override -- omitted fields are removed from the overlay (reverting to SDK defaults).

```bash
PUT /v1/config
```

### Authentication

Requires your **secret key** as a Bearer token.

### Example request

```bash
curl -X PUT https://api.shortkit.dev/v1/config \
  -H "Authorization: Bearer sk_live_your_secret_key" \
  -H "Content-Type: application/json" \
  -d '{
    "controls": {
      "scrubber": false,
      "shareButton": false
    },
    "playback": {
      "preloadCount": 5
    },
    "captions": {
      "defaultOn": true
    },
    "theme": {
      "primaryColor": "#FF5733"
    }
  }'
```

### Response

Returns the full overlay as saved:

```json
{
  "data": {
    "controls": {
      "scrubber": false,
      "shareButton": false
    },
    "playback": {
      "preloadCount": 5
    },
    "captions": {
      "defaultOn": true
    },
    "theme": {
      "primaryColor": "#FF5733"
    }
  },
  "meta": {
    "request_id": "req_xyz789"
  }
}
```

### Reset to defaults

To reset all overrides and revert entirely to SDK defaults, send an empty object:

```bash
curl -X PUT https://api.shortkit.dev/v1/config \
  -H "Authorization: Bearer sk_live_your_secret_key" \
  -H "Content-Type: application/json" \
  -d '{}'
```

## SDK sync

The SDK fetches the config overlay automatically on initialization and refreshes it periodically (default: every 5 minutes). Configuration changes take effect on the next SDK refresh without requiring an app update.

## Best practices

<AccordionGroup>
  <Accordion title="Keep overlays small">
    Only override what you need. A smaller overlay means faster config resolution and less risk of unintended side effects.
  </Accordion>
  <Accordion title="Test in staging first">
    Use your `pk_test_...` key to verify configuration changes before applying them in production.
  </Accordion>
  <Accordion title="Use A/B testing for significant changes">
    For changes that may affect engagement (hiding controls, changing preload behavior), use experiments to roll out to a subset of users first.
  </Accordion>
</AccordionGroup>
