---
title: "List Live Streams"
api: "GET /v1/live-streams"
description: "List your organization's live streams, optionally filtered by status."
---

Returns live streams in reverse chronological order by creation time. Authenticated with your secret key.

The response includes `playbackId`, `status`, and timing fields for each stream, but **does not include `rtmpUrl` or `streamKey`** — those credentials are only returned on the [Create Live Stream](/api/live-streams/create) response.

## Query parameters

<ParamField query="limit" type="integer" default="20">
  Maximum number of streams to return. Range 1–100.
</ParamField>

<ParamField query="status" type="string">
  Filter by lifecycle state. One of `idle`, `active`, or `ended`. Omit to return streams in any state.
</ParamField>

<RequestExample>
```bash cURL
curl "https://api.shortkit.dev/v1/live-streams?status=active&limit=20" \
  -H "Authorization: Bearer sk_live_your_secret_key"
```

```python Python
import requests

resp = requests.get(
    "https://api.shortkit.dev/v1/live-streams",
    params={"status": "active", "limit": 20},
    headers={"Authorization": "Bearer sk_live_your_secret_key"},
)
streams = resp.json()["data"]
```

```javascript Node.js
const params = new URLSearchParams({ status: "active", limit: "20" });
const resp = await fetch(
  `https://api.shortkit.dev/v1/live-streams?${params}`,
  { headers: { Authorization: "Bearer sk_live_your_secret_key" } }
);
const { data: streams } = await resp.json();
```
</RequestExample>
