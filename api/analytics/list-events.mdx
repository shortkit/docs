---
title: "List Events"
api: "GET /v1/events/explore"
description: "Query raw events with filtering by type, content, user, and time range."
---

Returns a paginated list of raw events. Use filters to narrow results by event type, content, user, or time range.

## Query parameters

<ParamField query="event_type" type="string">
  Filter by event type (e.g., `playStart`, `completion`, `like`).
</ParamField>

<ParamField query="content_id" type="string">
  Filter by content ID (UUID format).
</ParamField>

<ParamField query="viewer_id" type="string">
  Filter by user ID.
</ParamField>

<ParamField query="limit" type="integer" default={50}>
  Items per page. Min 1, max 200.
</ParamField>

<ParamField query="page" type="integer" default={1}>
  Page number (1-indexed).
</ParamField>

<ParamField query="timeframe[]" type="string" default="7:days">
  Time range filter. Format: `value:unit` (e.g., `7:days`, `24:hours`). Defaults to the last 7 days.
</ParamField>

<RequestExample>
```bash cURL
curl "https://api.shortkit.dev/v1/events/explore?event_type=playStart&content_id=d290f1ee-6c54-4b01-90e6-d701748f0851&limit=20" \
  -H "Authorization: Bearer sk_live_your_secret_key"
```

## Pagination

Uses page-based pagination. Pass `page` to navigate through results. The response includes `pagination.total`, `pagination.total_pages`, and the current `pagination.page`. Results are ordered by timestamp descending (newest first).

## Filtering by time range

Use the `timeframe[]` query parameter to specify a time window:

```bash
curl "https://api.shortkit.dev/v1/events/explore?event_type=completion&timeframe[]=24:hours" \
  -H "Authorization: Bearer sk_live_your_secret_key"
```

When omitted, defaults to the last 7 days.
