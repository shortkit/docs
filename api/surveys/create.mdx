---
title: "Create Survey"
api: "POST /v1/surveys"
description: "Create a new in-feed survey with answer options and placement rules."
---

## Request body

<ParamField body="title" type="string" required>
  Internal title (max 200 characters).
</ParamField>

<ParamField body="question" type="string" required>
  Question text shown to end users (max 500 characters).
</ParamField>

<ParamField body="status" type="string" default="draft">
  Initial status: `draft`, `active`, `paused`, or `archived`.
</ParamField>

<ParamField body="priority" type="integer" default={0}>
  Display priority. Higher values take precedence when multiple surveys compete.
</ParamField>

<ParamField body="autoAdvanceDelay" type="number">
  Seconds before auto-advancing after a response. Omit or `null` to disable.
</ParamField>

<ParamField body="options" type="object[]" required>
  Answer choices. Each must include `text` (string, max 200 chars) and `position` (integer).
</ParamField>

<ParamField body="placementRules" type="object[]" required>
  Placement rules. Each must include `ruleType` and `ruleValue`. Must contain at least one `min_videos_before` rule with a non-negative integer `n`.
</ParamField>

## Validation errors

| Condition | Status | Message |
|-----------|--------|---------|
| Missing `min_videos_before` rule | 400 | `placement_rules must include a min_videos_before rule` |
| `min_videos_before.n` is not a non-negative integer | 400 | `min_videos_before rule_value.n must be a non-negative integer` |
| Unknown `ruleType` | 400 | `Invalid rule_type: {value}` |
| Unknown `status` | 400 | `Invalid status: {value}` |
