---
title: "Submit Response"
api: "POST /v1/surveys/{id}/responses"
description: "Submit a user's response to a survey. Uses publishable key auth — called by the SDK automatically."
---

This endpoint is called by the SDK when a user answers a survey. It uses **publishable key** authentication, making it safe for client-side use.

## Path parameters

<ParamField path="id" type="string" required>
  Survey UUID.
</ParamField>

## Request body

<ParamField body="optionId" type="string" required>
  UUID of the selected answer option.
</ParamField>

<ParamField body="userId" type="string" required>
  Identifier for the responding user.
</ParamField>

<ParamField body="sessionId" type="string">
  Current session identifier.
</ParamField>

<ParamField body="deviceInfo" type="object">
  Optional device metadata (OS, app version, etc.).
</ParamField>

## Errors

| Condition | Status | Description |
|-----------|--------|-------------|
| Survey not found | 404 | The survey does not exist or does not belong to your organization. |
| Survey not active | 400 | Only surveys with status `active` accept responses. |
| Option not in survey | 400 | The `optionId` does not belong to this survey. |
| User already responded | 409 | Each user can only respond once per survey. |
