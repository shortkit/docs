The Views API provides access to individual video view records collected by the SDK's `viewEnd` event. Each view record captures a complete playback session with engagement metrics, quality-of-service data, and device context. Use these endpoints to build custom dashboards, investigate individual viewer behavior, and export raw data for external analysis.

## Endpoints

| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| `GET` | `/v1/views` | Secret key or Portal JWT | List video views with filtering and pagination |
| `GET` | `/v1/views/{view_id}` | Secret key or Portal JWT | Get a single view by ID |
| `GET` | `/v1/exports/views` | Secret key or Portal JWT | Export raw view data as JSON or CSV |
| `GET` | `/v1/dimensions/{dimension}/values` | Secret key or Portal JWT | List unique values for a dimension |

## Common query parameters

The views and export endpoints share two array-style query parameters for time ranges and dimension filtering.

### timeframe[]

Controls the time window for the query. Accepts two formats:

**Relative** -- a single value of `N:unit`:

```
?timeframe[]=7:days
?timeframe[]=24:hours
```

**Absolute** -- two Unix timestamps (start and end):

```
?timeframe[]=1709251200&timeframe[]=1709856000
```

Defaults to `7:days` when omitted.

### filters[]

Narrow results by dimension values. Each filter is a `key:value` pair. Multiple filters are ANDed together.

```
?filters[]=os_name:iOS&filters[]=device_category:phone
```

Valid filter keys match the set of available dimensions. See the [dimensions endpoint](#list-dimension-values) for the full list and how to retrieve filterable values.

## List views

List individual video view records with filtering, sorting, and pagination.

```bash
GET /v1/views
```

### Parameters

<ParamField query="viewer_id" type="string">
  Filter to views from a specific viewer.
</ParamField>

<ParamField query="content_id" type="string">
  Filter to views of a specific content item (UUID).
</ParamField>

<ParamField query="order_by" type="string" default="view_start">
  Sort field. One of: `view_start`, `watch_time_ms`, `playing_time_ms`.
</ParamField>

<ParamField query="order_direction" type="string" default="desc">
  Sort direction: `asc` or `desc`.
</ParamField>

<ParamField query="limit" type="integer" default={25}>
  Items per page (1-100).
</ParamField>

<ParamField query="page" type="integer" default={1}>
  Page number (1-indexed).
</ParamField>

<ParamField query="timeframe[]" type="string" default="7:days">
  Time range. See [common query parameters](#common-query-parameters).
</ParamField>

<ParamField query="filters[]" type="string">
  Dimension filters. See [common query parameters](#common-query-parameters).
</ParamField>

### Example request

```bash
curl "https://api.shortkit.dev/v1/views?viewer_id=user_456&order_by=watch_time_ms&order_direction=desc&limit=10&timeframe[]=30:days" \
  -H "Authorization: Bearer sk_live_your_secret_key"
```

### Pagination

Use `page` and `limit` for offset-based pagination. The response includes `total_row_count` so you can compute the total number of pages.

```bash
# Page 1
GET /v1/views?limit=25&page=1

# Page 2
GET /v1/views?limit=25&page=2
```

### Combining filters

All parameters compose. This query fetches the 10 longest iOS views for a specific video in the last 30 days:

```bash
curl "https://api.shortkit.dev/v1/views?content_id=a1b2c3d4-e5f6-7890-abcd-ef1234567890&order_by=watch_time_ms&order_direction=desc&limit=10&timeframe[]=30:days&filters[]=os_name:iOS" \
  -H "Authorization: Bearer sk_live_your_secret_key"
```

## Export views

Download raw view data as a JSON array or CSV file. The response is returned as a file attachment -- no async job is needed.

```bash
GET /v1/exports/views
```

### Parameters

<ParamField query="format" type="string" default="json">
  Output format: `json` or `csv`.
</ParamField>

<ParamField query="timeframe[]" type="string" default="7:days">
  Time range. See [common query parameters](#common-query-parameters).
</ParamField>

<ParamField query="filters[]" type="string">
  Dimension filters. See [common query parameters](#common-query-parameters).
</ParamField>

### Limits

| Limit | Value |
|-------|-------|
| Max rows per export | 100,000 |
| Sort order | `view_start` descending (most recent first) |

### Example request (JSON)

```bash
curl -OJ "https://api.shortkit.dev/v1/exports/views?format=json&timeframe[]=30:days&filters[]=os_name:iOS" \
  -H "Authorization: Bearer sk_live_your_secret_key"
```

The `-OJ` flags tell curl to save the file using the server-provided filename (`views_export.json` or `views_export.csv`).

### JSON response

The response is a JSON array of view objects with `Content-Disposition: attachment; filename=views_export.json`:

```json
[
  {
    "id": "d290f1ee-6c54-4b01-90e6-d701748f0851",
    "content_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "viewer_id": "user_456",
    "view_start": "2026-03-01T14:30:00Z",
    "watch_time_ms": 22000,
    "completed": true,
    "os_name": "iOS",
    "device_model": "iPhone15,3"
  }
]
```

<Note>
  The JSON export returns a flat array of view objects, not the standard response envelope. Each object has the same fields as the [view object](#view-object).
</Note>

### CSV response

The CSV export returns a file with `Content-Disposition: attachment; filename=views_export.csv`. Columns are a fixed subset of view fields:

```
id, content_id, viewer_id, view_session_id, view_start, view_end,
watch_time_ms, playing_time_ms, completed, looped, loop_count,
exit_before_start, video_startup_time_ms, rebuffer_count,
rebuffer_duration_ms, seek_count, seek_duration_ms, avg_bitrate_bps,
rendition_change_count, os_name, os_version, device_category,
device_model, connection_type, video_title, video_duration_ms,
client_app_name, custom_1, custom_2, custom_3, custom_4, custom_5
```

### Example request (CSV)

```bash
curl -OJ "https://api.shortkit.dev/v1/exports/views?format=csv&timeframe[]=7:days" \
  -H "Authorization: Bearer sk_live_your_secret_key"
```

### Loading into a data warehouse

```python
import pandas as pd

df = pd.read_csv("views_export.csv")
df["view_start"] = pd.to_datetime(df["view_start"])
df.to_parquet("views.parquet")  # upload to S3/GCS for BigQuery, Redshift, etc.
```

## List dimension values

Returns the unique values for a given dimension, ranked by view count. Use this to populate filter dropdowns in a custom analytics UI.

```bash
GET /v1/dimensions/{dimension}/values
```

<ParamField path="dimension" type="string" required>
  The dimension to query. See [valid dimensions](#valid-dimensions) below.
</ParamField>

<ParamField query="limit" type="integer" default={50}>
  Maximum number of values to return (1-500).
</ParamField>

<ParamField query="timeframe[]" type="string" default="7:days">
  Time range. See [common query parameters](#common-query-parameters).
</ParamField>

### Example request

```bash
curl "https://api.shortkit.dev/v1/dimensions/os_name/values?limit=10&timeframe[]=30:days" \
  -H "Authorization: Bearer sk_live_your_secret_key"
```

### Response

```json
{
  "data": [
    { "value": "iOS", "total_views": 3500 },
    { "value": "Android", "total_views": 1200 },
    { "value": "macOS", "total_views": 85 }
  ],
  "timeframe": [1708905600, 1711497600],
  "meta": {
    "request_id": "req_abc123def456"
  }
}
```

<ResponseField name="data[].value" type="string">
  The dimension value (e.g., `"iOS"`, `"phone"`, `"wifi"`). Null values are excluded from results.
</ResponseField>

<ResponseField name="data[].total_views" type="integer">
  Number of views with this dimension value in the given timeframe.
</ResponseField>

### Errors

| Status | Code | Cause |
|--------|------|-------|
| 400 | `invalid_request` | Dimension name is not in the valid set |

### Valid dimensions

| Dimension | Example values |
|-----------|---------------|
| `content_id` | UUID of the content item |
| `video_title` | `"Product Launch Video"` |
| `device_category` | `phone`, `tablet`, `desktop`, `tv` |
| `device_manufacturer` | `Apple`, `Samsung`, `Google` |
| `device_model` | `iPhone15,3`, `SM-S918B` |
| `os_name` | `iOS`, `Android`, `macOS` |
| `os_version` | `18.2`, `14` |
| `connection_type` | `wifi`, `cellular`, `wired` |
| `country` | `US`, `GB`, `DE` |
| `continent_code` | `NA`, `EU`, `AS` |

| `city` | `San Francisco`, `New York` |

| `client_app_name` | `MyApp` |

| `custom_1` through `custom_5` | Custom dimension values set by your application |

| `completed` | `true`, `false` |
| `exit_before_start` | `true`, `false` |
| `looped` | `true`, `false` |

### Building a filter UI

Query multiple dimensions to populate a set of filter dropdowns, then pass the user's selections back as `filters[]` on the list views or export endpoint:

```bash
# 1. Fetch available OS values
GET /v1/dimensions/os_name/values?timeframe[]=30:days

# 2. Fetch available device categories
GET /v1/dimensions/device_category/values?timeframe[]=30:days

# 3. User selects "iOS" and "phone" -- query filtered views
GET /v1/views?filters[]=os_name:iOS&filters[]=device_category:phone&timeframe[]=30:days
```
