The Ad Configuration API controls how ad slots are inserted into your feed responses. When ads are enabled, the server automatically interleaves `ad_slot` items between your content based on these settings. The SDK and web embed receive ad slots as part of the feed and handle rendering.

## Endpoints

| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| `GET` | `/v1/ad-config` | Secret key | Get current ad configuration |
| `PUT` | `/v1/ad-config` | Secret key | Update ad configuration |

Both endpoints require a **secret key** (`Authorization: Bearer sk_...`). Portal JWTs are not accepted.

---

## Ad config object

```json
{
  "enabled": false,
  "frequency": 5,
  "firstAdOffset": 3,
  "maxPerPage": 3,
  "adTagUrl": null,
  "adUnitId": null,
  "skipPolicy": "skippable_after",
  "skipDelaySeconds": 5,
  "defaultTargeting": {},
  "nativeEnabled": false,
  "nativeFormatId": null,
  "nativeAdUnitId": null,
  "nativeVideoUrlKey": "video_url",
  "nativeHeadlineKey": "headline",
  "nativeCtaKey": "cta_text",
  "nativeSponsorKey": "sponsor_name",
  "nativeSponsorLogoKey": "sponsor_logo"
}
```

### Field reference

#### Placement fields

<ResponseField name="enabled" type="boolean" default={false}>
  Master switch. When `false`, no ad slots are injected into the feed.
</ResponseField>

<ResponseField name="frequency" type="integer" default={5}>
  Number of content items between consecutive ad slots after the first ad. For example, `5` means an ad appears every 5 content items.
</ResponseField>

<ResponseField name="firstAdOffset" type="integer" default={3}>
  Position of the first ad slot, counted in content items. A value of `3` means the first ad appears after the third content item.
</ResponseField>

<ResponseField name="maxPerPage" type="integer" default={3}>
  Maximum number of ad slots injected per feed page. Once this limit is reached, no more ads are added to that page regardless of remaining content.
</ResponseField>

#### Ad source fields

<ResponseField name="adTagUrl" type="string | null" default="null">
  VAST/VMAP ad tag URL. Passed through to the SDK in each ad slot so the client-side ad player can request creatives from your ad server.
</ResponseField>

<ResponseField name="adUnitId" type="string | null" default="null">
  Ad unit identifier from your ad platform (e.g., Google Ad Manager, AppLovin). Passed through in each ad slot for client-side ad requests. When `nativeAdUnitId` is set, native ad slots use that value instead.
</ResponseField>

#### Skip behavior fields

<ResponseField name="skipPolicy" type="string" default="skippable_after">
  Controls whether users can skip ads. Passed through to the SDK in each ad slot. The SDK uses this value to render skip UI. Value is an opaque string -- set it to whatever your ad rendering layer expects (e.g., `"skippable_after"`, `"non_skippable"`, `"always_skippable"`).
</ResponseField>

<ResponseField name="skipDelaySeconds" type="integer" default={5}>
  Seconds the user must watch before a skip button appears. Only relevant when `skipPolicy` allows skipping.
</ResponseField>

#### Targeting fields

<ResponseField name="defaultTargeting" type="object" default="{}">
  Key-value pairs passed through in every ad slot's `targeting` field. Use this to send default targeting signals (e.g., content category, audience segment) to your ad server.
</ResponseField>

#### Native ad fields

These fields configure native ad rendering. When `nativeEnabled` is `true`, ad slots include `adFormat: "native"` and a `nativeAssetKeys` object that maps your native ad platform's response keys to the fields the SDK expects.

<ResponseField name="nativeEnabled" type="boolean" default={false}>
  When `true`, ad slots are served as native ads instead of VAST video ads. The ad slot includes `adFormat`, `nativeFormatId`, and `nativeAssetKeys` fields.
</ResponseField>

<ResponseField name="nativeFormatId" type="string | null" default="null">
  Identifier for your native ad format template, passed through in each native ad slot.
</ResponseField>

<ResponseField name="nativeAdUnitId" type="string | null" default="null">
  Ad unit ID used specifically for native ad slots. When set, native ad slots use this instead of the top-level `adUnitId`.
</ResponseField>

<ResponseField name="nativeVideoUrlKey" type="string" default="video_url">
  Key name in your native ad response that contains the video creative URL.
</ResponseField>

<ResponseField name="nativeHeadlineKey" type="string" default="headline">
  Key name for the ad headline text.
</ResponseField>

<ResponseField name="nativeCtaKey" type="string" default="cta_text">
  Key name for the call-to-action button text.
</ResponseField>

<ResponseField name="nativeSponsorKey" type="string" default="sponsor_name">
  Key name for the sponsor/advertiser name.
</ResponseField>

<ResponseField name="nativeSponsorLogoKey" type="string" default="sponsor_logo">
  Key name for the sponsor logo image URL.
</ResponseField>

## Get ad config

Returns the current ad configuration for your organization. If no configuration has been set, default values are returned.

```bash
GET /v1/ad-config
```

### Authentication

Requires a **secret key** via the `Authorization` header. Portal JWTs are not accepted for this endpoint.

### Example request

```bash
curl https://api.shortkit.dev/v1/ad-config \
  -H "Authorization: Bearer sk_live_your_secret_key"
```

### Response

```json
{
  "data": {
    "enabled": false,
    "frequency": 5,
    "firstAdOffset": 3,
    "maxPerPage": 3,
    "adTagUrl": null,
    "adUnitId": null,
    "skipPolicy": "skippable_after",
    "skipDelaySeconds": 5,
    "defaultTargeting": {},
    "nativeEnabled": false,
    "nativeFormatId": null,
    "nativeAdUnitId": null,
    "nativeVideoUrlKey": "video_url",
    "nativeHeadlineKey": "headline",
    "nativeCtaKey": "cta_text",
    "nativeSponsorKey": "sponsor_name",
    "nativeSponsorLogoKey": "sponsor_logo"

  "meta": {
    "request_id": "req_abc123"
  }
}
```

## Update ad config

Updates ad configuration settings. This is a **sparse update** -- only the fields you include in the request body are changed. Omitted fields retain their current values.

```bash
PUT /v1/ad-config
```

### Authentication

Requires a **secret key** via the `Authorization` header. Portal JWTs are not accepted for this endpoint.

### Request body

All fields are optional. Include only the fields you want to change.

<ParamField body="enabled" type="boolean">
  Enable or disable ad slot injection.
</ParamField>

<ParamField body="frequency" type="integer">
  Content items between ads after the first ad.
</ParamField>

<ParamField body="firstAdOffset" type="integer">
  Content position of the first ad.
</ParamField>

<ParamField body="maxPerPage" type="integer">
  Maximum ad slots per feed page.
</ParamField>

<ParamField body="adTagUrl" type="string | null">
  VAST/VMAP ad tag URL.
</ParamField>

<ParamField body="adUnitId" type="string | null">
  Default ad unit identifier.
</ParamField>

<ParamField body="skipPolicy" type="string">
  Skip behavior string passed to the SDK.
</ParamField>

<ParamField body="skipDelaySeconds" type="integer">
  Seconds before the skip button appears.
</ParamField>

<ParamField body="defaultTargeting" type="object">
  Default targeting key-value pairs.
</ParamField>

<ParamField body="nativeEnabled" type="boolean">
  Enable native ad format.
</ParamField>

<ParamField body="nativeFormatId" type="string | null">
  Native ad format template identifier.
</ParamField>

<ParamField body="nativeAdUnitId" type="string | null">
  Ad unit ID for native ad slots.
</ParamField>

<ParamField body="nativeVideoUrlKey" type="string">
  Asset key for video URL in native ad responses.
</ParamField>

<ParamField body="nativeHeadlineKey" type="string">
  Asset key for headline text.
</ParamField>

<ParamField body="nativeCtaKey" type="string">
  Asset key for CTA text.
</ParamField>

<ParamField body="nativeSponsorKey" type="string">
  Asset key for sponsor name.
</ParamField>

<ParamField body="nativeSponsorLogoKey" type="string">
  Asset key for sponsor logo URL.
</ParamField>

### Example: enable ads with VAST

```bash
curl -X PUT https://api.shortkit.dev/v1/ad-config \
  -H "Authorization: Bearer sk_live_your_secret_key" \
  -H "Content-Type: application/json" \
  -d '{
    "enabled": true,
    "adTagUrl": "https://ads.example.com/vast?placement=shortkit",
    "frequency": 4,
    "firstAdOffset": 2,
    "maxPerPage": 5,
    "skipPolicy": "skippable_after",
    "skipDelaySeconds": 3,
    "defaultTargeting": {
      "content_type": "news",
      "audience": "general"
    }
  }'
```

### Example: enable native ads

```bash
curl -X PUT https://api.shortkit.dev/v1/ad-config \
  -H "Authorization: Bearer sk_live_your_secret_key" \
  -H "Content-Type: application/json" \
  -d '{
    "nativeEnabled": true,
    "nativeFormatId": "vertical_video_card",
    "nativeAdUnitId": "/12345/native-feed",
    "nativeVideoUrlKey": "media_url",
    "nativeHeadlineKey": "title",
    "nativeCtaKey": "call_to_action",
    "nativeSponsorKey": "advertiser_name",
    "nativeSponsorLogoKey": "icon_url"
  }'
```

### Example: disable ads

```bash
curl -X PUT https://api.shortkit.dev/v1/ad-config \
  -H "Authorization: Bearer sk_live_your_secret_key" \
  -H "Content-Type: application/json" \
  -d '{ "enabled": false }'
```

## How ad config affects the feed

When `enabled` is `true`, the server interleaves `ad_slot` items into every feed response (`GET /v1/feed` and `GET /v1/feed/videos`). Ad slots are inserted **after** content ordering and pagination cursor computation, so they do not affect pagination stability.

### Placement algorithm

The server walks through the content items in the feed page and inserts ad slots using three rules:

1. **First ad position** -- The first ad slot is inserted after the content item at position `firstAdOffset`.
2. **Recurring frequency** -- After the first ad, another ad slot is inserted every `frequency` content items.
3. **Page cap** -- No more than `maxPerPage` ad slots are inserted per page.

Given `firstAdOffset: 3`, `frequency: 5`, and `maxPerPage: 3` with a page of 20 content items, the ad slots land at these positions:

```
Content 1
Content 2
Content 3
  -- Ad slot 1 (firstAdOffset reached) --
Content 4
Content 5
Content 6
Content 7
Content 8
  -- Ad slot 2 (5 items after first ad) --
Content 9
Content 10
Content 11
Content 12
Content 13
  -- Ad slot 3 (5 more items, maxPerPage reached) --
Content 14
Content 15
... (no more ads on this page)
```

### Ad slot shape in the feed

Each ad slot in the feed response contains all the fields needed for the client to request and render an ad:

```json
{
  "type": "ad_slot",
  "id": "ad_slot_p0_3_0",
  "position": 3,
  "adTagUrl": "https://ads.example.com/vast?placement=shortkit",
  "adUnitId": null,
  "targeting": { "content_type": "news" },
  "skipPolicy": "skippable_after",
  "skipDelaySeconds": 3,
  "adFormat": null,
  "nativeFormatId": null,
  "nativeAssetKeys": null
}
```

When `nativeEnabled` is `true`, the ad slot includes native-specific fields:

```json
{
  "type": "ad_slot",
  "id": "ad_slot_p0_3_0",
  "position": 3,
  "adTagUrl": null,
  "adUnitId": "/12345/native-feed",
  "targeting": {},
  "skipPolicy": "skippable_after",
  "skipDelaySeconds": 5,
  "adFormat": "native",
  "nativeFormatId": "vertical_video_card",
  "nativeAssetKeys": {
    "videoUrl": "media_url",
    "headline": "title",
    "ctaText": "call_to_action",
    "sponsorName": "advertiser_name",
    "sponsorLogo": "icon_url"
  }
}
```

The `nativeAssetKeys` object tells the SDK which keys to extract from the native ad response returned by your ad platform. The SDK does not fetch ads server-side -- it passes these values to the client-side ad rendering layer.

### What happens when ads are disabled

When `enabled` is `false`, the feed contains only content, survey, and carousel items. No `ad_slot` items appear in the response. You can safely update other ad config fields while ads are disabled; they take effect the next time you set `enabled` to `true`.
