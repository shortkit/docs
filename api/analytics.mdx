The Analytics API provides access to aggregated metrics derived from the events collected by the [Events API](/api/events). Use it to measure content performance, user engagement, and playback quality.

## Endpoints

| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| `GET` | `/v1/analytics/overview` | Secret key | High-level metrics summary |
| `GET` | `/v1/analytics/content` | Secret key | Content-level metrics |
| `GET` | `/v1/analytics/content/{id}` | Secret key | Single content metrics |
| `GET` | `/v1/analytics/engagement` | Secret key | Engagement metrics |
| `GET` | `/v1/analytics/quality` | Secret key | Playback quality metrics |

## Common query parameters

All analytics endpoints accept these parameters for filtering and time-series granularity:

<ParamField query="startDate" type="string" required>
  Start date (ISO 8601: `YYYY-MM-DD`).
</ParamField>

<ParamField query="endDate" type="string" required>
  End date (ISO 8601: `YYYY-MM-DD`).
</ParamField>

<ParamField query="granularity" type="string" default="day">
  Time granularity for timeseries data: `hour`, `day`, `week`.
</ParamField>

## Metrics taxonomy

Analytics are derived from the event types collected by the SDK. Here is how events map to metrics:

| Metric | Derived from event type | Description |
|--------|------------------------|-------------|
| Impressions | `impression` | Number of times content appeared in the viewport |
| Plays | `playStart` | Number of playback starts |
| Completions | `completion` | Number of times a video was watched to the end |
| Completion rate | `completion` / `playStart` | Fraction of plays that reached the end |
| Average watch time | `watchProgress`, `completion` | Mean seconds watched per play |
| Likes | `like` | Number of like actions |
| Shares | `share` | Number of share actions |
| Saves | `save` | Number of save actions |
| Rebuffer rate | `rebuffer` / `playStart` | Fraction of plays that experienced a stall |
| Startup time | `playStart` | Mean time to first frame (ms) |
| Error rate | `error` / `playStart` | Fraction of plays that encountered an error |

## Overview metrics

Get a high-level summary of key metrics across all content.

```bash
GET /v1/analytics/overview
```

### Example request

```bash
curl "https://api.shortkit.dev/v1/analytics/overview?startDate=2026-01-01&endDate=2026-01-31" \
  -H "Authorization: Bearer sk_live_your_secret_key"
```

## Content analytics

### List content metrics

Get per-content metrics ranked by a sort field.

```bash
GET /v1/analytics/content
```

<ParamField query="limit" type="integer" default={20}>
  Items per page (1-100).
</ParamField>

<ParamField query="cursor" type="string">
  Pagination cursor from a previous response.
</ParamField>

<ParamField query="sort" type="string" default="impressions">
  Sort field: `impressions`, `plays`, `completions`, `watchTime`.
</ParamField>

<ParamField query="order" type="string" default="desc">
  Sort order: `asc`, `desc`.
</ParamField>

<ParamField query="tags" type="string">
  Filter by tags (comma-separated).
</ParamField>

### Example request

```bash
curl "https://api.shortkit.dev/v1/analytics/content?startDate=2026-01-01&endDate=2026-01-31&sort=completions&limit=10" \
  -H "Authorization: Bearer sk_live_your_secret_key"
```

### Response

```json
{
  "data": [
    {
      "contentId": "sk_content_abc123",
      "title": "Product Launch Video",
      "metrics": {
        "impressions": 45000,
        "plays": 38000,
        "completions": 18000,
        "totalWatchTime": 684000,
        "averageWatchTime": 38.0,
        "completionRate": 0.474,
        "likes": 2400,
        "shares": 850,
        "saves": 320
      }
    }
  ],
  "meta": {
    "next_cursor": "eyJvZmZzZXQiOjIwfQ==",
    "request_id": "req_def456"
  }
}
```

### Single content metrics

Get detailed metrics and timeseries for a specific content item.

```bash
GET /v1/analytics/content/{id}
```

<ParamField path="id" type="string" required>
  Content ID.
</ParamField>

### Example request

```bash
curl "https://api.shortkit.dev/v1/analytics/content/sk_content_abc123?startDate=2026-01-01&endDate=2026-01-31&granularity=day" \
  -H "Authorization: Bearer sk_live_your_secret_key"
```

## Engagement analytics

```bash
GET /v1/analytics/engagement
```

<ParamField query="segment" type="string">

