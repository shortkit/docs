ShortKit has two analytics systems. **V1** provides event-count aggregates from the `events` table. **V2** provides per-view-session metrics from the `video_views` table with flexible timeframes, dimension breakdowns, and data export. V2 is the recommended system for all new integrations.

Both systems require a secret key or portal JWT (`Authorization: Bearer sk_...` or `Authorization: Bearer <jwt>`).

## Endpoints

| Method | Endpoint | System | Description |
|--------|----------|--------|-------------|
| `GET` | `/v1/analytics/overview` | V1 | Aggregate event counts |
| `GET` | `/v1/analytics/timeseries` | V1 | Time-bucketed event counts |
| `GET` | `/v1/analytics/top-content` | V1 | Content ranked by play starts |
| `GET` | `/v1/analytics/content/{id}` | V1 | Per-content event breakdown |
| `GET` | `/v1/metrics/{metric}` | V2 | Scalar or time-grouped metric |
| `GET` | `/v1/metrics/{metric}/breakdown` | V2 | Metric broken down by dimension |
| `GET` | `/v1/metrics/{metric}/timeseries` | V2 | Auto-granularity time series |
| `GET` | `/v1/dimensions/{dimension}/values` | V2 | Unique values for a dimension |
| `GET` | `/v1/views` | V2 | List individual view records |
| `GET` | `/v1/views/{view_id}` | V2 | Single view detail |
| `GET` | `/v1/exports/views` | V2 | Export raw view data (JSON or CSV) |

## V1: Event-based analytics

V1 endpoints query the `events` table (individual `impression`, `playStart`, `completion` events). They accept a `period` parameter and return aggregate counts.

<Note>
  V1 returns **synthetic demo data** by default. When an organization's `analytics_mode` is set to `"demo"`, these endpoints return generated numbers instead of querying real events. This is the default for new organizations. V2 endpoints always query real data regardless of analytics mode.
</Note>

### Overview

```bash
GET /v1/analytics/overview
```

Returns aggregate counts for a time period.

<ParamField query="period" type="string" default="7d">
  Time period: `24h`, `7d`, `30d`, or `90d`.
</ParamField>

```bash
curl "https://api.shortkit.dev/v1/analytics/overview?period=7d" \
  -H "Authorization: Bearer sk_live_your_secret_key"
```

```json
{
  "data": {
    "period": "7d",
    "impressions": 12500,
    "playStarts": 8300,
    "completions": 4200,
    "completionRate": 50.6,
    "uniqueViewers": 3100
  },
  "meta": { "request_id": "req_abc123" }
}
```

### Timeseries

```bash
GET /v1/analytics/timeseries
```

Returns time-bucketed event counts for charting a single metric.

<ParamField query="period" type="string" default="7d">
  Time period: `24h`, `7d`, `30d`, or `90d`.
</ParamField>

<ParamField query="metric" type="string" default="impressions">
  Metric to chart: `impressions`, `playStarts`, or `completions`.
</ParamField>

```bash
curl "https://api.shortkit.dev/v1/analytics/timeseries?period=7d&metric=playStarts" \
  -H "Authorization: Bearer sk_live_your_secret_key"
```

Response is an array of `{ timestamp, count }` objects.

### Top content

```bash
GET /v1/analytics/top-content
```

Returns content items ranked by play starts.

<ParamField query="period" type="string" default="7d">
  Time period: `24h`, `7d`, `30d`, or `90d`.
</ParamField>

<ParamField query="limit" type="integer" default={10}>
  Number of items to return (1--50).
</ParamField>

### Per-content breakdown

```bash
GET /v1/analytics/content/{id}
```

<ParamField path="id" type="string" required>
  Content ID.
</ParamField>

<ParamField query="period" type="string" default="7d">
  Time period: `24h`, `7d`, `30d`, or `90d`.
</ParamField>

## V2: View-based metrics

V2 endpoints query the `video_views` table. Each row represents a single view session, created when the SDK sends a `viewEnd` event. V2 supports arbitrary timeframes, dimension filtering, and 16 metrics across engagement, quality, and behavioral categories.

### Common query parameters

All V2 endpoints accept `timeframe[]` and `filters[]` parameters.

#### Timeframe

The `timeframe[]` parameter controls the time range. It accepts two formats:

| Format | Example | Meaning |
|--------|---------|---------|
| Relative | `timeframe[]=7:days` | Last 7 days from now |
| Relative | `timeframe[]=24:hours` | Last 24 hours from now |
| Absolute | `timeframe[]=1709251200&timeframe[]=1709856000` | Two Unix timestamps (start, end) |

Default when omitted: `7:days`.

#### Filters

The `filters[]` parameter narrows results by dimension values. Pass one or more `dimension:value` pairs. Multiple filters are ANDed together.

```
filters[]=device_category:phone&filters[]=custom_1:experiment_hero_v2
```

Only dimensions from the valid set (listed below) are accepted. Invalid dimension keys are silently dropped.

## Valid metrics

### Count metrics

These metrics count view sessions matching specific conditions. No `measurement` parameter is needed.

| Metric | Description |
|--------|-------------|
| `views` | Total view sessions |
| `unique_viewers` | Distinct viewer IDs |
| `started_views` | Views where playback started (not `exit_before_start`) |
| `exits_before_video_start` | Views where the viewer left before playback began |
| `completions` | Views where the video played to the end |

### Column metrics

These metrics aggregate a column from the `video_views` table. They support `sum`, `avg`, and `count` as the `measurement` parameter.

| Metric | Default measurement | Unit | Description |
|--------|---------------------|------|-------------|
| `watch_time` | `sum` | ms | Wall-clock watch duration |
| `playing_time` | `sum` | ms | Actual media playback time (excludes buffering) |
| `video_startup_time` | `avg` | ms | Time from impression to first frame |
| `rebuffer_count` | `avg` | count | Buffering interruptions per view |
| `rebuffer_duration` | `avg` | ms | Buffering time per view |
| `seek_count` | `avg` | count | User seeks per view |
| `seek_duration` | `avg` | ms | Seek time per view |
| `avg_bitrate` | `avg` | bps | Average playback bitrate |
| `rendition_changes` | `avg` | count | ABR quality switches per view |

### Derived metrics

These are computed from other columns. They do not accept a `measurement` parameter.

| Metric | Description |
|--------|-------------|
| `rebuffer_frequency` | Rebuffers per minute of playing time |
| `rebuffer_percentage` | Rebuffer time as a percentage of watch time |

## Valid dimensions

Dimensions can be used for both `filters[]` values and `breakdown` grouping.

### Content

| Dimension | Description |
|-----------|-------------|
| `content_id` | Content UUID. Breakdowns are enriched with `content_title` and `content_thumbnail_url`. |
| `video_title` | Content title at time of view |

### Device

| Dimension | Description |
|-----------|-------------|
| `device_category` | `"phone"`, `"tablet"`, `"other"` |
| `device_manufacturer` | e.g., `"Apple"` |
| `device_model` | e.g., `"iPhone16,1"` |
| `os_name` | e.g., `"iOS"` |
| `os_version` | e.g., `"18.3"` |
| `connection_type` | `"wifi"`, `"cellular"`, etc. |

### Geographic

| Dimension | Description |
|-----------|-------------|
| `country` | Country name (from IP geolocation) |
| `continent_code` | Continent code |
| `region` | Region or state |
| `city` | City |

### Viewer

| Dimension | Description |
|-----------|-------------|
| `viewer_id` | User ID or anonymous device ID |
| `client_app_name` | App name from SDK init |
| `client_app_version` | App version from SDK init |

### Boolean

| Dimension | Description |
|-----------|-------------|
| `captions_enabled` | Whether captions were on during the view |
| `completed` | Whether the video played to the end |
| `exit_before_start` | Whether the viewer left before playback |
| `looped` | Whether the video auto-looped |

### Custom

Five arbitrary string fields set at SDK initialization. Use these for A/B test variants, user tiers, campaign sources, or any business-specific segmentation.

| Dimension | Description |
|-----------|-------------|
| `custom_1` | First custom dimension |
| `custom_2` | Second custom dimension |
| `custom_3` | Third custom dimension |
| `custom_4` | Fourth custom dimension |
| `custom_5` | Fifth custom dimension |

## Get metric

```bash
GET /v1/metrics/{metric}
```

Returns a scalar aggregate value. Add `group_by` to get a time series instead.

<ParamField path="metric" type="string" required>
  One of the valid metrics listed above.
</ParamField>

<ParamField query="measurement" type="string">
  Aggregation function: `sum`, `avg`, or `count`. Defaults depend on the metric (see table above). Ignored for count metrics and derived metrics.
</ParamField>

<ParamField query="group_by" type="string">
  Set to `hour` or `day` to return a time series instead of a scalar.
</ParamField>

<ParamField query="timeframe[]" type="string">
  Time range. Default: `7:days`.
</ParamField>

<ParamField query="filters[]" type="string">
  Dimension filters in `dimension:value` format.
</ParamField>

### Scalar response

```bash
curl "https://api.shortkit.dev/v1/metrics/views?timeframe[]=7:days" \
  -H "Authorization: Bearer sk_live_your_secret_key"
```

```json
{
  "data": { "value": 12450, "total_views": 12450 },
  "timeframe": [1740700800, 1741305600],
  "meta": { "request_id": "req_abc123" }
}
```

### Time series response (with group_by)

```bash
curl "https://api.shortkit.dev/v1/metrics/views?timeframe[]=24:hours&group_by=hour" \
  -H "Authorization: Bearer sk_live_your_secret_key"
```

```json
{
  "data": [
    { "date": "2026-02-28T00:00:00Z", "value": 150 },
    { "date": "2026-02-28T01:00:00Z", "value": 132 }
  ],
  "timeframe": [1740700800, 1740787200],
  "meta": { "request_id": "req_def456" }
}
```

## Get metric breakdown

```bash
GET /v1/metrics/{metric}/breakdown
```

Breaks down a metric by a dimension. Returns the top N dimension values ranked by metric value or view count.

<ParamField path="metric" type="string" required>
  One of the valid metrics listed above.
</ParamField>

<ParamField query="dimension" type="string" required>
  Dimension to group by. One of the valid dimensions listed above.
</ParamField>

<ParamField query="measurement" type="string">
  Aggregation function: `sum`, `avg`, or `count`.
</ParamField>

<ParamField query="limit" type="integer" default={10}>
  Maximum rows to return (1--100).
</ParamField>

<ParamField query="order_by" type="string" default="value">
  Sort order: `value` (metric value descending) or `views` (view count descending).
</ParamField>

<ParamField query="timeframe[]" type="string">
  Time range. Default: `7:days`.
</ParamField>

<ParamField query="filters[]" type="string">
  Dimension filters in `dimension:value` format.
</ParamField>

### Example: views by device category

```bash
curl "https://api.shortkit.dev/v1/metrics/views/breakdown?dimension=device_category&timeframe[]=7:days" \
  -H "Authorization: Bearer sk_live_your_secret_key"
```

```json
{
  "data": [
    { "dimension_value": "phone", "metric_value": 9500, "total_views": 9500 },
    { "dimension_value": "tablet", "metric_value": 2200, "total_views": 2200 },
    { "dimension_value": "other", "metric_value": 750, "total_views": 750 }
  ],
  "timeframe": [1740700800, 1741305600],
  "meta": { "request_id": "req_bd001" }
}
```

### Example: top content by watch time

```bash
curl "https://api.shortkit.dev/v1/metrics/watch_time/breakdown?dimension=content_id&measurement=sum&limit=10&order_by=value&timeframe[]=7:days" \
  -H "Authorization: Bearer sk_live_your_secret_key"
```

When `dimension=content_id`, each row is enriched with `content_title` and `content_thumbnail_url`.

```json
{
  "data": [
    {
      "dimension_value": "abc-123-uuid",
      "metric_value": 8450000,
      "total_views": 4200,
      "content_title": "Product Launch Recap",
      "content_thumbnail_url": "https://image.shortkit.dev/abc123/thumbnail.jpg?time=0"
    }
  ],
  "timeframe": [1740700800, 1741305600],
  "meta": { "request_id": "req_bd002" }
}
```

<ParamField path="metric" type="string" required>

  Time range. Default: `7:days`.
</ParamField>

<ParamField query="filters[]" type="string">

### Example: daily views over 30 days for a specific video

curl "https://api.shortkit.dev/v1/metrics/views/timeseries?timeframe[]=30:days&filters[]=content_id:abc-123-uuid" \

## Export views

```bash
GET /v1/exports/views
```

Exports raw view data as JSON or CSV. The response is a file download.

<ParamField query="format" type="string" default="json">
  Output format: `json` or `csv`.
</ParamField>

<ParamField query="timeframe[]" type="string">
  Time range. Default: `7:days`.
</ParamField>

<ParamField query="filters[]" type="string">
  Dimension filters in `dimension:value` format.
</ParamField>

```bash
curl "https://api.shortkit.dev/v1/exports/views?format=csv&timeframe[]=30:days&filters[]=device_category:phone" \
  -H "Authorization: Bearer sk_live_your_secret_key" \
  -o views.csv
```

Maximum **100,000 rows** per export. Results are ordered by `view_start` descending. For larger datasets, use absolute timeframe windows to paginate through the data in chunks.

## Query examples

**Total views this week:**

```
GET /v1/metrics/views?timeframe[]=7:days
```

**Average startup time by connection type:**

```
GET /v1/metrics/video_startup_time/breakdown?dimension=connection_type&measurement=avg&timeframe[]=7:days
```

**Completions for premium users over 30 days:**

```
GET /v1/metrics/completions?timeframe[]=30:days&filters[]=custom_3:premium
```

**A/B test comparison by custom dimension:**

```
GET /v1/metrics/views/breakdown?dimension=custom_1&timeframe[]=7:days
```

**Hourly view trend for the last 24 hours:**

```
GET /v1/metrics/views?timeframe[]=24:hours&group_by=hour
```

**Top 10 content by total watch time:**

```
GET /v1/metrics/watch_time/breakdown?dimension=content_id&measurement=sum&limit=10&order_by=value&timeframe[]=7:days
```

**Populate a device filter dropdown:**

```
GET /v1/dimensions/device_category/values?timeframe[]=30:days
```
