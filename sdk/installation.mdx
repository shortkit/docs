Get ShortKit installed and verified in your app. Choose your platform below.

## Requirements

| Platform | Minimum requirements |
|---|---|
| **iOS** | iOS 16.0+, Xcode 15+, Swift 5.9+ |
| **Android** | Android API 24+ (Android 7.0), Kotlin 1.9+, Android Studio Hedgehog+ |
| **React Native** | React Native 0.76+, iOS 16.0+, Android API 24+ |
| **Expo** | Expo SDK 52+, React Native 0.76+, iOS 16.0+, Android API 24+ |
| **Web** | Chrome 80+, Safari 14+, Firefox 78+, Edge 80+, Node.js 16+ (if using npm) |

## Package structure

<Tabs>
  <Tab title="Swift">
    The SDK ships as a pre-compiled binary XCFramework via Swift Package Manager:

    | Product | Purpose | External dependencies |
    |---|---|---|
    | `ShortKitSDK` | Core SDK: feed, player, overlays, analytics, captions | **None.** Uses only Foundation, UIKit, AVFoundation, and SwiftUI. |

    Import the module in your code with `import ShortKitSDK`.
  </Tab>
  <Tab title="Kotlin">
    The SDK ships two Maven artifacts:

    | Artifact | Purpose | Key dependencies |
    |---|---|---|
    | `dev.shortkit:shortkit` | Core SDK: feed, player, overlays, analytics, captions | Media3 (ExoPlayer), OkHttp, AndroidX Fragment, ViewPager2 |
    | `dev.shortkit:shortkit-google-ads` | Google ad adapters (`GoogleAdsProvider`) | Google Play Services Ads >= 23.6.0, Google IMA SDK >= 3.35.1 |

    Most integrations only need `dev.shortkit:shortkit`. Add `dev.shortkit:shortkit-google-ads` if you want to monetize with Google IMA VAST or Google Ad Manager native ads. Transitive dependencies are resolved automatically by Gradle.
  </Tab>
  <Tab title="React Native">
    The SDK ships as a single npm package that includes native modules for both platforms:

    | Package | Purpose | Key dependencies |
    |---|---|---|
    | `@shortkitsdk/react-native` | Core SDK: feed, player, overlays, analytics, captions | Native iOS and Android SDKs (bundled), React Native 0.76+ |

    The package provides the following exports:

    | Export | Type | Purpose |
    |---|---|---|
    | `ShortKitProvider` | Component | Wraps the feed and supplies API key, embed ID, and configuration |
    | `ShortKitFeed` | Component | Renders the video feed (must be inside a `ShortKitProvider`) |
    | `useShortKitPlayer()` | Hook | Access player state and controls (play, pause, seek, captions, etc.) |
    | `useShortKit()` | Hook | Access SDK-level methods (`setUserId`, `clearUserId`) |

    Import from `@shortkitsdk/react-native` in your code.
  </Tab>
</Tabs>

## Package installation

<Tabs>
  <Tab title="Swift">
    ### Option A: Xcode UI

    <Steps>
      <Step title="Open your project in Xcode">
        Go to **File** then **Add Package Dependencies**.
      </Step>
      <Step title="Enter the repository URL">
        Paste the following URL into the search field:

        ```
        https://github.com/shortkit/shortkit-ios
        ```
      </Step>
      <Step title="Select your version rule">
        Choose **Up to Next Major Version** and set the minimum to `0.1.0`. Click **Add Package**.
      </Step>
      <Step title="Add to your target">
        Select the `ShortKitSDK` library product and add it to your app target. If you need Google ad support, also select `ShortKitGoogleAds`.
      </Step>
    </Steps>

    ### Option B: Package.swift

    Add ShortKit to your dependencies array:

    ```swift Package.swift
    dependencies: [
        .package(url: "https://github.com/shortkit/shortkit-ios.git", from: "0.1.0")
    ]
    ```

    Then add the products you need to your target:

    ```swift Package.swift
    targets: [
        .target(
            name: "YourApp",
            dependencies: [
                .product(name: "ShortKitSDK", package: "shortkit-ios"),
                // Optional: add only if you need Google ad support
                .product(name: "ShortKitGoogleAds", package: "shortkit-ios")
            ]
        )
    ]
    ```
  </Tab>
  <Tab title="Kotlin">
    ### Gradle

    Add the dependency to your app-level `build.gradle.kts`:

    ```kotlin build.gradle.kts
    dependencies {
        implementation("dev.shortkit:shortkit:0.1.0")
    }
    ```

    Sync your project. In Android Studio, click **Sync Now** when prompted, or go to **File** then **Sync Project with Gradle Files**.

    No custom repository URL is needed — ShortKit is published to Maven Central, which is included by default in Android projects.
  </Tab>
  <Tab title="React Native">
    Install the package with your preferred package manager:

    <CodeGroup>

    ```bash npm
    npm install @shortkitsdk/react-native
    ```

    ```bash yarn
    yarn add @shortkitsdk/react-native
    ```

    </CodeGroup>

    Then install the iOS native dependencies:

    ```bash
    cd ios && pod install
    ```
  </Tab>
  <Tab title="Expo">
    Install the package using the Expo CLI:

    ```bash
    npx expo install @shortkitsdk/react-native
    ```

    ### Add the config plugin

    Add `@shortkitsdk/react-native` to the `plugins` array in your `app.json` (or `app.config.js`):

    ```json app.json
    {
      "expo": {
        "plugins": [
          "@shortkitsdk/react-native"
        ]
      }
    }
    ```

    The plugin automatically sets the iOS deployment target to 16.0 and the Android `minSdkVersion` to 24.

    ### Rebuild native projects

    After adding the plugin, regenerate the native projects:

    ```bash
    npx expo prebuild --clean
    ```

    Then build and run:

    <CodeGroup>

    ```bash iOS
    npx expo run:ios
    ```

    ```bash Android
    npx expo run:android
    ```

    </CodeGroup>

    <Note>
      If you use [EAS Build](https://docs.expo.dev/build/introduction/), the config plugin runs automatically during the cloud build — no need to run `prebuild` locally.
    </Note>
  </Tab>
  <Tab title="Web">
    ### Option A: npm

    ```bash
    npm install @shortkit/web
    ```

    ### Option B: CDN

    Add the script tag to your HTML:

    ```html
    <script src="https://cdn.shortkit.dev/v1/shortkit.min.js"></script>
    ```

    The CDN script exposes a global `ShortKit` object.
  </Tab>
</Tabs>

## Google Ads setup

This section applies only if you added the `ShortKitGoogleAds` product (Swift) or `dev.shortkit:shortkit-google-ads` artifact (Kotlin). Skip it if you are not serving ads.

<Tabs>
  <Tab title="Swift">
    ### 1. Add your Google app ID to Info.plist

    ```xml Info.plist
    <key>GADApplicationIdentifier</key>
    <string>ca-app-pub-XXXXXXXXXXXXXXXX~YYYYYYYYYY</string>
    ```

    Replace the value with your real Google Ad Manager or AdMob application ID.

    ### 2. Initialize the Google Mobile Ads SDK at launch

    ```swift AppDelegate.swift
    import GoogleMobileAds

    func application(
        _ application: UIApplication,
        didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]?
    ) -> Bool {
        MobileAds.shared.start()
        return true
    }
    ```

    Call `MobileAds.shared.start()` once, before creating any `ShortKit` instance with a Google ad provider. Calling it in `application(_:didFinishLaunchingWithOptions:)` is the standard approach.

    ### 3. Pass a Google ad provider to ShortKit

    ```swift
    import ShortKitSDK
    import ShortKitGoogleAds

    let shortKit = ShortKit(
        apiKey: "pk_live_...",
        config: FeedConfig(),
        adProvider: GoogleAdsProvider()
    )
    ```

    `GoogleAdsProvider` runs a waterfall: custom native, then system-defined native, then IMA VAST, then no-fill. For VAST-only integrations, use `GoogleIMAAdProvider()` instead.

    ### Dependencies pulled by ShortKitGoogleAds

    | Dependency | Minimum version | Purpose |
    |---|---|---|
    | `GoogleInteractiveMediaAds` | 3.23.0 | IMA SDK for VAST/VPAID video ads |
    | `GoogleMobileAds` | 12.0.0 | Google Mobile Ads SDK for native ad formats |

    These are resolved automatically by SPM when you add the `ShortKitGoogleAds` product.
  </Tab>
  <Tab title="Kotlin">
    ### 1. Add the Google Ads dependency

    ```kotlin build.gradle.kts
    dependencies {
        implementation("dev.shortkit:shortkit:0.1.0")
        implementation("dev.shortkit:shortkit-google-ads:0.1.0")
    }
    ```

    ### 2. Add your Google app ID to AndroidManifest.xml

    ```xml AndroidManifest.xml
    <application>
        <meta-data
            android:name="com.google.android.gms.ads.APPLICATION_ID"
            android:value="ca-app-pub-XXXXXXXXXXXXXXXX~YYYYYYYYYY" />
    </application>
    ```

    Replace the value with your real Google Ad Manager or AdMob application ID.

    ### 3. Initialize the Google Mobile Ads SDK at launch

    ```kotlin Application.kt
    import com.google.android.gms.ads.MobileAds

    class MyApplication : Application() {
        override fun onCreate() {
            super.onCreate()
            MobileAds.initialize(this)
        }
    }
    ```

    Call `MobileAds.initialize()` once, before creating any `ShortKit` instance with a Google ad provider.

    ### 4. Pass a Google ad provider to ShortKit

    ```kotlin
    import com.shortkit.sdk.ShortKit
    import com.shortkit.sdk.config.FeedConfig
    import com.shortkit.ads.GoogleAdsProvider

    val shortKit = ShortKit(
        context = this,
        apiKey = "pk_live_...",
        config = FeedConfig(),
        adProvider = GoogleAdsProvider(this)
    )
    ```

    ### Dependencies pulled by shortkit-google-ads

    | Dependency | Minimum version | Purpose |
    |---|---|---|
    | `com.google.android.gms:play-services-ads` | 23.6.0 | Google Mobile Ads SDK for native ad formats |
    | `com.google.ads.interactivemedia.v3:interactivemedia` | 3.35.1 | IMA SDK for VAST/VPAID video ads |

    These are resolved automatically by Gradle when you add the `dev.shortkit:shortkit-google-ads` dependency.
  </Tab>
</Tabs>

## Verify installation

Build and run this minimal example to confirm the import resolves and the SDK initializes.

<Tabs>
  <Tab title="Swift (SwiftUI)">
    ```swift App.swift
    import SwiftUI
    import ShortKitSDK

    @main
    struct MyApp: App {
        let shortKit = ShortKit(
            apiKey: "pk_live_your_publishable_key",
            config: FeedConfig()
        )

        var body: some Scene {
            WindowGroup {
                ShortKitFeedView(shortKit: shortKit)
                    .ignoresSafeArea()
            }
        }
    }
    ```
  </Tab>
  <Tab title="Swift (UIKit)">
    ```swift ViewController.swift
    import UIKit
    import ShortKitSDK

    class ViewController: UIViewController {
        override func viewDidLoad() {
            super.viewDidLoad()

            let shortKit = ShortKit(
                apiKey: "pk_live_your_publishable_key",
                config: FeedConfig()
            )

            let feedVC = ShortKitFeedViewController(shortKit: shortKit)
            addChild(feedVC)
            feedVC.view.frame = view.bounds
            feedVC.view.autoresizingMask = [.flexibleWidth, .flexibleHeight]
            view.addSubview(feedVC.view)
            feedVC.didMove(toParent: self)
        }
    }
    ```
  </Tab>
  <Tab title="Kotlin">
    ```kotlin MainActivity.kt
    import android.os.Bundle
    import androidx.appcompat.app.AppCompatActivity
    import com.shortkit.sdk.ShortKit
    import com.shortkit.sdk.config.FeedConfig
    import com.shortkit.sdk.feed.ShortKitFeedFragment

    class MainActivity : AppCompatActivity() {
        override fun onCreate(savedInstanceState: Bundle?) {
            super.onCreate(savedInstanceState)

            val shortKit = ShortKit(
                context = this,
                apiKey = "pk_live_your_publishable_key",
                config = FeedConfig()
            )

            if (savedInstanceState == null) {
                supportFragmentManager.beginTransaction()
                    .replace(android.R.id.content, ShortKitFeedFragment.newInstance(shortKit))
                    .commit()
            }
        }
    }
    ```
  </Tab>
  <Tab title="React Native">

## Troubleshooting

<AccordionGroup>
  <Accordion title="iOS: 'No such module ShortKitSDK'">
    This means Xcode has not resolved the Swift Package dependency.

    1. Verify the package URL is exactly `https://github.com/shortkit/shortkit-ios`
    2. Go to **File** then **Packages** then **Reset Package Caches**
    3. Clean the build folder: **Product** then **Clean Build Folder** (Shift+Cmd+K)
    4. Close and reopen Xcode
    5. Confirm your deployment target is iOS 16.0 or higher in your project settings
  </Accordion>
  <Accordion title="iOS: Deployment target too low">
    ShortKit requires iOS 16.0 or later. If you see a build error referencing a minimum deployment target:

    1. Select your project in the Xcode navigator
    2. Under the **General** tab, set **Minimum Deployments** to iOS 16.0 or higher
    3. If you have multiple targets, update each one
  </Accordion>
  <Accordion title="iOS: XCFramework download fails or checksum mismatch">
    ShortKitSDK is distributed as a pre-compiled XCFramework. If the download fails:

    1. Check your internet connection — the binary is downloaded from GitHub Releases
    2. Go to **File** then **Packages** then **Reset Package Caches** and resolve again
    3. If you see a checksum mismatch, ensure you are on the latest version of the package
  </Accordion>
  <Accordion title="iOS: Google Ads linker errors">
    If you see undefined symbol errors related to Google SDKs after adding `ShortKitGoogleAds`:

    1. Verify the `-ObjC` linker flag is present. SPM sets this automatically, but if you use a wrapper Xcode project, add `-ObjC` under **Build Settings** then **Other Linker Flags**.
    2. Confirm you called `MobileAds.shared.start()` in your app delegate before creating a `GoogleAdsProvider`.
    3. Clean the build folder and resolve packages again.
  </Accordion>
  <Accordion title="Android: Dependency resolution failed">
    If Gradle cannot resolve the ShortKit dependency:

    1. Confirm your project includes Maven Central in its repositories. Most new Android projects include it by default in `settings.gradle.kts`.
    2. Check your internet connection and any proxy configuration
    3. Run `./gradlew --refresh-dependencies` to clear the dependency cache
    4. Confirm your `minSdk` is 24 or higher
  </Accordion>
  <Accordion title="Android: Duplicate class errors">
    If you see duplicate class errors after adding ShortKit, another dependency in your project may include a conflicting version of a transitive dependency (e.g., Media3 or OkHttp).

    1. Run `./gradlew app:dependencies` to inspect the full dependency tree
    2. Use Gradle's `resolutionStrategy` or `constraints` block to force a consistent version
  </Accordion>
  <Accordion title="React Native: Native module not linked">
    If you see an error about `ShortKit` not being found at runtime:

    - **React Native CLI:** Run `cd ios && pod install` and rebuild
    - Verify autolinking is enabled (check `react-native.config.js` is not excluding the package)
    - Clear the Metro cache: `npx react-native start --reset-cache`
  </Accordion>
  <Accordion title="Expo: Config plugin not applied / pod install fails">
    If the native build fails after adding the package:

    1. Confirm `"@shortkitsdk/react-native"` is in the `plugins` array in your `app.json` or `app.config.js`
    2. Run `npx expo prebuild --clean` to regenerate native projects from scratch
    3. If using EAS Build, ensure `@shortkitsdk/react-native` is listed in your `package.json` dependencies (not just devDependencies)
    4. Clear the Metro cache: `npx expo start --clear`
  </Accordion>
  <Accordion title="Web: ShortKit is not defined">
    If you are using the CDN script tag and see `ShortKit is not defined`:

    1. Confirm the script tag loads before your code runs. Place it above your initialization script, or use the `defer` attribute and initialize in a `DOMContentLoaded` listener.
    2. Check the browser console for network errors loading the script.
    3. Verify you are using `ShortKit` (capital S, capital K) — the global is case-sensitive.
  </Accordion>
  <Accordion title="All platforms: Feed loads but shows no content">
    If the SDK initializes successfully but the feed is empty:

    1. Confirm you have uploaded at least one video to your ShortKit account. You can check in the [Admin Portal](https://portal.shortkit.dev) under **Content**.
    2. Verify the video status is **Ready** (processing can take 1-2 minutes after upload)
    3. Check that you are using the correct API key for the environment where your content lives (`pk_live_...` for production content, `pk_test_...` for test content)
  </Accordion>
  <Accordion title="All platforms: 401 Unauthorized error">
    A 401 error means the API key is invalid or not recognized:

    1. Double-check the full key string, including the `pk_live_` or `pk_test_` prefix
    2. Make sure you are using the **publishable key**, not the secret key, in client-side code
    3. Verify the key has not been revoked in the Admin Portal
    4. Confirm the key matches the environment you are targeting
  </Accordion>
</AccordionGroup>
