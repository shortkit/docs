This page documents all configuration options available when initializing the ShortKit SDK. Every option has a sensible default, so you only need to set the fields you want to customize. `FeedConfig()` with no arguments produces a working feed.

## How configuration works

ShortKit uses a three-layer configuration model:

1. **Code defaults** -- Every field in `FeedConfig` has a built-in default value. These are the source of truth when no other configuration is present.
2. **Your code** -- You set fields on `FeedConfig` at initialization time. Anything you do not set keeps the built-in default.
3. **Remote overlay** -- On initialization, the SDK fetches a sparse configuration overlay from the ShortKit API. Any fields present in the remote overlay are merged on top of your code-level config. Fields not present in the overlay are left unchanged.

This model means the feed renders immediately using your code-level config while the remote overlay loads in the background. If the remote fetch times out or fails, your code-level config is used as-is.

Remote overlay enables you to adjust feed behavior, run A/B tests, or roll out configuration changes from the ShortKit dashboard -- without shipping an app update.

### Remote overlay details

- Fetched from `GET /v1/config` on SDK initialization (non-blocking, 2-second timeout)
- Cached to disk with a configurable TTL (default: 5 minutes)
- If a fresh cached overlay exists on disk at init, it is applied immediately while a background refresh occurs
- If the fetch fails and no cache is available, code-level defaults are used

## Initialization parameters

<ResponseField name="apiKey" type="String" required>
  Your organization's publishable API key. Starts with `pk_live_` or `pk_test_`.
</ResponseField>

<ResponseField name="config" type="FeedConfig" required>
  Configuration object defining feed layout, controls, theme, and captions. See `FeedConfig` reference below.
</ResponseField>

<ResponseField name="environment" type="Environment" default=".production">
  Target environment. Controls API endpoint routing and logging verbosity.

  | Value | Description |
  |-------|-------------|
  | `.production` | Production API endpoint |
  | `.staging` | Staging API endpoint |
  | `.development` | Development API endpoint |
  | `.custom(URL)` | Custom API endpoint. Useful for local development or proxy setups. |
</ResponseField>

<ResponseField name="remoteConfigTTL" type="Int" default="300">
  Cache TTL for the remote configuration overlay, in seconds. The SDK serves the cached overlay while refreshing in the background.
</ResponseField>

<ResponseField name="remoteConfigTimeout" type="Int" default="2000">
  Timeout for the remote config fetch, in milliseconds. If the fetch exceeds this duration, the SDK falls back to the cached overlay or code-level defaults.
</ResponseField>

## FeedConfig

The top-level configuration struct for the video feed.

<ResponseField name="feedHeight" type="FeedHeight" default=".fullscreen">
  Height of the feed view.

  | Value | Description |
  |-------|-------------|
  | `.fullscreen` | Occupies the entire screen minus safe areas |
  | `.percentage(CGFloat)` | Percentage of the container (e.g., `.percentage(0.85)` for 85%) |
</ResponseField>

<ResponseField name="controls" type="ControlsConfig" default="ControlsConfig()">
  Control toggles and behavior configuration. See `ControlsConfig` reference below.
</ResponseField>

<ResponseField name="theme" type="ThemeConfig" default="ThemeConfig()">
  Visual theme for all SDK-rendered UI. See the [Theming](/sdk/theming) page for full reference.
</ResponseField>

## ControlsConfig

Each control can be independently enabled or disabled.

<ResponseField name="playPause" type="Bool" default="true">
  Tap-to-pause/play overlay. When enabled, tapping anywhere on the video toggles playback with brief animated icon feedback.
</ResponseField>

<ResponseField name="scrubber" type="Bool" default="true">
  Progress bar and seek scrubber. Shows played progress, buffered progress, and total duration. Draggable for seeking. Styled via `ThemeConfig.scrubberStyle`.
</ResponseField>

<ResponseField name="captions" type="CaptionConfig" default="CaptionConfig()">
  Caption toggle and default state.

  <Expandable title="properties">
    <ResponseField name="enabled" type="Bool" default="true">
      Whether to show the caption toggle button.
    </ResponseField>
    <ResponseField name="defaultOn" type="Bool" default="false">
      Whether captions are visible by default when a video starts playing.
    </ResponseField>
    <ResponseField name="style" type="CaptionStyle" default="CaptionStyle()">
      Caption visual styling. See the [Theming](/sdk/theming#caption-styling) page for full reference.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="volume" type="Bool" default="true">
  Mute/unmute toggle icon visibility.
</ResponseField>

<ResponseField name="muteOnStart" type="Bool" default="true">
  Whether videos start muted. Once the user unmutes, the unmuted state persists across videos for the rest of the session.
</ResponseField>

<ResponseField name="share" type="ShareConfig" default="ShareConfig()">
  Share button configuration.

  <Expandable title="properties">
    <ResponseField name="enabled" type="Bool" default="true">
      Whether to show the share button. When tapped, triggers the native system share sheet.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="autoHideControls" type="AutoHideConfig" default="AutoHideConfig()">
  Auto-hide behavior for control overlays.

  <Expandable title="properties">
    <ResponseField name="enabled" type="Bool" default="true">
      Whether controls automatically hide after a period of inactivity. Any tap reveals them again.
    </ResponseField>
    <ResponseField name="delayMs" type="Int" default="3000">
      Milliseconds of inactivity before controls fade out.
    </ResponseField>
  </Expandable>
</ResponseField>

## Complete example

```swift
import ShortKit

// Initialize with full configuration
let shortKit = ShortKit(
    apiKey: "pk_live_your_publishable_key",
    config: FeedConfig(
        feedHeight: .fullscreen,
        controls: ControlsConfig(
            playPause: true,
            scrubber: true,
            captions: CaptionConfig(
                enabled: true,
                defaultOn: false,
                style: CaptionStyle(
                    fontSize: 16,
                    textColor: .white,
                    backgroundColor: .black,
                    backgroundOpacity: 0.7,
                    position: .bottom
                )
            ),
            volume: true,
            muteOnStart: true,
            share: ShareConfig(enabled: true),
            autoHideControls: AutoHideConfig(
                enabled: true,
                delayMs: 3000
            )
        ),
        theme: ThemeConfig(
            primaryColor: .white,
            backgroundColor: .black,
            controlsTint: .white,
            controlsBackgroundOpacity: 0.4,
            fontFamily: "System",
            scrubberStyle: ScrubberStyle(
                height: 3.0,
                position: .bottom,
                edgeOffset: 0
            ),
            captionStyle: CaptionStyle(
                fontSize: 16,
                position: .bottom
            )
        )
    ),
    environment: .production,
    remoteConfigTTL: 300,
    remoteConfigTimeout: 2000
)

// Optionally identify the user after authentication
shortKit.setUserId("user-456")

// SwiftUI -- drop into any view hierarchy
struct ContentView: View {
    var body: some View {
        ShortKitFeedView(shortKit: shortKit)
    }
}

// UIKit -- push or present
let feedVC = ShortKitFeedViewController(shortKit: shortKit)
navigationController?.pushViewController(feedVC, animated: true)
```

Most integrations only need a few lines. The defaults are production-ready:

```swift
let shortKit = ShortKit(
    apiKey: "pk_live_your_publishable_key",
    config: FeedConfig()
)
```
