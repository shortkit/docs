The ShortKit React Native SDK provides a full-screen, vertically swipeable video feed for React Native apps. It bridges to the native iOS and Android SDKs, delivering native playback performance with a React-idiomatic API.

## Requirements

| Parameter | Value |
|---|---|
| React Native | 0.72+ |
| iOS deployment target | 16.0+ |
| Android minimum API | 24 (Android 7.0) |
| TypeScript | 5.0+ (recommended) |

For Expo:
- Expo SDK 50+
- Development build (not Expo Go)

## Initialization

Wrap your app root with `ShortKitProvider`. This initializes the SDK and makes it available to all child components via context.

```tsx App.tsx
import { ShortKitProvider } from '@shortkit/react-native';

export default function App() {
  return (
    <ShortKitProvider
      apiKey="pk_live_your_publishable_key"
      config={{
        feedHeight: 'fullscreen',
        controls: {
          playPause: true,
          scrubber: true,
          captions: { enabled: true, defaultOn: false },
          volume: true,
          muteOnStart: true,
          share: { enabled: true },
        },
        theme: {
          primaryColor: '#FFFFFF',
          controlsTint: '#FFFFFF',
          backgroundColor: '#000000',
        },
      }}
      environment="production"
    >
      <NavigationContainer>
        {/* Your app navigation */}
      </NavigationContainer>
    </ShortKitProvider>
  );
}
```

<Warning>
  Never expose your secret key (`sk_live_...` or `sk_test_...`) in client-side code, source control, or your app binary. Use only your publishable key (`pk_live_...`) for SDK initialization.
</Warning>

## Displaying the feed

Use the `ShortKitFeed` component anywhere in your tree. It handles its own layout, playback, and gesture recognition.

```tsx VideoFeedScreen.tsx
import React from 'react';
import { StyleSheet, View } from 'react-native';
import { ShortKitFeed } from '@shortkit/react-native';

export function VideoFeedScreen() {
  return (
    <View style={styles.container}>
      <ShortKitFeed style={styles.feed} />
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1 },
  feed: { flex: 1 },
});
```

### With React Navigation

```tsx MainTabs.tsx
import { createBottomTabNavigator } from '@react-navigation/bottom-tabs';
import { ShortKitFeed } from '@shortkit/react-native';

const Tab = createBottomTabNavigator();

function VideoTab() {
  return <ShortKitFeed style={{ flex: 1 }} />;
}

export function MainTabs() {
  return (
    <Tab.Navigator>
      <Tab.Screen name="Home" component={HomeScreen} />
      <Tab.Screen name="Videos" component={VideoTab} />
      <Tab.Screen name="Profile" component={ProfileScreen} />
    </Tab.Navigator>
  );
}
```

### Partial-height feed

```tsx PartialFeedScreen.tsx
import { ShortKitFeed } from '@shortkit/react-native';

export function PartialFeedScreen() {
  return (
    <View style={{ flex: 1 }}>
      <ShortKitFeed
        style={{ flex: 1 }}
        feedHeight={{ percentage: 0.85 }}
      />
    </View>
  );
}
```

## Configuration

### FeedConfig

<ResponseField name="feedHeight" type="'fullscreen' | { percentage: number }" default="'fullscreen'">
  Height of the feed. `'fullscreen'` occupies the entire screen minus safe areas. Pass `{ percentage: 0.85 }` for 85% height.
</ResponseField>

<ResponseField name="controls" type="ControlsConfig" default="{}">
  Control visibility and behavior. See ControlsConfig below.
</ResponseField>

<ResponseField name="theme" type="ThemeConfig" default="{}">
  Visual styling for all SDK-rendered UI. See ThemeConfig below.
</ResponseField>

### ControlsConfig

<ResponseField name="playPause" type="boolean" default="true">
  Show tap-to-pause/play overlay.
</ResponseField>

<ResponseField name="scrubber" type="boolean" default="true">
  Show the progress bar / seek scrubber.
</ResponseField>

<ResponseField name="captions" type="CaptionConfig" default="{ enabled: true, defaultOn: false }">
  Caption toggle visibility and default state.
</ResponseField>

<ResponseField name="volume" type="boolean" default="true">
  Show the mute/unmute toggle.
</ResponseField>

<ResponseField name="muteOnStart" type="boolean" default="true">
  Whether videos start muted. Once the user unmutes, the unmuted state persists across videos within a session.
</ResponseField>

<ResponseField name="share" type="ShareConfig" default="{ enabled: true }">
  Share button configuration.
</ResponseField>

<ResponseField name="autoHideControls" type="AutoHideConfig" default="{ enabled: true, delayMs: 3000 }">
  Auto-hide controls after a period of inactivity.
</ResponseField>

### ThemeConfig

<ResponseField name="primaryColor" type="string" default="'#FFFFFF'">
  Accent color for interactive elements (scrubber fill, active toggles). Hex string.
</ResponseField>

<ResponseField name="backgroundColor" type="string" default="'#000000'">
  Feed background color, visible during transitions. Hex string.
</ResponseField>

<ResponseField name="controlsTint" type="string" default="'#FFFFFF'">
  Tint color for all control icons. Hex string.
</ResponseField>

<ResponseField name="controlsBackgroundOpacity" type="number" default="0.4">
  Opacity of the gradient behind controls for visibility over video.
</ResponseField>

<ResponseField name="fontFamily" type="string" default="'System'">
  Font for all SDK-rendered text (captions, labels).
</ResponseField>

<ResponseField name="scrubberStyle" type="ScrubberStyle" default="{}">
  Detailed scrubber styling: height, knob size, played/buffered/unplayed colors.
</ResponseField>

<ResponseField name="captionStyle" type="CaptionStyle" default="{}">
  Caption appearance: font size, text color, background, position.
</ResponseField>

### Remote configuration

The SDK fetches a remote configuration overlay on initialization. Server-side overrides merge on top of your code-level defaults, enabling A/B testing and remote control without app updates.

Configuration resolves in this order:

1. Your code-level config renders the feed immediately.
2. A remote overlay loads in the background (2-second timeout, non-blocking).
3. Server-provided fields override your defaults. Fields not included in the overlay keep your values.
4. The overlay is cached to disk with a configurable TTL (default 5 minutes).

## User identity

ShortKit generates an anonymous device ID on first initialization. All engagement tracking and feed personalization work immediately without requiring a user ID.

Use the `useShortKit` hook to manage identity:

```tsx
import { useShortKit } from '@shortkit/react-native';

function ProfileScreen() {
  const { setUserId, clearUserId } = useShortKit();

  async function handleLogin() {
    const user = await authenticateUser();
    setUserId(user.id);
  }

  function handleLogout() {
    clearUserId();
  }

  return (
    // Your profile UI
  );
}
```

<Note>
  Calling `setUserId` triggers backend identity resolution, merging the anonymous session's engagement data with the identified user profile. Call `clearUserId` when the user logs out.
</Note>

## useShortKit hook

The `useShortKit` hook provides access to identity methods and SDK state from any component inside `ShortKitProvider`.

```tsx
import { useShortKit } from '@shortkit/react-native';

function MyComponent() {
  const { setUserId, clearUserId } = useShortKit();

  // ...
}
```

| Property | Type | Description |
|---|---|---|
| `setUserId` | `(id: string) => void` | Transition from anonymous to identified mode. |
| `clearUserId` | `() => void` | Revert to anonymous mode. Generates a new anonymous device ID. |

## TypeScript types

The SDK ships with full TypeScript definitions. All config objects, components, and hooks are fully typed.

```typescript
import type {
  FeedConfig,
  ControlsConfig,
  ThemeConfig,
  CaptionConfig,
  ShareConfig,
  AutoHideConfig,
  ScrubberStyle,
  CaptionStyle,
  FeedHeight,
  CaptionPosition,
} from '@shortkit/react-native';

// FeedHeight is a union type
type FeedHeight = 'fullscreen' | { percentage: number };

// CaptionPosition is a string literal union
type CaptionPosition = 'top' | 'bottom';
```

## Environment

Control which ShortKit environment the SDK connects to:

```tsx
<ShortKitProvider
  apiKey="pk_live_your_publishable_key"
  config={{ /* ... */ }}
  environment="production"
>
```

| Environment | Description |
|---|---|
| `'production'` | Live environment. Use in release builds. |
| `'staging'` | Pre-release testing environment. |
| `'development'` | Local development. Enables debug logging in the native layer. |

<Tip>
  Use `__DEV__` to switch environments automatically:
  ```tsx
  environment={__DEV__ ? 'development' : 'production'}
  ```
</Tip>
