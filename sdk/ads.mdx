Monetize your ShortKit feed with full-screen in-feed video ads. Ad placement is server-driven -- you configure frequency, position, and format in the ShortKit dashboard. The SDK handles prefetching, rendering, and automatic cleanup.

## Enabling ads

Pass an `adProvider` to the `ShortKit` initializer. When `adProvider` is `nil` (the default), all ad slots in the feed response are stripped out.

<Tabs>
  <Tab title="iOS">
    ```swift
    import ShortKitSDK
    import ShortKitGoogleAds

    let shortKit = ShortKit(
        apiKey: "pk_live_...",
        config: FeedConfig(),
        adProvider: GoogleAdsProvider()
    )
    ```
  </Tab>
  <Tab title="Android">
    ```kotlin
    import dev.shortkit.ShortKit
    import dev.shortkit.FeedConfig
    import dev.shortkit.ads.google.GoogleAdsProvider

    val shortKit = ShortKit(
        apiKey = "pk_live_...",
        config = FeedConfig(),
        adProvider = GoogleAdsProvider()
    )
    ```
  </Tab>
  <Tab title="React Native">
    Ad integration for React Native is coming soon. Ads are currently supported on iOS and Android native SDKs only.
  </Tab>
</Tabs>

The `adProvider` parameter is the only ad-specific configuration on `ShortKit`. Ad slot positions, frequency, format, and skip policy are all controlled server-side via the feed API response.

## Built-in providers

ShortKit ships two Google ad adapters in a separate `ShortKitGoogleAds` module that has no impact on your binary size if you do not import it.

### GoogleAdsProvider (recommended)

The composite provider runs a waterfall strategy for each ad slot:

1. **Custom native** -- plays the ad video through ShortKit's own player. Best UX: instant playback, looping, seamless swipe transitions.
2. **System-defined native** -- renders using Google's `MediaView`. Good UX.
3. **IMA VAST** -- renders using the Google IMA SDK's black-box player. Acceptable UX.
4. **No fill** -- collapses the ad slot from the feed. The user never sees an empty slot.

The waterfall path is driven by the `adFormat` field on each ad slot (set server-side):

| `adFormat` value | Waterfall behavior |
|---|---|
| `"native"` | Custom native, then system native, then IMA VAST, then no-fill |
| `"vast"` | Skips native, goes directly to IMA VAST |
| `nil` (default) | Same as `"vast"` |

| Parameter | Type | Default | Description |
|---|---|---|---|
| `nativeLoadTimeout` | `TimeInterval` | `6.0` | Maximum seconds to wait for a native ad to load before falling through to the next waterfall step. |
| `imaLoadTimeout` | `TimeInterval` | `8.0` | Maximum seconds to wait for an IMA VAST ad to load. |
| `language` | `String?` | `nil` | BCP 47 language tag for ad UI elements. `nil` uses the device locale. |
| `enableDebugMode` | `Bool` | `false` | Shows the IMA debug overlay. Use during development only. |

All parameters are optional. `GoogleAdsProvider()` with no arguments is production-ready.

## How ads work

### Server-driven placement

Ad slots are defined server-side and delivered as `FeedItem` entries with type `"ad_slot"` in the feed API response. The server controls frequency, position, format, skip policy, and targeting. There are no client-side configuration options for ad placement.

### Prefetching

The SDK prefetches ads within a 3-item window ahead of the current feed position. Prefetching is triggered on initial feed load, after each page load, and after each swipe. For custom native ads with video URLs, the video is additionally prefetched into the player pool for instant playback.

### Auto-collapse on failure

When an ad fails to load (no fill, timeout, network error, provider error), the ad slot is removed from the feed with an animation. The user never sees an empty or error state for an ad slot.

### Auto-advance on completion

When an ad finishes playing or is skipped, the feed auto-advances to the next item and the ad slot is collapsed out of the feed.

## Ad formats

All ads are **in-feed interstitials** -- full-screen ad cells inserted between content items in the vertical swipe feed. There are no pre-roll or mid-roll ads.

### VAST video (IMA)

Full-screen video rendered by the Google IMA SDK. IMA handles its own playback controls, skip button, and click-through browser.

### Custom native video

Full-screen video played through ShortKit's own player. Provides the best user experience:

- Instant playback (video is prefetched into the player pool)
- Seamless swipe transitions identical to content
- Looping playback
- Native overlay with "Ad" badge, AdChoices icon, sponsor logo, headline, and CTA button

### System-defined native

Full-screen ad rendered using Google's `GADNativeAdView` with `GADMediaView`. The view hierarchy is pre-built during load to eliminate black-frame flicker. Includes video playback, dim overlay, headline, and CTA button.

## Skip policies

Skip behavior is set per ad slot on the server. Three policies are available:

<Tabs>
  <Tab title="iOS">
    ```swift
    public enum SkipPolicy: Sendable, Equatable {
        case nonSkippable                          // Ad cannot be skipped
        case skippableAfter(seconds: TimeInterval) // Skippable after N seconds
        case alwaysSkippable                       // Skippable immediately
    }
    ```
  </Tab>
  <Tab title="Android">
    ```kotlin
    sealed class SkipPolicy {
        object NonSkippable : SkipPolicy()
        data class SkippableAfter(val seconds: Double) : SkipPolicy()
        object AlwaysSkippable : SkipPolicy()
    }
    ```
  </Tab>
  <Tab title="React Native">
    ```typescript
    type SkipPolicy =
      | { type: 'nonSkippable' }
      | { type: 'skippableAfter'; seconds: number }
      | { type: 'alwaysSkippable' };
    ```
  </Tab>
</Tabs>

| Policy | Behavior |
|---|---|
| `nonSkippable` | The ad plays to completion. The user cannot skip or swipe past it. |
| `skippableAfter(seconds:)` | A skip button appears after the specified number of seconds. |
| `alwaysSkippable` | A skip button is visible from the start. |

When the user swipes away from a skippable ad, it counts as a skip and the accumulated watch time is tracked.

## Custom ad providers

Implement the `ShortKitAdProvider` protocol to integrate any ad SDK.

<Tabs>
  <Tab title="iOS">
    ```swift
    public protocol ShortKitAdProvider: AnyObject {
        /// Called when the slot enters the prefetch window (~3 items ahead).
        func loadAd(
            for slot: AdSlot,
            completion: @escaping (Result<PreparedAd, AdError>) -> Void
        )

        /// Display a previously loaded ad in the provided container view.
        func displayAd(
            _ ad: PreparedAd,
            in container: UIView,
            presentingViewController: UIViewController,
            delegate: ShortKitAdDelegate
        )

        /// Dismiss an ad (user swiped away, ad completed, or error).
        func dismissAd(_ ad: PreparedAd)
    }
    ```
  </Tab>
  <Tab title="Android">
    ```kotlin
    interface ShortKitAdProvider {
        fun loadAd(slot: AdSlot, callback: AdLoadCallback)
        fun displayAd(ad: PreparedAd, container: ViewGroup, activity: Activity, delegate: ShortKitAdDelegate)
        fun dismissAd(ad: PreparedAd)
    }
    ```
  </Tab>

### PreparedAd

Returned by `loadAd` on success. Carries the loaded ad data for later display.

When `nativeContent` is non-nil, the feed controller plays the video through the player pool instead of delegating to the provider's rendering. Set `prebuiltView` during `loadAd` to eliminate black-frame flicker when the user scrolls to the ad cell.

## Ad lifecycle delegate

The `ShortKitAdDelegate` protocol receives lifecycle callbacks from ad providers. The SDK implements this internally on the feed view controller to drive analytics and UI transitions. Ad providers must call these methods at the appropriate times.

<Tabs>
  <Tab title="iOS">
    ```swift
    public protocol ShortKitAdDelegate: AnyObject {
        func adDidStart(_ ad: PreparedAd)
        func adDidReachQuartile(_ ad: PreparedAd, quartile: AdQuartile)
        func adDidComplete(_ ad: PreparedAd)
        func adWasSkipped(_ ad: PreparedAd, watchTime: TimeInterval)
        func adDidFail(_ ad: PreparedAd, error: AdError)
        func adDidReceiveClick(_ ad: PreparedAd)
    }
    ```
  </Tab>
  <Tab title="Android">
    ```kotlin
    interface ShortKitAdDelegate {
        fun adDidStart(ad: PreparedAd)
        fun adDidReachQuartile(ad: PreparedAd, quartile: AdQuartile)
        fun adDidComplete(ad: PreparedAd)
        fun adWasSkipped(ad: PreparedAd, watchTime: Double)
        fun adDidFail(ad: PreparedAd, error: AdError)
        fun adDidReceiveClick(ad: PreparedAd)
    }
    ```
  </Tab>
  <Tab title="React Native">
    Ad integration for React Native is coming soon. Ads are currently supported on iOS and Android native SDKs only.
  </Tab>
</Tabs>

| Callback | Analytics event | UI effect |
|---|---|---|
| `adDidStart` | `adStart` | None |
| `adDidReachQuartile` | `adProgress` (with quartile name) | None |
| `adDidComplete` | `adComplete` | Auto-advance to next item, collapse ad slot |
| `adWasSkipped` | `adSkip` (with watch time) | Auto-advance to next item, collapse ad slot |
| `adDidFail` | None | Collapse ad slot immediately |
| `adDidReceiveClick` | None | Click-through handled by provider |

## Required setup for Google ads

If you are using `GoogleAdsProvider` or `GoogleIMAAdProvider`, complete the following setup in your host app.

### 1. Add the ShortKitGoogleAds dependency

<Tabs>
  <Tab title="iOS">
    Add `ShortKitGoogleAds` to your target alongside `ShortKit`:

    ```swift Package.swift
    dependencies: [
        .package(url: "https://github.com/AudienceKit/shortkit-ios.git", from: "1.0.0")
    ],
    targets: [
        .target(
            name: "YourApp",
            dependencies: [
                .product(name: "ShortKitSDK", package: "shortkit-ios"),
                .product(name: "ShortKitGoogleAds", package: "shortkit-ios")
            ]
        )
    ]
    ```

    `ShortKitGoogleAds` transitively pulls in:
    - `GoogleInteractiveMediaAds` >= 3.23.0 (IMA SDK)
    - `GoogleMobileAds` >= 12.0.0

    The `-ObjC` linker flag is configured automatically by the SPM package.
  </Tab>
  <Tab title="Android">
    Add the Google ads companion module to your `build.gradle.kts`:

    ```kotlin build.gradle.kts
    dependencies {
        implementation("dev.shortkit:shortkit-android:1.0.0")
        implementation("dev.shortkit:shortkit-google-ads:1.0.0")
    }
    ```
  </Tab>
  <Tab title="React Native">
    Ad integration for React Native is coming soon. Ads are currently supported on iOS and Android native SDKs only.
  </Tab>
</Tabs>

### 2. Add your Google Ad Manager app ID

<Tabs>
  <Tab title="iOS">
    Add your `GADApplicationIdentifier` to `Info.plist`:

    ```xml Info.plist
    <key>GADApplicationIdentifier</key>
    <string>ca-app-pub-XXXXXXXXXXXXXXXX~YYYYYYYYYY</string>
    ```
  </Tab>
  <Tab title="Android">
    Add the `APPLICATION_ID` meta-data to your `AndroidManifest.xml`:

    ```xml AndroidManifest.xml
    <meta-data
        android:name="com.google.android.gms.ads.APPLICATION_ID"
        android:value="ca-app-pub-XXXXXXXXXXXXXXXX~YYYYYYYYYY" />
    ```
  </Tab>
  <Tab title="React Native">
    Follow the iOS and Android steps above for each platform in your native project files.
  </Tab>
</Tabs>

### 3. Initialize the Google Mobile Ads SDK

Call `MobileAds.shared.start()` at app launch, before initializing ShortKit. This is a Google requirement for all apps using the Google Mobile Ads SDK.

<Tabs>
  <Tab title="iOS">
    ```swift AppDelegate.swift
    import GoogleMobileAds

    func application(
        _ application: UIApplication,
        didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]?
    ) -> Bool {
        MobileAds.shared.start()
        return true
    }
    ```
  </Tab>
  <Tab title="Android">
    ```kotlin Application.kt
    import com.google.android.gms.ads.MobileAds

    class MyApplication : Application() {
        override fun onCreate() {
            super.onCreate()
            MobileAds.initialize(this)
        }
    }
    ```
  </Tab>
  <Tab title="React Native">
    Ad integration for React Native is coming soon. Ads are currently supported on iOS and Android native SDKs only.
  </Tab>
</Tabs>

### 4. Configure ad units in Google Ad Manager

Set up your ad unit IDs and (for custom native ads) custom native format IDs in the Google Ad Manager dashboard. These values are referenced server-side in your ShortKit ad slot configuration -- you do not need to hardcode them in your app.
