The feed is the core UI surface of ShortKit -- a vertically-snapping, full-screen video player that handles pagination, player pooling, ads, overlays, captions, and engagement tracking. You create a `ShortKit` instance and hand it to one of two feed components depending on your UI framework.

## ShortKitFeedViewController (UIKit)

```swift
public class ShortKitFeedViewController: UIViewController {
    public init(shortKit: ShortKit)
}
```

`ShortKitFeedViewController` renders the feed as a vertically-snapping `UICollectionView`. It manages the entire lifecycle internally: pagination, player rotation, ad loading, overlay rendering, captions, and memory cleanup. All state is driven by the `ShortKit` instance you pass at init.

### Embedding in a navigation stack

<Tabs>
  <Tab title="iOS (UIKit)">

    ```swift ViewController.swift
    import ShortKit

    class ViewController: UIViewController {

            shortKit = ShortKit(
                apiKey: "pk_live_your_publishable_key",

            navigationController?.pushViewController(feedVC, animated: true)
    ```

### Embedding as a child view controller

Use standard UIKit container view controller APIs when you need the feed to occupy part of the screen:

```swift
let feedVC = ShortKitFeedViewController(shortKit: shortKit)
addChild(feedVC)
feedVC.view.frame = view.bounds
feedVC.view.autoresizingMask = [.flexibleWidth, .flexibleHeight]
view.addSubview(feedVC.view)
feedVC.didMove(toParent: self)
```

### Lifecycle

`ShortKitFeedViewController` responds to standard UIKit lifecycle events. Playback pauses automatically when the view controller disappears and resumes when it reappears. You do not need to call `play()` or `pause()` manually during navigation transitions.

## ShortKitFeedView (SwiftUI)

```swift
public struct ShortKitFeedView: UIViewControllerRepresentable {
    public init(shortKit: ShortKit)
}
```

`ShortKitFeedView` is a `UIViewControllerRepresentable` wrapper around `ShortKitFeedViewController`. It works anywhere a SwiftUI `View` is accepted.

<Tabs>
  <Tab title="iOS (SwiftUI)">
    ```swift ContentView.swift
    import SwiftUI
    import ShortKit

    struct ContentView: View {
        let shortKit = ShortKit(
            apiKey: "pk_live_your_publishable_key",
            config: FeedConfig()
        )

        var body: some View {
            ShortKitFeedView(shortKit: shortKit)
                .ignoresSafeArea()
        }
    }
    ```
  </Tab>
  <Tab title="Android">
    ```kotlin
    import dev.shortkit.ShortKit
    import dev.shortkit.FeedConfig
    import dev.shortkit.ShortKitFeedView

    @Composable
    fun FeedScreen() {
        val shortKit = remember {
            ShortKit(
                apiKey = "pk_live_your_publishable_key",
                config = FeedConfig()
            )
        }

        ShortKitFeedView(shortKit)
    }
    ```
  </Tab>
  <Tab title="React Native">
    ```tsx
    import React from 'react';
    import { ShortKit, ShortKitFeed } from '@shortkit/react-native';

    const shortKit = ShortKit.initialize({
      apiKey: 'pk_live_your_publishable_key',
    ```

<Warning>
  Store the `ShortKit` instance outside the view body. Creating it inside the `body` property causes re-initialization on every SwiftUI render pass.
</Warning>

### Partial-height feed in SwiftUI

Use `FeedConfig.feedHeight` to make the feed occupy a portion of the screen:

```swift
let shortKit = ShortKit(
    apiKey: "pk_live_your_publishable_key",
    config: FeedConfig(feedHeight: .percentage(0.75))
)

var body: some View {
    VStack {
        ShortKitFeedView(shortKit: shortKit)
        CustomBottomBar()
    }
}
```

## Feed item types

The feed is a sequence of `FeedItem` values. Each item is a discriminated union (tagged enum) that the SDK decodes from the API response and renders with the appropriate cell type.

```swift
public enum FeedItem: Sendable, Hashable, Codable {
    case content(ContentItem)
    case imageCarousel(ImageCarouselItem)
    case survey(SurveyItem)
    case adSlot(AdSlot)

    public var id: String
}
```

### `.content(ContentItem)`

A short-form video. This is the primary feed item type. Contains the HLS streaming URL, thumbnail, caption tracks, and optional news metadata (author, article URL, comment count).

```swift
public struct ContentItem: Codable, Equatable, Sendable {
    public let id: String
    public let title: String
    public let description: String?
    public let duration: Double
    public let streamingUrl: String
    public let thumbnailUrl: String
    public let captionTracks: [CaptionTrack]
    public let customMetadata: [String: JSONValue]?
    public let author: String?
    public let articleUrl: String?
    public let commentCount: Int?
}
```

See the [Player](/sdk/reference) documentation for controlling video playback via `shortKit.player`.

### `.imageCarousel(ImageCarouselItem)`

A horizontally-paged set of images with optional auto-scroll. Rendered inline in the feed between videos. Configure the carousel UI with `FeedConfig.carouselMode`.

```swift
public struct ImageCarouselItem: Codable, Equatable, Sendable {
    public let id: String
    public let images: [CarouselImage]
    public let autoScrollInterval: Double?
    public let caption: String?
    public let title: String?
    public let description: String?
    public let author: String?
    public let section: String?
    public let articleUrl: String?
}
```

Each `CarouselImage` has a `url` and optional `alt` text.

### `.survey(SurveyItem)`

An inline poll or question with tappable options. After the user selects an option, the SDK fires the `didRespondToSurvey` delegate callback and auto-advances to the next item after `autoAdvanceDelay` seconds. Configure the survey UI with `FeedConfig.surveyMode`.

```swift
public struct SurveyItem: Codable, Equatable, Sendable {
    public let id: String
    public let question: String
    public let options: [SurveyOption]
    public let autoAdvanceDelay: Double?
}
```

Each `SurveyOption` has an `id` and display `text`.

### `.adSlot(AdSlot)`

A placeholder for a monetization unit. The SDK calls your `ShortKitAdProvider` to load and display an ad when this slot enters the viewport. Ad slots are inserted by the backend at configured intervals. See the [Ad Integration](/sdk/ads) guide for setup.

```swift
public struct AdSlot: Codable, Sendable, Equatable {
    public let id: String
    public let position: Int
    public let adTagUrl: String?
    public let adUnitId: String?
    public let targeting: [String: String]
    public let skipPolicy: SkipPolicy
}
```

### Convenience accessors

Every `FeedItem` exposes type-check booleans and unwrap accessors:

```swift
let item: FeedItem = ...

item.isContent        // true if .content
item.isImageCarousel  // true if .imageCarousel
item.isSurvey         // true if .survey
item.isAdSlot         // true if .adSlot
item.isNonVideo       // true for carousel, survey, and ad items

item.contentItem          // ContentItem? -- nil unless .content
item.imageCarouselItem    // ImageCarouselItem? -- nil unless .imageCarousel
item.surveyItem           // SurveyItem? -- nil unless .survey
item.adSlot               // AdSlot? -- nil unless .adSlot
```
